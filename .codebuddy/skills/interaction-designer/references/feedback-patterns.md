# 反馈模式案例库

## 常见产品的交互反馈对标

### 微信

| 操作 | 反馈 |
|------|------|
| 发消息 | 即时上屏（乐观更新）+ 发送中圆圈 + 失败红色感叹号 |
| 点赞朋友圈 | 无 loading，即时切换❤️状态（乐观） |
| 下拉刷新 | 自定义动画 + 松手后 spinner |
| 转账 | 密码键盘 + 指纹 → 全屏成功动画 |
| 撤回消息 | 即时消失 + "你撤回了一条消息" 系统提示 |

### 抖音

| 操作 | 反馈 |
|------|------|
| 双击点赞 | 大爱心动画浮出 + 右侧❤️变红跳动 |
| 上滑切换视频 | 下一个视频从底部推入 + 无缝衔接播放 |
| 评论发送 | 即时插入到评论列表顶部（乐观） |
| 关注 | 按钮即时变"已关注" + 轻微 scale 弹跳 |

### Notion

| 操作 | 反馈 |
|------|------|
| 输入文字 | 实时保存，无需手动 + 左上角"保存中..." |
| 拖拽 block | 蓝色插入线指示位置 + 释放后滑动到位 |
| 删除页面 | 底部 Snackbar "已移到回收站" + 撤销按钮(5s) |
| 导出 | 顶部进度条 + 完成后弹下载链接 |

## 反馈层级决策树

```
用户操作
├── 结果可预测（点赞/收藏/开关）
│   └── L1 微反馈: 乐观更新 + 图标动画
├── 需要网络但很快（<1s）
│   └── L2 轻提示: 短暂 loading + Toast 结果
├── 需要网络且可能慢（1-5s）
│   └── L3 中反馈: 持续 loading + 成功/失败处理
├── 不可逆操作
│   └── L4 强确认: Modal 二次确认 + 操作中 loading
└── 改变整个页面状态
    └── L5 全屏: 骨架屏/空态/错误占位
```

## 乐观更新 vs 悲观更新

| 策略 | 适用场景 | 实现 |
|------|---------|------|
| 乐观更新 | 成功率 > 99%（点赞/收藏/已读） | 先改 UI 再发请求，失败回滚 |
| 悲观更新 | 涉及金额/权限/不可逆 | 等请求成功再改 UI |
| 半乐观 | 列表添加/删除 | 先在 UI 标记"处理中"，成功确认/失败回滚 |

## Toast 设计规范

```
位置: 顶部居中（移动端）/ 右上角（桌面端）
层级: z-index 最高，不被任何元素遮挡
时长: 
  - 成功: 2s（用户不需要操作）
  - 警告: 3s（用户可能需要注意）
  - 错误: 4s 或手动关闭（用户需要知道出了什么问题）
动画: 从上方 slide-down + fade-in, 200ms ease-out
消失: fade-out + slide-up, 150ms ease-in
堆叠: 最多显示 3 条，新的把旧的往下推
```

## Shake 动画规范

```css
@keyframes shake {
  0%, 100% { transform: translateX(0); }
  10%, 50%, 90% { transform: translateX(-4px); }
  30%, 70% { transform: translateX(4px); }
}

/* 用法: 表单校验失败时对容器应用 */
.shake {
  animation: shake 400ms ease-in-out;
}
```

触发场景：
- 密码错误（登录/注册）
- 必填字段为空点提交
- 验证码错误
