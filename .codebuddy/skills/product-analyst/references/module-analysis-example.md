# 模块分析示例 — 注册模块

> 以下展示 Phase 2 五步分析法（Step A-E）的完整示例，供 AI 参考格式和深度。

## Step A: 用户故事设计

以"作为[角色]，我希望[功能]，以便[价值]"的格式，穷举该模块的所有用户故事：

```markdown
### 注册模块 - 用户故事

核心故事：
- 作为新用户，我希望用手机号快速注册，以便尽快体验产品
- 作为新用户，我希望用微信一键登录，以便不用记密码

扩展故事：
- 作为已注册用户，我希望能修改手机号，以便号码变更后继续使用
- 作为用户，我希望设置密码找回方式，以便忘记密码时恢复

异常故事：
- 作为用户，如果手机号已被注册，我希望得到明确提示和指引
- 作为用户，如果网络异常，我希望不会丢失已填写的信息
```

## Step B: 核心流程设计

```markdown
### 注册流程

主线流程（Happy Path）：
1. 进入注册页 → 2. 填写手机号 → 3. 获取验证码 → 4. 填写验证码 
→ 5. 设置密码 → 6. 填写基本信息(昵称/学校) → 7. 注册成功 → 8. 进入主页

异常分支：
├── Step 2: 手机号格式错误 → 即时校验提示
├── Step 2: 手机号已注册 → 引导去登录 + 找回密码
├── Step 3: 60秒内重复获取 → 倒计时禁用
├── Step 3: 短信发送失败 → 提示"稍后重试" + 备用方案(语音)
├── Step 4: 验证码错误 → 提示重新输入(3次后重新获取)
├── Step 4: 验证码过期(5分钟) → 提示重新获取
├── Step 5: 密码不符合规则 → 实时校验 + 规则说明
├── Step 6: 昵称违规 → 内容审核提示
└── Step 7: 网络中断 → 本地暂存 + 恢复重试
```

## Step C: 业务规则 & 边界 Case

```markdown
### 注册模块 - 业务规则

数据规则：
- 手机号：11位，+86，唯一性校验
- 验证码：6位数字，5分钟有效，同号60秒间隔
- 密码：6-20位，至少含字母+数字
- 昵称：2-16字符，不含特殊符号，唯一性可选
- 学校：从预设列表选择（覆盖全国高校）

业务规则：
- 同一手机号每日最多获取10次验证码
- 注册后24小时内未完善资料 → push 提醒
- 新用户首次登录引导流程（3步引导卡片）
- 注册来源追踪（邀请码/分享链接/自然流量）

安全规则：
- 密码 bcrypt 加密存储
- 注册接口限流（同 IP 每分钟5次）
- 短信通道防刷（图形验证码前置）
- 设备指纹识别（防批量注册）

边界 Case：
- 用户注册到一半退出 → 30分钟内回来能恢复进度
- 用户换设备继续注册 → 不能恢复（安全考虑）
- 同时在两个设备注册同一手机号 → 先完成的成功
- 学校不在列表中 → 提供"其他"选项 + 手动输入 + 后台审核
```

## Step D: 数据模型概要

```markdown
### 注册模块 - 数据需求

涉及实体：
- User（核心）：phone, password_hash, nickname, school, avatar, status, created_at
- VerifyCode（辅助）：phone, code, type, expires_at, used, attempts
- UserProfile（扩展）：user_id, bio, gender, grade, interests[]

涉及接口：
- POST /api/auth/send-code — 发送验证码
- POST /api/auth/register — 提交注册
- POST /api/auth/check-phone — 手机号查重
```

## Step E: 交互要点

```markdown
### 注册模块 - 交互设计要点

- 表单实时校验（输入即反馈，不等提交）
- 密码强度可视化指示器
- 验证码输入框自动聚焦 + 粘贴自动填充
- 注册按钮 loading 态（防重复提交）
- 学校选择器：支持拼音搜索 + 热门推荐
- 错误提示：inline 展示（不用 toast 打断流程）
- 页面切换动画：左右滑动（暗示进度）
```
