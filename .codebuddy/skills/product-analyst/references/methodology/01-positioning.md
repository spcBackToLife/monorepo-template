# 产品定位四要素

每个产品分析的起点。Phase 1 的 P-overview 任务必须给出全部 4 项答案。

## 四要素

### 1. 一句话定位

**目标**：用一句话讲清楚"这个产品是什么 / 为谁解决什么问题"。

**测试标准**：把这句话给一个不了解这个项目的人看，对方能否理解？

**反例**：
- ❌ "一款现代化的高效协作工具"（空洞，谁都能套）
- ❌ "为年轻人打造的社交平台"（什么社交？什么场景？）

**正例**：
- ✅ "为大学生打造的基于地理位置的同城同好匹配 App，核心场景是通过共同兴趣发现 1km 内可线下见面的同学"
- ✅ "面向程序员的命令行 AI 助手，在终端里直接对话不切上下文"

### 2. 核心价值

**目标**：回答"为什么用户要用这个，而不是现有方案"。

**思考提示**：
- 用户当前怎么解决这个问题？（替代方案）
- 你的方案在哪个维度做到 10 倍好？
- 不可被替代的差异化在哪？

**注意**：核心价值不是 feature 列表，是**用户感知的核心收益**。

### 3. 目标用户（Primary / Secondary 分层）

**目标**：用户画像分层，至少 Primary + Secondary 两层。

**字段建议**：
- 年龄段
- 职业 / 身份
- 主要使用场景
- 核心痛点
- 现有替代方案

**反例**：
- ❌ "所有人"（没有产品是给所有人的）
- ❌ "18-35 岁年轻人"（只有人口学，没有场景和痛点）

### 4. 使用场景

**目标**：高频 / 中频 / 低频场景分别穷举，每个场景标注**设备 / 网络 / 情绪状态**。

**模板**：

| 频次 | 场景 | 设备 | 网络 | 情绪 |
|------|------|------|------|------|
| 高频 | 通勤路上刷信息流 | 手机 | 4G/弱网 | 碎片化、走神 |
| 中频 | 晚上睡前发动态 | 手机 | Wi-Fi | 放松、分享欲 |
| 低频 | 个人资料维护 | 手机 | Wi-Fi | 专注 |

**关键洞察**：弱网 / 碎片化 / 走神状态决定了 UI 必须有 loading 态、错误兜底、可中断恢复。

## 沉淀到 schema

```jsonc
project.meta.targetUser    = { summary: "<Primary 一句话> + <Secondary 一句话>" }
project.meta.coreScenarios = [
  { id: "S1", summary: "<高频场景描述>" },
  { id: "S2", summary: "<中频场景描述>" },
  { id: "S3", summary: "<低频场景描述>" }
]
// 一句话定位 + 核心价值 浓缩进 targetUser.summary 或 styleDirection.summary
```

完整字段映射见 `../schema-spec/project-meta.md`。
