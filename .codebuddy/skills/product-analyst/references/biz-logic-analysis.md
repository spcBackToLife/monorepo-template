# 业务逻辑深度分析模板

## 单模块分析框架

对每个功能模块，按以下 5 个维度系统分析：

### 维度 1: 数据规则

定义该模块涉及的所有数据字段的约束：

```markdown
| 字段 | 类型 | 约束 | 校验规则 | 错误提示 |
|------|------|------|---------|---------|
| phone | string | 必填、唯一 | /^1[3-9]\d{9}$/ | "请输入正确的手机号" |
| nickname | string | 必填、2-16字符 | 无特殊符号、不含敏感词 | "昵称不符合规范" |
```

### 维度 2: 状态机

该模块涉及的状态流转：

```
[初始态] → 触发条件 → [中间态] → 触发条件 → [终态]
                         ↓ 异常
                    [异常态] → 恢复 → [中间态]
```

示例（订单状态机）：
```
待支付 → 支付成功 → 待发货 → 已发货 → 已签收 → 已完成
  ↓        ↓          ↓        ↓
超时取消  支付失败    取消     拒收→退货
```

### 维度 3: 权限矩阵

```markdown
| 操作 | 游客 | 普通用户 | 内容作者 | 管理员 |
|------|:----:|:-------:|:-------:|:------:|
| 查看内容 | ✅ | ✅ | ✅ | ✅ |
| 发布内容 | ❌ | ✅ | ✅ | ✅ |
| 编辑内容 | ❌ | ❌ | ✅ | ✅ |
| 删除任何内容 | ❌ | ❌ | ❌ | ✅ |
```

### 维度 4: 并发 & 一致性

- 两个用户同时操作同一数据会怎样？
- 乐观锁 vs 悲观锁的选择
- 最终一致性可接受的场景

### 维度 5: 性能 & 限制

- 列表数据量预估（百/千/万/百万级）
- 需要分页？无限滚动？虚拟列表？
- 需要缓存？缓存失效策略？
- 需要限流？限流粒度（用户/IP/全局）？

## 常见模块的典型边界 Case

### 用户注册/登录

| 场景 | 边界 Case | 处理策略 |
|------|----------|---------|
| 手机号 | 已注册的手机号再次注册 | 提示"已注册" + 引导登录 |
| 手机号 | 用户换号后旧号被新用户注册 | 不影响（手机号只是登录凭证） |
| 验证码 | 同时发多条短信 | 只有最后一条有效 |
| 验证码 | 短信到达延迟(>5分钟) | 过期后提示重新获取 |
| 密码 | 连续错误超限 | 账号临时锁定15分钟 |
| 第三方登录 | 微信解绑后重新绑定 | 视为新关联（不恢复旧数据） |
| 多设备 | 同账号多设备同时在线 | 默认允许，可选"踢下线" |

### 内容发布

| 场景 | 边界 Case | 处理策略 |
|------|----------|---------|
| 字数 | 超过最大字数限制 | 实时计数 + 截断提示 |
| 图片 | 上传超大图片(>10MB) | 前端压缩 or 提示"图片过大" |
| 图片 | 图片格式不支持 | 明确列出支持格式 |
| 敏感内容 | 包含敏感词 | 发布时检测 → 机审 → 人审 |
| 网络 | 发布过程中断网 | 保存草稿 + 网络恢复后重试 |
| 并发 | 快速双击发布按钮 | 前端防重 + 后端幂等键 |
| 删除 | 有人正在评论/点赞时删除 | 允许删除，评论/赞随之消失 |

### 列表/Feed

| 场景 | 边界 Case | 处理策略 |
|------|----------|---------|
| 空数据 | 首次使用无内容 | 空状态引导（推荐关注/发布） |
| 大量数据 | 下拉加载更多 | 游标分页（不用 offset） |
| 实时性 | 浏览时有新内容发布 | "有N条新动态" 提示条 |
| 删除 | 正在浏览的内容被作者删除 | 刷新后消失 or 显示"已删除" |
| 排序 | 时间序 vs 推荐序 | 可切换 Tab，默认推荐 |

### 社交/关注

| 场景 | 边界 Case | 处理策略 |
|------|----------|---------|
| 互关 | A关注B，B关注A | 自动变为"互相关注" |
| 取关 | 取消关注 | 对方不收到通知 |
| 拉黑 | 拉黑后对方发的消息 | 不可见但不删除 |
| 关注上限 | 防止恶意关注 | 上限5000人 |
| 粉丝列表 | 百万级粉丝 | 只显示最近活跃 + 搜索 |

## 分析检查清单

每个模块分析完毕后，用此清单验证完整性：

- [ ] 所有用户角色都覆盖了（普通用户/管理员/游客）
- [ ] 主线流程清晰完整
- [ ] 所有异常分支都有处理策略
- [ ] 数据校验规则明确（前端+后端双校验）
- [ ] 状态流转图完整（含不可逆操作的确认）
- [ ] 权限矩阵明确
- [ ] 并发场景考虑
- [ ] 性能瓶颈识别
- [ ] 安全风险识别
- [ ] 与其他模块的依赖关系标注
