# 方法论 2（v3）：视觉概念 (Visual Concept) — mood / 灵魂句 / 风格关键词

> 适用任务：`D-X-concept`（briefing 之后、strategy 之前）
>
> **核心**：策略前必须先有概念。**视觉概念 = 一句话浓缩本屏要给用户的视觉感受**——这句话决定后续所有视觉决策。
>
> 没有视觉概念 → strategy 没有方向 → craft 任务自创无章法 → 输出简陋。

---

## 1. 视觉概念的三件套

每屏必出三件：

### 1.1 灵魂句（Soul Sentence）

一句话，10-25 字，浓缩本屏要让用户在 0.5 秒内感受到的"东西"。

例：
- 登录页（校园社交）：「**清新校园温度，不浮夸不冷漠**」
- 首页（健身 app）：「**专注于行动，激发力量感**」
- 支付页：「**可靠、清晰、零干扰，让用户安心点确认**」
- 引导页：「**明亮的开始，让用户对新功能有期待**」

**判定标准**：
- 一句话能让陌生设计师立刻明白"画面应该长什么样"
- 不是空话（如"漂亮"、"现代"、"高级感"）
- 含一个**情绪核心**（温度 / 力量 / 信任 / 期待 / 兴奋 / 平静）

### 1.2 风格关键词 3 个（Style Keywords）

3 个具体可视化的形容词，覆盖：
1. **色彩气质**（暖白 / 冷蓝 / 黑金 / 绿意 / 紫雾）
2. **形状气质**（直角理性 / 大圆角柔和 / 有机曲线 / 几何切割）
3. **装饰气质**（极简留白 / 轻装饰节制 / 丰富插画 / 几何科技）

例（登录页）：「**暖白 / 极简 / 单色温度**」
例（首页健身 app）：「**冷蓝 / 直角力量 / 几何科技**」
例（支付页）：「**冷白 / 大圆角 / 极简留白**」

**判定标准**：
- 每个词都对应到 token / 形状 / 装饰策略中的具体选择
- 3 个词组合互不冲突（如不能"暖色 + 冷峻 + 活泼"）
- 不抽象（避免"独特 / 创新 / 用户友好"等无具体视觉对应的词）

### 1.3 mood board 描述（Mood Board Words）

用 3-5 个**生活场景词**勾勒本屏视觉气氛，帮自己（和评审）形成画面感。

例（登录页）：「**晨光透过窗 / 笔记本上的便签 / 球场围栏的剪影 / 学校公告板**」
例（健身首页）：「**清晨的跑道 / 健身房的金属杠铃 / 心率波形 / 运动鞋纹理**」
例（支付页）：「**银行柜台的整洁感 / 收据纸张的克制 / 玻璃栏杆 / 微凸的按键**」

**判定标准**：
- 不要写品牌名（如"Apple 风"、"小红书风"）
- 不要写"美的 / 好看的"，要写**具体场景物**
- 写完读一遍：能不能想象出对应画面？

## 2. 候选 + 否决（必做）

写 concept.md 时必须给至少 **2-3 个候选风格方向**，然后在其中**选 1**。

例（登录页）：

| 候选 | 灵魂句 | 风格关键词 3 | mood board | 选/否决 |
|---|---|---|---|---|
| A 温暖治愈 | 暖光中的简洁，让用户被欢迎 | 暖白 / 极简 / 单色温度 | 晨光 / 便签 / 球场 / 公告板 | ✅ 选定 |
| B 冷静专业 | 严肃可靠的工具感，传递安全 | 冷白 / 直角 / 极简 | 银行柜台 / 玻璃 / 网格 / 收据 | ❌ 否决：与产品「校园社交」气质冲突，过于冷漠 |
| C 活泼俏皮 | 校园的活力色，让用户兴奋 | 多色 / 圆角 / 装饰丰富 | 学校画廊 / 涂鸦墙 / 气球 / 彩虹 | ❌ 否决：登录页是工具入口，活泼会让用户觉得不专业；活泼留给首页 / Feed |

**判定标准**：
- 候选 ≥ 2 个
- 每个候选都给完整三件套
- 否决理由具体（"与 X 冲突"、"产品语义不符"）
- 选定理由具体（"与 theme.intent='warm-minimal' 完全契合"、"与 product 目标用户'95-00 后大学生'的轻焦虑情绪相匹配"）

## 3. 视觉概念 → 后续 strategy 的传递

视觉概念决定 strategy 5 维的方向：

```
灵魂句 + 风格关键词 → 决定：
  
  色（color strategy）
    暖白：选 #FCFCFD 类暖底；强调色用蓝紫（中性 + 一点温度）
    冷白：选 #F8FAFC 类冷底；强调色用纯蓝
    黑金：黑底 + 金字（高端场景）
  
  字（typography）
    极简：字号梯度收紧（28→22→14→12），字重克制（500 而非 700）
    活泼：字号梯度大（36→24→14→12），字重对比强（700 vs 400）
    专业：等宽数字 / 衬线标题
  
  形（shape）
    圆角柔和：12px / 16px 大圆角
    直角理性：0px / 4px 微圆角
    有机曲线：自由曲线装饰 + 大圆角
  
  饰（decoration system）
    极简：1-2 处单色光斑 / 0 装饰
    丰富：插画 + 多处装饰节点
    几何：直线 / 网格 / 多边形
  
  律（rhythm）
    紧凑：4-8-16-24 间距梯度
    呼吸：8-16-24-32-48 间距梯度
    节奏强：对比比例大
```

详见 `methodology/03-color / 04-typography / 05-shape / 06-decoration / 07-rhythm`。

## 5. 红线

- ❌ 跳过 concept 直接 strategy → R-PHASE-FUNNEL-01
- ❌ 灵魂句空话（"漂亮"、"现代"、"用户友好"）→ R-CONCEPT-01
- ❌ 风格关键词只有 1-2 个 / 互相冲突 / 不可视化 → R-CONCEPT-02
- ❌ mood board 用品牌名（"小红书风"）→ R-CONCEPT-03
- ❌ 候选 < 2 个 → 没做选择就上路 → R-CONCEPT-04
- ❌ 否决理由空话（"不好"、"不合适"）→ R-CONCEPT-05
- ❌ 选定理由无产品/主题/交互的契合度论证 → R-CONCEPT-06
