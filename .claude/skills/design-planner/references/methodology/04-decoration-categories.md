# 方法论 4：装饰元素 7 大类决策框架

> 适用任务：`D-X-decorations`、`D-X-materials`（装饰素材时）
> 装饰不是"好看"——是**视觉手段**，每个装饰必须至少服务一个目的。

## 1. 装饰的终极目的（不是好看）

每个装饰都必须至少服务于以下 6 个目的之一：

```
1. 引导注意力    → 光晕指向 CTA 按钮方向
2. 建立空间感    → 前景清晰 + 背景模糊 = 深度
3. 传递品牌调性  → 看到装饰就知道是什么品牌
4. 填充空白      → 避免页面"空荡荡"
5. 营造情绪      → 温暖/冷静/兴奋/专注
6. 暗示可交互    → 按钮光效暗示"可以点"

无目的的装饰 = 视觉噪音 = 必须删除
```

## 2. 装饰元素 7 大类

### 类 1：几何类
| 子类型 | 形态 | 传递感受 | 适用风格 |
|--------|------|---------|---------|
| 直线/线段 | 平行线、交叉线、放射线 | 秩序、方向、速度 | 科技/商务/极简 |
| 网格/矩阵 | 规则网格、透视网格 | 系统、结构、技术感 | 科技/数据/后台 |
| 三角形 | 锐角、等边、组合 | 稳定(正)、动态(倒)、锐利 | 现代/潮流/体育 |
| 矩形/方块 | 像素块、色块组合 | 模块化、数字感 | 科技/游戏/开发者 |
| 多边形 | 六边形、八边形 | 复杂、精密、蜂窝协作 | 科技/企业/协作 |
| 菱形/钻石 | 旋转 45° 方块 | 精致、premium、宝石 | 奢华/金融/高端 |

### 类 7：裁剪/溢出装饰类（Clipped Shapes）★

**最重要的现代装饰手法**：图形只露出一部分，被容器边缘裁剪。

| 子类型 | 形态 | 传递感受 | 适用风格 |
|--------|------|---------|---------|
| 截断矩形/色块 | 矩形伸出画面边缘 | 空间延伸、现代、层次 | 企业/SaaS/现代 |
| 半圆/四分之一圆 | 圆形被裁剪，只露弧线 | 柔和、框架感、包围 | 社交/教育/金融 |
| 斜切色块 | 用斜线/弧线切割的大色块 | 动态、区分、引导 | 营销/落地页 |
| 溢出圆环 | 大圆环从角落伸出 | 氛围、深度、宏大 | 科技/暗色/仪表盘 |
| 弧线分区 | 用曲线替代直线分隔区域 | 流动、自然、不呆板 | 生活/社交/内容 |
| 不规则色块(Blob) | 自由形状的大面积渐变填充 | 有机、亲和、现代 | 科技/SaaS/社交 |
| 叠层卡片 | 多个矩形错位重叠 | 层次、丰富、信息感 | 产品/作品集 |

#### 为什么截断比完整更好

```
1. 完整 = 闭合 = 视觉终止 = 显得狭小
   截断 = 开放 = 暗示画面外延伸 = 空间感

2. 完整图形占据确定面积 = 跟内容"抢地盘"
   截断图形面积不确定 = 在背景层"若隐若现" = 不干扰内容

3. 完整图形有明确边界 = 生硬
   截断图形边界被裁掉 = 柔和融入页面 = 自然
```

## 3. 装饰风格 → 主题匹配表

```
品牌调性 → 主装饰类型 + 辅助装饰类型

科技/未来    → 主: 几何(网格/线段) + 光效(光晕/射线)    辅: 数学(曲线)
社交/温暖    → 主: 有机(光斑/气泡) + 符号(心形/星星)    辅: 光效(渐变)
商务/专业    → 主: 几何(直线/矩形) + 纹理(噪点)         辅: 光效(镜面)
创意/艺术    → 主: 有机(波浪/花瓣) + 纹理(半色调)       辅: 光效(极光)
游戏/活力    → 主: 符号(星星/闪烁) + 几何(碎片)         辅: 光效(霓虹)
极简/高端    → 主: 几何(细线) + 光效(微妙渐变)          辅: 纹理(噪点)
教育/成长    → 主: 符号(星星/箭头) + 有机(圆/光斑)      辅: 几何(点阵)
青春治愈     → 主: 有机(花瓣/水滴) + 光效(渐变光晕)     辅: 裁剪(blob)
```

## 4. 装饰用量决策树

```
Q1: 这个页面的核心内容是什么？
  ├── 内容本身很丰富（Feed/图片/视频）→ 装饰极少(0-2)
  └── 内容简单/空白较多（登录/空态/设置）→ 继续 Q2

Q2: 页面的情感需求是什么？
  ├── 纯功能（设置/列表）→ 几乎不要装饰(0-1)
  ├── 品牌表达（登录/注册/启动）→ 少量精致装饰(2-4)
  ├── 情绪引导（空态/错误/成就）→ 中等装饰(4-8)
  └── 庆典/特殊（会员/活动/节日）→ 可以密集(8-15)

Q3: 装饰的主次关系？
  ├── 1 个主视觉(占 60% 视觉重量) + N 个点缀(占 40%)
  └── 禁止：所有装饰同等重要 = 没有重点

Q4: 去掉任何一个装饰后，页面变差了吗？
  ├── 是 → 保留
  └── 否 → 删除（"如果不知道该不该加，就不加"）
```

## 5. 装饰用量配方

### 科技暗色风
```
主装饰: 渐变光晕(primary 15%, 右上角, 大面积模糊)
次装饰: 细几何线条(primary 8%, 对角方向, 2-3 条)
微装饰: 小光点(3-5 个, 随机散布, 2-4px, primary 10%)
配色: 仅使用 primary 的不同透明度
总数: 6-8 个元素
```

### 青春治愈风（粉色/暖白）
```
主装饰: 大圆形渐变(primaryLight 12%, 右上角溢出)
次装饰: 有机叶片(secondary 8%, 左下角, PNG 素材)
微装饰: 小光点(3-4 个, 散布, primary/secondary 6%)
配色: primary 系 + secondary 系
总数: 5-7 个元素
```

### 极简高端风
```
主装饰: 无（留白即是设计）
次装饰: 一条极细线(border-color 3%, 水平分隔)
微装饰: 微妙噪点纹理(全局 2% 覆盖)
配色: 仅黑白灰 + 极少量 primary 点缀
总数: 1-3 个元素
```

### 活力游戏风
```
主装饰: 不规则几何碎片(多色, 15%, 动态旋转)
次装饰: 星芒/闪烁(primary/secondary, 随机位置)
微装饰: 小十字/圆点阵列(4-6px, 10%, 重复图案)
配色: 多色(primary + secondary + accent)
总数: 10-15 个元素
```

## 6. 组合规则

### 6.1 同类不超过 2 种变体
```
❌ 圆形光斑 + 椭圆光斑 + 环形 + 模糊圆 (4 种)
✅ 圆形光斑(大) + 环形(小) (2 种, 主次分明)
```

### 6.2 跨类最多 3 种
```
❌ 几何线条 + 光斑 + 星星 + 波浪 + 噪点 (5 种 = 视觉灾难)
✅ 几何线条 + 渐变光晕 + 微妙星星点缀 (3 种 = 丰富不杂乱)
```

### 6.3 大小/透明度必须有层次
```
1 个主装饰: 大面积 + 15-25% 透明度（定调性）
2-3 个次装饰: 中等面积 + 5-15% 透明度（丰富感）
N 个微装饰: 小面积 + 3-8% 透明度（精致感）

❌ 所有装饰一样大一样亮 = 噪音
✅ 主次分明 = 节奏 + 呼吸
```

### 6.4 颜色控制
```
同色系(最安全): 全部使用 primary 不同透明度（5%/10%/20%）
双色系(丰富): primary + secondary 交替
多色系(慎用): 仅用于庆典/游戏/创意场景

❌ 装饰颜色比内容鲜艳 = 喧宾夺主
✅ 装饰颜色永远比内容淡（低饱和/低透明度）
```

## 7. 装饰位置策略

```
装饰只能出现在 L4 氛围层（z-index = 0/1）：

┌─────────────────────────────┐
│  L1 前景交互层 (z=20+)       │ ← 绝对不放装饰（干扰操作）
│  ─────────────────────────  │
│  L2 内容层 (z=10)            │ ← 不放装饰（干扰阅读）
│  ─────────────────────────  │
│  L3 结构层 (z=5)             │ ← 极克制（分割线可带微渐变）
│  ─────────────────────────  │
│  L4 氛围层 (z=0)             │ ← 装饰在这里
└─────────────────────────────┘

具体位置：
  角落: 光晕/几何图案放四角（不占中心注意力）
  边缘: 波浪/曲线沿顶部或底部（空间分隔+氛围）
  背景: 大面积低对比度渐变/纹理（整体调性）
  点缀: 小符号(星/点/十字)散布空白（活力感）
```

## 8. 装饰节点 4 类（落库）

详见 `schema-spec/decoration-nodes.md`。装饰节点必须：
- `position: absolute` + 低 z-index（0/1）
- `pointerEvents: none`（不拦截内容操作）
- `meta.design.visualSpec.role = "氛围-装饰"`
- `meta.design.materialSpec` 必填（即使是纯 CSS 也写 `renderHint: "css-gradient"`）

| 类型 | 例子 | renderHint |
|------|------|-----------|
| 背景氛围 | PinkCircleDeco / GradientGlow / MintLeafSplash | css-gradient 或 png |
| 角落溢出 | TopLeftBlob / CornerStripe | css 或 png |
| 分割装饰 | DividerLine / OrnamentSeparator | css 或 svg |
| 品牌点缀 | BrandPattern / Watermark | png |

## 10. 红线

- ❌ 装饰类型 > 3 类 → 视觉灾难
- ❌ 同类装饰 > 2 种变体
- ❌ 所有装饰透明度一致 → 没有层次
- ❌ 装饰颜色比内容鲜艳 → 喧宾夺主
- ❌ 装饰放到 L1/L2 内容层 → 干扰操作/阅读
- ❌ 装饰节点不写 `pointerEvents: none` → 误拦截点击
- ❌ 装饰节点不放 `position: absolute` → 占据布局
- ❌ "全 CSS 不画素材"——C 端每屏至少图标/装饰之一，特殊场景需逐条论证（如纯文本输入页才允许）
