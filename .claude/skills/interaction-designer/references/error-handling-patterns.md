# 错误处理最佳实践

## 错误处理的核心原则

1. **用户不关心技术原因** — 不要显示 "Error 500" / "ECONNREFUSED"
2. **告诉用户能做什么** — 不只说"出错了"，还要说"你可以..."
3. **区分可恢复 vs 不可恢复** — 可恢复给重试按钮，不可恢复给替代方案
4. **错误信息紧贴上下文** — 哪个字段错就在哪个字段旁边显示

---

## 错误分类矩阵

| 类型 | 来源 | 用户感知 | UI 模式 | 恢复方式 |
|------|------|---------|---------|---------|
| 表单校验 | 前端 | "我填错了" | Inline 红字 | 修改输入 |
| 业务拒绝 | 后端 200+error code | "不允许这样做" | Inline/Toast | 按提示修改 |
| 认证过期 | 后端 401 | "需要重新登录" | Modal | 重新登录 |
| 权限不足 | 后端 403 | "没有权限" | Toast + 引导 | 联系管理员 |
| 资源不存在 | 后端 404 | "找不到了" | 全屏空态 | 返回上一页 |
| 频率限制 | 后端 429 | "操作太频繁" | Toast + 禁用 | 等待冷却 |
| 网络异常 | 客户端 | "没有网络" | Toast + 重试 | 检查网络 |
| 超时 | 客户端 | "太慢了" | Toast + 重试 | 重试 |
| 服务错误 | 后端 500 | "服务出问题了" | Toast | 稍后重试 |
| 未知错误 | 任何 | "出了点问题" | Toast + 上报 | 刷新/联系客服 |

## 各类错误的 UI 处理方案

### 表单校验错误

```
位置: 错误字段正下方（不是弹窗/不是Toast）
样式: 红色小字(12px) + 字段 border 变红 + 左侧小 ⚠️ 图标
时机: onBlur 或 onSubmit 触发
消失: 用户修改该字段后立即消失
多字段错误: 滚动到第一个错误字段 + 聚焦
```

### 业务逻辑错误（如"手机号已注册"）

```
位置: 相关字段下方（和校验错误一样的位置）
样式: 同上
特殊: 可能附带操作按钮（"去登录" / "找回密码"）
```

### 网络错误

```
首次加载失败:
  - 全屏错误占位
  - 插画 + "网络异常" + [重试] 按钮
  - 重试时显示 loading，不清空错误 UI（防止闪烁）

操作中断网:
  - Toast "网络异常，请检查网络后重试"
  - 操作回滚到之前状态
  - 按钮恢复可点

长期离线:
  - 顶部固定黄色条 "当前无网络连接"
  - 如有本地缓存 → 正常展示 + "数据可能不是最新"
  - 无缓存 → 全屏离线提示
```

### 超时处理

```
阈值: 
  - 普通请求: 15s
  - 文件上传: 60s
  - 长轮询: 30s

超时后:
  - Toast "请求超时，请重试"
  - 按钮/UI 恢复可操作状态
  - 不自动重试（防止雪崩）
```

## 错误文案规范

### 原则

```
❌ 技术语言: "Error: Network request failed (ETIMEDOUT)"
❌ 无意义:   "发生错误"
✅ 人话:     "网络不太好，请检查网络后重试"

❌ 只说问题: "密码错误"
✅ 给方案:   "密码错误，忘记密码？"
```

### 常见错误文案模板

| 场景 | 文案 |
|------|------|
| 密码错误 | 密码不正确，请重新输入 |
| 账号不存在 | 该手机号未注册，去注册？ |
| 手机号已注册 | 该手机号已注册，去登录？ |
| 验证码错误 | 验证码不正确，请重新输入 |
| 验证码过期 | 验证码已过期，请重新获取 |
| 网络错误 | 网络异常，请检查网络连接 |
| 服务错误 | 服务暂时不可用，请稍后再试 |
| 请求超时 | 请求超时，请稍后重试 |
| 频率限制 | 操作太频繁，请 X 秒后再试 |
| 权限不足 | 暂无权限，请联系管理员 |
| 文件过大 | 文件大小超过限制（最大 Xmb） |
| 格式不支持 | 不支持该文件格式，请上传 jpg/png |

## 错误恢复策略

### 自动重试

```
适用: 网络抖动导致的偶发失败
策略: 
  - 最多重试 3 次
  - 间隔: 1s → 2s → 4s（指数退避）
  - 静默重试（不提示用户）
  - 3次全部失败后才显示错误 UI
不适用: 
  - 业务错误（重试也不会成功）
  - 用户主动操作触发的请求（应该让用户决定是否重试）
```

### 降级方案

```
图片加载失败 → 显示占位图（灰色+图标）
头像加载失败 → 显示首字母圆形
列表加载失败 → 显示缓存数据 + "数据可能不是最新"
功能不可用   → 隐藏入口或 disabled + tooltip
```
