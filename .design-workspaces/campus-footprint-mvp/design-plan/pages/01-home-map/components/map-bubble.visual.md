# MapBubble — 视觉设计分析

> 层级: L2 组件深钻 | 上级: pages/01-home-map/visual.md | 角色: 主角-内容

---

## 1. 情感与氛围目标

### 1.1 情感定位

| 维度 | 回答 |
|------|------|
| 用户此刻的心理状态 | 在暗色地图上扫视，寻找「有趣的东西」 |
| 目标感受 | **发现、活跃、可触达** — 像夜空中可以摘到的星星 |
| 情绪曲线 | 注意(看到发光) → 好奇(它在动) → 行动(点击探索) |
| 与主题风格的关系 | 是「午夜发光」的核心视觉承载——暗底上的荧光物 |
| 页面给我的视觉预算 | 角色:主角-内容, 权重:7/10, 允许:纯色/渐变+glow+浮动, 装饰上限:中(多个共存不能太花) |

### 1.2 品牌感要素

- 圆形+发光+浮动 = 品牌独有的「位置标记」视觉语言
- 同屏多个气泡形成「星群」效果，是App最核心的品牌截图画面

## 2. 视觉层级设计

### 2.1 空间深度(组件内)

```
[装饰层] ─── 外发光(box-shadow, 最弱存在感)
    ↑
[容器层] ─── 圆形底色(primary/gold, 主体)
    ↑
[内容层] ─── 中心内容(数字/头像缩略图, 最强清晰度)
```

### 2.2 视觉权重(组件内)

| 元素 | 权重(1-10) | 手段 | 为什么 |
|------|:-:|------|--------|
| 圆形底色 | 8 | 高饱和纯色 | 在暗底上是最醒目的存在 |
| 外发光 | 4 | 同色glow模糊扩散 | 增加"发光感"但不抢底色 |
| 中心内容 | 6 | 白色/高对比 | 信息需可读但不比气泡本身更抢眼 |

## 3. 视觉手段清单

### 3.1 色彩运用

| 色彩手段 | 描述 | 面积 | 感受 | Token |
|---------|------|------|------|-------|
| 蓝色底(公开动态) | Primary实心填充 | 整个圆形 | 标准/可发现 | primary |
| 金色底(定向动态) | Gold实心填充 | 整个圆形 | 特殊/珍贵 | gold |
| 灰色底(已读) | text-tertiary低透明度 | 整个圆形 | 已探索/弱化 | text-tertiary at 40% |
| 白色内容 | 纯白文字/图标 | 中心小区域 | 高对比可读 | text-primary |

### 3.2 光影效果

| 光影 | 描述 | 对象 | 感受 | 参数 |
|------|------|------|------|------|
| 蓝glow | 同色柔和扩散 | 蓝色气泡 | "地面光点" | 0 0 8px rgba(79,140,255,0.3) |
| 金glow | 金色扩散(更强) | 金色气泡 | "宝物光芒" | 0 0 12px rgba(255,184,48,0.4) |
| 无glow | — | 已读气泡 | 光芒消退 | none |

### 3.3 动效

| 动效 | 触发 | 效果 | 感受 | 参数 |
|------|------|------|------|------|
| 浮动 | idle(未读) | translateY(±2px) | "活着的" | 2.5s ease-in-out infinite |
| 脉冲 | idle(定向) | scale(1→1.1→1) | "注意我！" | 2s ease-in-out infinite |
| 入场 | 首次出现 | scale(0→1.05→1)+opacity | "从地面冒出" | 400ms spring |
| 点击聚焦 | tap | scale(1→1.2) | "你选中了我" | 200ms ease-out |
| 静止 | idle(已读) | 无动效 | 已不重要 | — |

## 4. 实现分类

| # | 元素 | 分类 | 理由 |
|---|------|:-:|------|
| 1 | 圆形底色 | CSS | border-radius:50%+background |
| 2 | 外发光 | CSS | box-shadow |
| 3 | 浮动/脉冲动效 | CSS | animation keyframes |
| 4 | 入场弹跳 | CSS | transition/animation |
| 5 | 中心文字 | CSS | 纯布局 |

**结论: 本组件无Canvas素材需求，全部CSS可精美实现。**

论证: MapBubble是简单几何形状(圆形)，视觉丰富度完全由色彩+光影+动效达成，不需要复杂形状/多色图标/不规则图案。CSS的border-radius:50%+box-shadow+animation组合已经是最精美的实现方式。

---

## 5. 素材需求清单

无。本组件全部视觉效果由CSS实现。

## 7. 风格一致性检查

| 检查项 | 回答 |
|--------|------|
| 色彩来自Token？ | ✅ primary/gold/text-tertiary |
| 光效与页面一致？ | ✅ glow参数与页面FAB同体系 |
| 动效时长标准？ | ✅ 2.5s/2s/400ms/200ms 全部引用标准 |
| 视觉权重在预算内？ | ✅ 权重7，不超页面分配 |
| 多个共存不冲突？ | ✅ 动效stagger错开+已读静止，不会视觉爆炸 |
