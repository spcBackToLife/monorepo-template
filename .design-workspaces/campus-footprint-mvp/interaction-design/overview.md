# 全局交互规范

---

## 一、反馈层级体系

本产品的所有 UI 反馈统一遵循 6 级层级：

| 层级 | 名称 | 适用场景 | UI 形式 | 时长 |
|:----:|------|---------|---------|------|
| L0 | 即时触感 | 所有可点击元素 | scale(0.96) + 轻震 | 100ms |
| L1 | 微反馈 | 点赞/收藏/切Tab | 图标动画 + 颜色变化 | 200-400ms |
| L2 | 轻提示 | 操作成功/失败信息 | Toast（底部上浮） | 2.5s 自消失 |
| L3 | 中反馈 | 网络请求/提交 | Button loading + 结果 | 跟随异步 |
| L4 | 强确认 | 删除动态/举报/付费 | Modal + 双按钮 | 用户关闭 |
| L5 | 全屏状态 | 空数据/断网/严重错误 | 全屏占位 + CTA按钮 | 持续到恢复 |

### 本产品特殊反馈

| 场景 | 反馈形式 | 说明 |
|------|---------|------|
| 撒网捞人 | 全屏动效（2-3秒） | 网甩出+水花+卡片浮出 |
| 炸弹爆炸 | 震屏+粒子+卡片雨 | 付费道具需要强烈反馈 |
| 定向动态触发 | 系统推送+App内金色横幅 | 核心惊喜时刻 |
| 动态发布成功 | 粒子飘落到地图锚点 | 强化「留在这里」感知 |
| 精准网渐进 | 高斯模糊递减（3档） | 期待感递增 |

## 二、加载策略总则

### 2.1 按场景选择加载模式

| 场景 | 策略 | 具体表现 |
|------|------|---------|
| 首次进入页面 | 骨架屏 | 模拟布局的灰色占位块 |
| 地图加载动态 | 渐显 | 气泡从透明→实体，由近到远依次出现 |
| 捞人结果 | 定制动效 | 不用通用loading，用水面动效替代 |
| 列表下拉刷新 | 顶部指示器 | 下拉→松手→旋转→完成/失败 |
| 列表加载更多 | 底部spinner | 到底→自动加载→追加 |
| 图片加载 | 渐进式 | 模糊缩略图→清晰大图 (progressive JPEG) |
| 付费操作 | 按钮内spinner | 不跳转，原地等待支付结果 |

### 2.2 加载时间分级

| 时长 | 用户感知 | UI 策略 |
|------|---------|---------|
| <300ms | 瞬时 | 不需要 loading 态 |
| 300ms-2s | 短暂等待 | 按钮 spinner / 骨架屏 |
| 2s-5s | 明显等待 | 骨架屏 + 进度暗示 |
| >5s | 太长了 | 进度条 + 文案（「正在搜索附近的人...」） |
| >15s | 超时 | 自动取消 + 提示重试 |

### 2.3 失败恢复策略

| 失败类型 | 首次加载 | 刷新时 | 加载更多 |
|---------|---------|--------|---------|
| 断网 | 全屏「无网络」+ 重试 | Toast + 保留旧数据 | 底部提示 + 重试 |
| 超时 | 全屏「加载超时」+ 重试 | Toast | 底部提示 |
| 服务器错误 | 全屏「服务异常」 | Toast「服务繁忙」 | 底部提示 |
| 空数据 | 空状态插画 + CTA | — | 「没有更多了」 |

## 三、错误处理总则

### 3.1 错误类型 → UI 模式映射

| 错误类型 | 触发原因 | UI 模式 | 可恢复 |
|---------|---------|---------|:------:|
| 表单校验 | 输入格式/规则不符 | Inline红字 + 字段红框 | ✅ |
| 业务错误 | 手机号已注册/密码错误/余额不足 | Inline红字 或 Toast | ✅ |
| 权限错误 | 位置未授权/未登录 | 引导弹窗（去设置/去登录） | ✅ |
| 网络错误 | 断网/DNS/超时 | Toast + 重试 | ✅ |
| 服务错误 | 500/503 | Toast「服务繁忙，稍后再试」 | ⏳ |
| 位置错误 | GPS精度不足/不在校园内 | 专属提示弹窗 + 说明 | ✅ |

### 3.2 表单校验时机

```
onChange（即时）：
├── 手机号位数（输入第11位后即时校验格式）
├── 验证码位数（6位后自动提交）
└── 文字长度实时计数

onBlur（失焦）：
├── 密码强度
├── 昵称长度/字符规则
└── 签名长度

onSubmit（提交时）：
├── 所有必填项非空检查
├── 两次密码一致性
└── 异步唯一性（手机号是否已注册）
```

### 3.3 错误消失时机

- Inline错误：用户修改对应字段后立即消失
- Toast：2.5秒自动消失 或 用户上滑关闭
- Modal错误：用户点击关闭/确认后消失

## 四、转场动画规则

### 4.1 页面级转场

| 导航类型 | 动画 | 时长 | 缓动 |
|---------|------|------|------|
| Push（进入子页） | 新页右滑入，当前页左移20%+暗化 | 300ms | ease-out |
| Pop（返回） | 当前页右滑出，下层页右移恢复+亮化 | 250ms | ease-in |
| Modal（弹出） | 底部滑入 + 背景暗化到0.5 | 300ms | spring(1, 0.9, 0) |
| Modal关闭 | 底部滑出 + 背景恢复 | 250ms | ease-in |
| Tab切换 | 交叉淡入淡出（无方向） | 200ms | ease |

### 4.2 本产品特殊转场

| 场景 | 转场效果 | 说明 |
|------|---------|------|
| 点击地图气泡→动态详情 | 气泡展开为卡片（shared element） | 空间连续性 |
| 捞人卡片→对方资料 | 卡片翻转 | 揭示感 |
| 发布成功→回到地图 | 缩小+飘落到对应位置 | 强化「锚定」感 |
| 定向动态推送→动态详情 | 信封拆开动效 | 仪式感 |

## 五、手势规范

| 手势 | 场景 | 效果 |
|------|------|------|
| 下拉 | 列表/地图 | 刷新 |
| 上滑 | 列表 | 加载更多 |
| 左滑卡片 | 捞人卡片 | 跳过/下一个 |
| 右滑卡片 | 捞人卡片 | 感兴趣 |
| 双指缩放 | 地图 | 放大/缩小 |
| 长按 | 动态/消息 | 弹出操作菜单（删除/举报/复制） |
| 右滑返回 | 全局 | 返回上一页（iOS原生） |

## 六、震动反馈（Haptics）

| 场景 | 震动类型 | 说明 |
|------|---------|------|
| 按钮点击 | Light impact | 轻触感 |
| 撒网甩出 | Medium impact | 甩出力度 |
| 捞到人/卡片浮出 | Success notification | 正向反馈 |
| 炸弹爆炸 | Heavy impact × 2 | 强烈冲击 |
| 操作失败 | Error notification | 负向提示 |
| 到达动态位置 | Success notification | 解锁成就感 |

---

## 七、无障碍基础

| 要求 | 实现 |
|------|------|
| 最小触摸区域 | 44×44pt |
| 对比度 | 文字≥4.5:1，大字≥3:1 |
| 动画尊重系统设置 | `prefers-reduced-motion` 时关闭装饰性动效 |
| 语音朗读 | 所有操作按钮有 accessibility label |

---

> 本文档为全局交互规范基线，所有页面交互设计必须遵循此规范。
> 特殊场景如需突破层级，需在对应页面文档中标注理由。
