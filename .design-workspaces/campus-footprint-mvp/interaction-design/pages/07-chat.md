# 聊天页 — 交互规格

---

## 状态机

```
States:
  loading       → 加载历史消息
  chatting      → 正常对话中
  sending       → 消息发送中
  empty         → 无历史消息（新对话）
  blocked       → 已被对方屏蔽/自己屏蔽对方
  offline       → 网络断开

Transitions:
  loading → chatting:     历史消息加载完毕
  loading → empty:        无历史记录
  chatting → sending:     发送消息
  sending → chatting:     发送成功（消息追加）
  sending → chatting:     发送失败（消息标红）
  chatting → offline:     网络断开
  offline → chatting:     网络恢复

Effects:
  → loading:    消息区骨架屏
  → chatting:   消息列表展示，新消息从底部滑入
  → sending:    消息立即显示（乐观UI）+ 右侧转圈
  → empty:      中间显示「你们还没有说过话，打个招呼吧」
  → blocked:    底部输入框消失 + 显示「对话已结束」
```

## 核心交互

### 发送消息
- 输入非空后「发送」按钮激活（灰→蓝）
- 点击发送：消息立即出现在右侧（乐观更新）+ 右下角小spinner
- 发送成功：spinner变✓✓（已送达）
- 发送失败：消息气泡左侧出现红色❗+ 点击重发

### 接收消息
- 新消息从底部弹入 + 轻震动
- 如果不在底部（用户在看历史）→ 底部浮窗「有新消息 ↓」
- 点击浮窗→滚动到底部

### 图片消息
- 点击「+」展开更多操作：[相册] [拍照] [位置卡片]
- 选择图片后直接发送（带上传进度条）
- 收到图片消息：缩略图+点击查看大图

### 消息状态
- ✓ 已发送（服务器收到）
- ✓✓ 已送达（对方设备收到）
- 蓝色✓✓ 已读

### 首次对话
- 从捞人打招呼进入时，顶部横幅：
  「小明通过「捞人」向你打招呼」
- 显示打招呼原始消息作为第一条

## 边界情况

| 情况 | 处理 |
|------|------|
| 消息发送中退出 | 后台继续发送，回来后显示结果 |
| 对方正在输入 | 顶部显示「对方正在输入...」（3秒超时消失） |
| 历史消息过多 | 初始加载最近20条 + 上拉加载更早的 |
| 发送图片过大 | 自动压缩 + 进度条 |
| 对方屏蔽我 | 发送后显示❗+ 「消息未送达」（不暴露被屏蔽） |
