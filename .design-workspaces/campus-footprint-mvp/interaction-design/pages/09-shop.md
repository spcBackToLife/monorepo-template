# 道具商店 — 交互规格

---

## 状态机

```
States:
  loading       → 加载商品列表
  browsing      → 浏览商品
  confirming    → 确认购买弹窗
  paying        → 支付中
  success       → 购买成功
  failed        → 购买/支付失败

Transitions:
  loading → browsing:     商品列表加载完毕
  browsing → confirming:  点击购买按钮
  confirming → paying:    确认支付
  confirming → browsing:  取消
  paying → success:       支付成功回调
  paying → failed:        支付失败/取消
  success → browsing:     关闭成功提示
  failed → browsing:      关闭失败提示
```

## 核心交互

### 购买确认弹窗

```
┌─────────────────────────────────┐
│                                  │
│       确认购买                   │
│                                  │
│    社交炸弹 💣 × 1               │
│    ¥5.00                         │
│                                  │
│    [微信支付]  [支付宝]          │
│                                  │
│    [确认支付 ¥5.00]              │
│    取消                          │
│                                  │
└─────────────────────────────────┘
```

- 选择支付方式 → 点击确认 → 拉起支付
- 支付成功：弹窗变为✓ + 「购买成功！」+ 道具飞入动效
- 支付失败/取消：Toast「支付未完成」+ 弹窗关闭

### 道具卡片
- 显示：图标 + 名称 + 单价 + 已拥有数量
- 有库存时：「购买」按钮 + 右上角数量角标
- hover/press：卡片微抬+阴影

### 首充引导
- 首次进入商店：顶部横幅「首充任意金额送炸弹+3张招呼卡！」
- 横幅有倒计时/一次性样式

## 边界情况

| 情况 | 处理 |
|------|------|
| 支付超时 | 15秒后提示「支付超时，请重试」 |
| 连续购买 | 允许，每次独立确认 |
| 余额不足(支付方) | 支付渠道自行提示 |
| 商品下架/价格变更 | 刷新后更新，购买中价格以确认时为准 |
