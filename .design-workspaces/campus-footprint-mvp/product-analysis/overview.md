# 校园足迹社交 — 产品需求概览（Phase 1）

---

## 一、产品定位

### 一句话定位

**一款基于地理位置的校园社交产品，让动态「长」在物理世界的坐标上，用空间承载记忆、用趣味玩法促进破冰。**

### 核心价值主张

| 维度 | 价值 |
|------|------|
| 差异化 | 动态锚定物理位置，不是「刷 Feed」而是「走到那里才能看到」 |
| 情感 | 回忆有了空间坐标，多年后走回原地可以「触景生情」 |
| 社交破冰 | 游戏化的「捞人」机制降低搭讪心理门槛 |
| 延伸性 | 位置锚定打通跨校园、跨城市甚至全球社交 |

### 与现有方案对比

| 产品 | 本产品差异点 |
|------|-------------|
| 朋友圈/QQ空间 | 它们是「人」维度的 Feed；我们是「地点」维度，到了才能看 |
| Soul/Uki | 匿名随机匹配；我们是基于位置+游戏化的熟半熟社交 |
| 探探 | 纯看脸匹配；我们有位置玩法和内容沉淀 |
| 树洞/校园墙 | 纯文字匿名；我们有位置绑定+富媒体+游戏化 |

## 二、目标用户 & 核心场景

### 2.1 用户画像

**Primary User — 在校大学生（18-24岁）**
- 社交需求旺盛，渴望认识新人但不好意思直接搭讪
- 热衷新奇玩法，愿意为有趣体验付费（小额）
- 每天在校园内固定路线活动（宿舍-教学楼-食堂-图书馆）
- 对回忆和仪式感有天然需求（毕业季/纪念日）

**Secondary User — 校友/毕业生**
- 返校时想看看当年的「数字足迹」
- 对母校有情感连接，愿意远程浏览校园动态

### 2.2 核心使用场景

| 场景 | 描述 | 情感诉求 |
|------|------|---------|
| 🎯 暧昧告白 | 在TA必经之路留一条只有TA能看到的定时消息 | 浪漫、心动、期待 |
| 🎣 宿舍楼社交 | 晚上无聊，在宿舍楼附近「捞一捞」看看谁在线 | 好奇、刺激、社交渴望 |
| 📍 日常打卡 | 在图书馆/食堂/操场发动态，标记自己的校园生活 | 记录、分享、存在感 |
| 🌍 跨校探索 | 浏览其他大学的位置动态，了解不同校园文化 | 好奇、探索、社交拓展 |
| 💭 毕业回忆 | 多年后回到校园，看到当年自己发的动态 | 怀旧、感动、仪式感 |

## 三、功能模块全景

```
校园足迹社交
├── 📱 用户体系
│   ├── 注册/登录（手机号+学校认证）
│   ├── 个人资料（头像/昵称/学校/年级/兴趣标签）
│   ├── 隐私设置（谁能看我/谁能捞我）
│   └── 账号安全
│
├── 📍 位置动态体系（核心亮点 1）
│   ├── 发布动态（视频/图文 + 锚定当前位置）
│   ├── 可见性规则
│   │   ├── 公开模式：任何人走到该位置可见
│   │   ├── 定向模式：指定某人到达该位置时可见
│   │   └── 定时模式：指定时间+位置条件触发
│   ├── 浏览动态（到达位置后解锁当前坐标的内容）
│   ├── 互动（点赞/评论/转发/回复动态）
│   └── 动态生命周期（永久/自动过期/手动删除）
│
├── 🎣 捞人社交体系（核心亮点 2）
│   ├── 捞人机制
│   │   ├── 基础「撒网」：随机匹配附近N人的卡片
│   │   ├── 「炸弹」玩法：一次炸出一堆人
│   │   ├── 「精准网」：多次尝试才捞到特定类型
│   │   └── 「金网」：付费高级网，匹配质量更高
│   ├── 匹配展示（头像/昵称/一句话/兴趣标签/距离）
│   ├── 破冰互动（打招呼模板/小游戏/共同话题推荐）
│   └── 付费体系（免费次数+付费道具）
│
├── 🌍 跨校探索体系（核心亮点 3）
│   ├── 校园地图浏览（选择目标学校看动态）
│   ├── 校园广场（每个学校的公开 Feed）
│   ├── 跨校交友（关注/私信其他学校的人）
│   └── 校际互动（比赛组队/话题PK/联合活动）
│
├── 💬 消息体系
│   ├── 私信（一对一聊天）
│   ├── 系统通知（动态被互动/被捞到/定时触发提醒）
│   ├── 位置触发通知（到达某位置时的推送）
│   └── 好友/关注列表
│
├── 💰 商业化体系
│   ├── 捞人道具（网/炸弹/加速器）
│   ├── 动态增强（置顶/扩大可见范围/延长存活时间）
│   ├── 个人装饰（头像框/昵称特效/专属标签）
│   └── 会员体系（每日免费次数提升/专属玩法）
│
├── 📊 内容安全 & 审核
│   ├── 内容审核（AI+人工）
│   ├── 举报/屏蔽
│   ├── 用户信用体系
│   └── 位置动态的合规管理
│
├── 🔔 通知 & 推送
│   ├── 位置触发推送（到达目标位置时）
│   ├── 社交互动推送
│   ├── 运营活动推送
│   └── 推送频率控制
│
└── ⚙️ 设置 & 其他
    ├── 位置权限管理
    ├── 隐私偏好
    ├── 通知偏好
    └── 帮助/反馈
```

## 四、MVP 范围建议

### 核心判断逻辑

这个产品的**灵魂**是「位置锚定的动态 + 游戏化社交破冰」。MVP 必须验证两个核心假设：
1. **用户愿不愿意专门走到某个地方看内容？**（位置动态是否有吸引力）
2. **游戏化的捞人机制能否有效促进陌生人破冰？**（付费意愿和社交转化率）

### 优先级划分

| 优先级 | 模块 | MVP 范围 | 理由 |
|:------:|------|---------|------|
| **P0** | 用户体系 | 手机号注册+学校选择+基础资料 | 产品基础设施 |
| **P0** | 位置动态 | 发布(图文)+公开可见+浏览+基础互动 | 核心假设验证1 |
| **P0** | 捞人社交 | 基础撒网+匹配展示+打招呼 | 核心假设验证2 |
| **P1** | 定向/定时动态 | 指定某人可见+时间触发 | 差异化玩法，高情感价值 |
| **P1** | 消息体系 | 私信+系统通知 | 社交闭环必备 |
| **P1** | 付费体系 | 捞人道具(1-2种)+付费购买 | 验证付费意愿 |
| **P2** | 跨校探索 | 校园地图+公开Feed浏览 | 扩展性验证 |
| **P2** | 视频动态 | 短视频发布+播放 | 内容丰富度 |
| **P2** | 炸弹/高级玩法 | 多种捞人方式 | 玩法深度 |
| **P3** | 校际互动 | 比赛组队/话题PK | 需要足够用户基数 |
| **P3** | 会员体系 | 完整商业化 | 规模化后 |
| **P3** | 内容安全 | 完善的审核+信用体系 | 运营规模后 |

### MVP V1.0 功能清单

```
✅ V1.0 必须有：
├── 手机号注册 + 学校认证（edu邮箱或学生证）
├── 个人资料编辑（头像/昵称/年级/兴趣标签）
├── 发布位置动态（图文，锚定当前GPS坐标）
├── 浏览位置动态（进入一定范围后解锁可见）
├── 动态互动（点赞/评论）
├── 基础捞人（每日3次免费撒网，随机匹配附近用户）
├── 匹配卡片展示（头像/昵称/一句话签名/距离）
├── 打招呼（预设模板消息）
├── 简单私信
└── 基础通知（被点赞/被评论/被捞到）
```

## 五、关键产品决策点（需确认）

### 5.1 位置精度问题
- **方案A**：精确到10米（更真实，但隐私风险大）
- **方案B**：精确到50米范围（模糊化，平衡体验与隐私）
- **方案C**：锚定到「地标」（如「图书馆」「食堂」而非精确坐标）
- **建议**：方案C为主+B补充。校园场景用地标更自然。

### 5.2 内容可见距离
- 用户需要走到多近才能看到位置动态？
- **建议**：默认50米半径，发布者可选扩大到200米

### 5.3 捞人的付费力度
- 免费次数太多→没有付费动力；太少→用户流失
- **建议**：每日3次免费，额外次数1元/次，道具3-10元

### 5.4 学校认证强度
- **方案A**：必须 edu 邮箱验证（强认证，保证纯净度）
- **方案B**：学生证照片审核（覆盖面广，但审核成本高）
- **方案C**：选择学校即可，不强制验证（门槛低但难保真实性）
- **建议**：MVP 阶段用方案C快速验证，后续升级到A

### 5.5 动态是否永久保留
- **方案A**：永久（契合「回忆」定位）
- **方案B**：默认7天过期，可花钱永久化
- **建议**：方案A，永久保留是核心情感卖点

## 六、竞品速览 & 差异化矩阵

| 竞品 | 核心模式 | 我们的差异 |
|------|---------|-----------|
| **微信-附近的人** | 基于位置看人 | 我们是看「内容」+「游戏化互动」，不是直接看人 |
| **Soul** | 灵魂匹配+匿名社交 | 我们有物理空间锚定，不是纯线上 |
| **校园集市/超话** | 校园信息聚合 | 我们是位置维度而非话题维度 |
| **Zenly/Snap Map** | 位置分享给好友 | 我们是位置承载内容，不只是分享实时位置 |
| **Pokémon GO** | AR+位置游戏 | 我们是社交+内容，但「走到才能解锁」的机制类似 |

**我们的独特组合**：位置锚定内容 × 游戏化破冰 × 校园熟人/半熟人 × 回忆沉淀

## 七、技术挑战预判

| 挑战 | 说明 | 建议方案 |
|------|------|---------|
| GPS 精度 | 室内定位精度差（宿舍楼层） | 结合 WiFi 指纹+蓝牙信标辅助 |
| 电量消耗 | 持续定位耗电 | 地理围栏+低功耗模式，非实时扫描 |
| 位置作弊 | GPS 模拟器/虚拟定位 | 多源校验（WiFi+基站+运动传感器） |
| 内容密度 | 冷启动时某些位置无内容 | 运营引导+官方种子内容+奖励机制 |
| 隐私合规 | 位置数据合规要求高 | 明确授权+数据最小化+本地处理优先 |
