# 付费 & 商业化体系 — 详细设计

---

## 一、商业模式概览

### 核心收入模型：**道具消费（虚拟物品）**

校园用户 ARPU 值低但付费意愿有趣味驱动，采用「小额高频」模式：

```
收入结构（预期占比）：
├── 捞人道具消费：60%（核心收入）
├── 动态增强道具：20%
├── 个人装饰（头像框/特效）：10%
└── 会员订阅：10%（V2.0+）
```

---

## 二、道具体系

### 2.1 捞人道具（核心付费）

| 道具 | 价格 | 效果 | 复购率预期 |
|------|------|------|-----------|
| 额外撒网 | 1元/次 | 基础撒网+1次 | 极高（每日消耗） |
| 社交炸弹💣 | 5元/个 | 一次炸出8-15人 | 高（周末/无聊时） |
| 精准网🎯 | 3元/个 | 3次渐进式匹配 | 中（有目标时） |
| 金色招呼卡✉️ | 2元/张 | 打招呼置顶+金色样式 | 中（重要对象） |
| 隐身斗篷🫥 | 1元/24h | 捞别人但不被捞 | 低（特定需求） |
| 回溯望远镜🔭 | 2元/次 | 看到今天捞到你的人是谁 | 中高（好奇心） |

### 2.2 动态增强道具

| 道具 | 价格 | 效果 |
|------|------|------|
| 扩大范围 | 1元 | 可见范围从10米→50米（24小时） |
| 置顶飘带 | 2元 | 该位置优先展示，带金色标记 |
| 动态特效 | 1元 | 发布时附带粒子/光效动效 |
| 匿名发布 | 1元/条 | 隐藏发布者身份 |

### 2.3 个人装饰

| 道具 | 价格 | 效果 |
|------|------|------|
| 头像框 | 3-8元 | 卡片展示时有专属边框 |
| 昵称特效 | 5元 | 彩色/渐变/发光昵称 |
| 专属标签 | 2元 | 卡片多一个自定义标签位 |
| 入场特效 | 5元 | 被捞到时卡片有专属动效 |

## 三、定价策略

### 3.1 核心原则

- **低门槛**：最低1元起，大学生零花钱能接受
- **高频消耗**：核心道具是消耗品不是永久品
- **价值锚定**：5元炸弹 vs 1元单次，让用户觉得炸弹「划算」
- **心理定价**：避免整数（后续可考虑虚拟币体系）

### 3.2 价格带分布

```
1元档：额外撒网/扩大范围/隐身/动态特效/匿名 ← 冲动消费
2元档：金色招呼卡/置顶/望远镜/专属标签 ← 轻度付费
3-5元档：精准网/炸弹/头像框 ← 中度付费
5-8元档：昵称特效/入场特效/高级头像框 ← 装饰消费
```

### 3.3 首充与促销

| 策略 | 内容 |
|------|------|
| 首充礼包 | 首次充值任意金额送1个炸弹+3张招呼卡（价值9元） |
| 新人3天 | 每天5次免费撒网+第3天送精准网1个 |
| 周末活动 | 周六免费次数翻倍（6次） |
| 充值优惠 | 充10送2、充30送8（阶梯激励） |

## 四、支付与货币

### MVP 阶段：直接人民币支付

- 支付方式：微信支付 / 支付宝
- 最低充值：1元
- 无虚拟货币中间层（简化流程）
- 每笔购买实时扣款

### V2.0 规划：虚拟币体系

```
「鱼饵」= 虚拟货币
├── 1元 = 10鱼饵
├── 额外撒网 = 10鱼饵
├── 炸弹 = 50鱼饵
├── 精准网 = 30鱼饵
└── 好处：解耦定价、灵活运营、任务奖励发放
```

## 五、数据模型

```
UserWallet（用户钱包/道具库存）
├── user_id: string (PK)
├── items: Map<ItemType, number> (各道具数量)
└── updated_at: timestamp

PurchaseOrder（订单）
├── id: string
├── user_id: string
├── items: [{ type, quantity, unit_price }]
├── total_amount: number (分)
├── payment_method: enum (WECHAT | ALIPAY)
├── payment_id: string? (第三方支付单号)
├── status: enum (PENDING | PAID | FAILED | REFUNDED)
├── created_at: timestamp
└── paid_at: timestamp?

ItemConsumption（道具使用记录）
├── id: string
├── user_id: string
├── item_type: string
├── context: string (使用场景：fishing/moment)
├── related_id: string? (关联的撒网/动态ID)
└── created_at: timestamp
```

### 核心接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/shop/items | 商店道具列表+价格 |
| POST | /api/shop/purchase | 创建订单+拉起支付 |
| POST | /api/shop/purchase/callback | 支付回调 |
| GET | /api/wallet | 我的道具库存 |
| GET | /api/wallet/history | 消费记录 |

## 六、核心商业指标

| 指标 | 目标（MVP验证） |
|------|---------------|
| 付费渗透率 | ≥8% DAU产生付费行为 |

| 道具使用率 | 购买后24h内使用≥80% |
