| # | 角色 | 故事 | 价值 |
|---|------|------|------|

| S2 | 浏览者 | 走到食堂门口，看看这里别人留了什么 | 好奇、发现新内容 |
| S3 | 暧昧者 | 在TA每天必经的路上留一条只有TA能看的消息 | 浪漫、心动 |
| S4 | 定时者 | 设定周五晚8点TA到操场时看到我的祝福 | 惊喜、仪式感 |
| S5 | 毕业生 | 多年后回到校园，看到自己大一时发的动态 | 怀旧、触景生情 |

### 扩展故事

| # | 角色 | 故事 | 价值 |

| S6 | 发布者 | 想删掉以前发的一条尴尬动态 | 控制感、隐私 |

| # | 场景 | 预期处理 |
|---|------|---------|
| E1 | GPS信号弱/漂移 | 提示「定位不准确，请到开阔地带重试」 |

## 二、核心流程

### 2.1 发布位置动态（Happy Path）

```
1. 用户点击发布按钮
2. 系统获取当前GPS坐标
3. 校验是否在校园围栏内
   ├── 否 → 提示「仅支持校园内发布」→ 结束
   └── 是 → 继续
4. 选择内容类型（图文/视频）
5. 编辑内容（文字+图片/视频）
6. 选择可见性规则：
   ├── 公开：任何人走到10米内可见
   ├── 定向：选择目标用户，TA走到10米内可见
   └── 定时定向：选择目标用户+触发时间段
7. 确认发布位置（地图预览，可微调）
8. 提交 → 内容审核（异步，秒级）
9. 发布成功 → 动态锚定在该坐标
```

### 2.2 浏览位置动态（Happy Path）

```
1. 用户在校园内移动（后台低功耗定位）
2. 进入某条动态的10米触发范围
3. 系统推送轻提醒：「附近有 N 条动态等你解锁」
4. 用户打开App / 点击通知
5. 展示当前位置附近的动态列表（按距离排序）
6. 点击查看详情 → 浏览内容
7. 互动：点赞 / 评论 / 回复
```

### 2.3 定向/定时动态触发

```
1. 目标用户进入动态锚点的10米范围
2. 系统校验：
   ├── 是否是指定的目标用户？
   ├── 是否在指定时间窗口内？（定时模式）
   └── 全部满足 → 触发
3. 向目标用户推送通知：「有人在这里给你留了一条消息 💌」
4. 目标用户查看动态
5. 可选：发布者收到「TA已读」回执
```

### 2.4 异常分支汇总

| 节点 | 异常 | 处理 |
|------|------|------|
| 获取GPS | 定位权限未授予 | 引导开启+说明理由 |
| 获取GPS | 信号弱/精度>30米 | 提示等待/重试 |
| 校园围栏校验 | 在校园边界模糊区 | 20米容差，边界内放行 |
| 内容审核 | 违规内容 | 通知发布者+不展示 |
| 定位触发 | GPS漂移误触发 | 停留3秒以上才算有效到达 |
| 定向动态 | 目标用户已注销 | 动态转为「永不触发」状态 |

## 三、业务规则 & 边界 Case

### 3.1 位置规则

| 规则 | 值 | 说明 |
|------|------|------|
| 发布范围限制 | 校园电子围栏内 | 基于学校注册时设定的多边形围栏 |
| GPS精度要求 | ≤10米 | 精度不达标时提示用户 |
| 动态触发半径 | 10米 | 进入10米内视为「到达」 |
| 到达确认时间 | 停留≥3秒 | 防止路过误触发 |
| 围栏容差 | 20米 | 校园边界20米内仍可发布 |
| 位置微调范围 | 发布时可拖动±20米 | 让用户精确标记 |

### 3.2 内容规则

| 规则 | 值 |
|------|------|
| 文字长度 | 1-500字 |
| 图片数量 | 1-9张 |
| 图片大小 | 单张≤10MB |
| 视频时长 | ≤60秒（MVP可暂不支持视频） |
| 视频大小 | ≤100MB |
| 发布频率 | 同一位置每小时最多发1条 |
| 每日发布上限 | 20条/天 |
| 动态保留时间 | 永久 |

### 3.3 可见性规则详解

**公开模式**：
- 任何在校用户进入10米范围即可见
- 发布后立即生效
- 可被所有人互动（点赞/评论）

**定向模式**：
- 仅指定用户进入10米范围时可见
- 最多指定5个目标用户
- 目标用户可以是好友/通过搜索用户名指定
- 发布者自己在任何位置可见（管理用途）
- 目标用户查看后，发布者收到「已读」通知

**定时定向模式**：
- 在定向基础上增加时间窗口
- 设定开始时间+结束时间（如：周五 18:00-22:00）
- 时间窗口外到达不触发
- 时间窗口过期后，动态仍保留但变为仅发布者可见
- 可设为「重复」（每周五）或「一次性」

### 3.4 边界 Case

| Case | 处理方案 |
|------|---------|
| 同一位置动态过多（>100条） | 分页+按热度/时间排序，最新优先 |
| 用户发布后立刻离开 | 正常锚定，不要求发布者停留 |
| 两个用户在完全相同坐标发布 | 正常共存，按时间排序 |
| 校园围栏变更（新建筑） | 后台可调整围栏，已有动态不受影响 |
| 用户被封禁 | 其动态全部不可见（不删除，解封后恢复） |
| 设备时区不同 | 统一用服务器时间（UTC+8） |

## 四、数据模型概要

### 4.1 核心实体

```
Moment（位置动态）
├── id: string (UUID)
├── author_id: string (FK → User)
├── content: { text, media_urls[], media_type }
├── location: { lat, lng, altitude? }
├── campus_id: string (FK → Campus)
├── visibility: enum (PUBLIC | TARGETED | TIMED_TARGETED)
├── target_users: string[] (定向模式下的目标用户ID)
├── time_window: { start, end, repeat_rule? } (定时模式)
├── status: enum (ACTIVE | DELETED | UNDER_REVIEW | BLOCKED)
├── stats: { view_count, like_count, comment_count }
├── created_at: timestamp
└── updated_at: timestamp

MomentInteraction（互动）
├── id: string
├── moment_id: string (FK → Moment)
├── user_id: string (FK → User)
├── type: enum (VIEW | LIKE | COMMENT)
├── content: string? (评论内容)
├── created_at: timestamp
└── location: { lat, lng } (互动时的位置)

CampusGeofence（校园围栏）
├── campus_id: string (FK → Campus)
├── polygon: GeoJSON (校园边界多边形)
├── center: { lat, lng }
└── radius_hint: number (最大覆盖半径，用于快速过滤)

MomentTriggerLog（触发记录）
├── moment_id: string
├── user_id: string
├── triggered_at: timestamp
├── location: { lat, lng }
└── read_at: timestamp?
```

### 4.2 核心接口

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/moments | 发布动态 |
| GET | /api/moments/nearby?lat=&lng=&radius=10 | 获取附近动态 |
| GET | /api/moments/:id | 获取动态详情 |
| DELETE | /api/moments/:id | 删除动态 |
| POST | /api/moments/:id/like | 点赞 |
| POST | /api/moments/:id/comment | 评论 |
| GET | /api/moments/mine | 我发布的动态 |
| GET | /api/moments/targeted-for-me | 指向我的定向动态 |
| POST | /api/geofence/check | 校验坐标是否在校园内 |

## 五、交互设计要点

### 5.1 发布体验

- 发布入口：底部Tab中央「+」按钮（常驻显眼位置）
- 发布前先展示当前位置地图预览（3D俯视角/卫星图）
- 位置微调：拖拽地图大头针，限制±20米范围
- 可见性选择器：底部Sheet三选一，定向模式有用户搜索/选择
- 发布成功动效：动态从手机「飘」落到地图位置上的粒子动效

### 5.2 浏览体验

- **地图模式**（主视图）：以自己为中心的地图，10米内的动态以气泡/光点形式出现
- **列表模式**（辅助）：当前位置附近按距离排序的动态列表
- 气泡样式：
  - 公开动态 → 蓝色气泡
  - 定向给我的 → 金色气泡 + 闪烁动效
  - 已读 → 半透明
- 走近时渐显动效：从虚化→清晰，有「解锁」的仪式感

### 5.3 定向动态的情感设计

- 收到定向动态通知时：特殊推送样式「有人在这里给你留了悄悄话 💌」
- 查看时：信封拆开动效 → 展示内容
- 查看后：发布者收到「TA已打开你的留言」+ 对方查看时的位置

### 5.4 触发机制的感知

- 用户不需要一直开着App
- 基于 iOS/Android 地理围栏 API（系统级，省电）
- 进入触发范围 → 系统推送 → 点击打开App查看
- App内：顶部横幅「你附近有 3 条新动态」

### 5.5 空状态设计

| 场景 | 空状态文案 |
|------|-----------|
| 当前位置无动态 | 「这里还没有人留下足迹，成为第一个吧 ✨」 |
| 定向动态无人查看 | 「等TA路过这里...」+ 等待动效 |
| 我的动态为空 | 「留下你的第一个足迹，让校园记住你」 |
