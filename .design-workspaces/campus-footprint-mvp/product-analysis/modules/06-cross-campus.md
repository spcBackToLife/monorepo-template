# 跨校探索体系 — 详细设计

---

## 一、用户故事

| # | 角色 | 故事 | 价值 |
|---|------|------|------|
| S1 | 好奇用户 | 想看看隔壁学校的人每天在干什么 | 好奇心、探索 |
| S2 | 异地恋/异校友 | 想看看朋友学校的校园动态 | 保持联系、了解对方生活 |
| S3 | 跨校社交者 | 想认识其他学校的人，拓展社交圈 | 社交扩展 |
| S4 | 竞赛/活动者 | 想跨校组队/找搭子 | 实用需求 |

---

## 二、核心功能

### 2.1 校园地图（发现页）

```
展示形式：
├── 地图模式：中国高校地图，每个学校一个光点
│   ├── 光点大小 = 活跃度（动态数/用户数）
│   ├── 点击学校 → 进入该校「校园广场」
│   └── 支持缩放/搜索学校名
│
└── 列表模式：按热度/距离排序的学校列表
    ├── 学校名 + 城市 + 在线人数 + 今日动态数
    └── 可按省份/城市筛选
```

### 2.2 校园广场

每个学校有一个公开 Feed（仅展示设为「公开」的动态）：

```
校园广场页：
├── 顶部：学校名+校徽+在线人数+简介
├── 内容：公开动态流（按时间/热度排序）
│   └── 注意：只展示文字+图片缩略图，不展示精确位置
├── 互动：可点赞/评论（需登录）
└── 社交：可关注该校用户/发起私信
```

### 2.3 跨校交友

- 浏览其他学校动态时，可点击作者头像查看资料
- 可关注（单向），对方同意后互相关注
- 跨校用户之间私信无限制（已互关）
- 跨校用户不可使用「捞人」功能（仅限本校）

### 2.4 校际互动（V2.0+）

- 话题PK：两所学校同一话题投票/比拼
- 组队招募：跨校比赛/项目找队友的公告板
- 联合活动：官方组织的跨校线上活动

## 三、业务规则

| 规则 | 值 | 说明 |
|------|------|------|
| 可浏览其他学校 | 不限制 | 自由探索 |
| 跨校动态展示 | 仅公开动态 | 定向/定时不跨校可见 |
| 位置信息 | 不展示精确坐标 | 仅显示「图书馆附近」等模糊位置 |
| 跨校关注 | 需对方同意 | 比本校更谨慎 |
| 跨校私信 | 需互关 | 防骚扰 |
| 每日浏览学校数 | 不限制 | 鼓励探索 |

---

## 四、数据模型

```
SchoolProfile（学校公开主页）
├── school_id: string (FK)
├── description: string
├── cover_url: string?
├── active_user_count: number
├── today_moment_count: number
└── hot_tags: string[] (该校热门标签)

CrossSchoolFollow（跨校关注）
├── follower_id: string
├── followee_id: string
├── status: enum (PENDING | ACCEPTED | REJECTED)
└── created_at: timestamp
```

### 核心接口

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/explore/schools | 学校列表（热度/距离） |
| GET | /api/explore/schools/:id | 校园广场Feed |
| GET | /api/explore/schools/:id/info | 学校公开信息 |
| POST | /api/follow | 关注（跨校需审批） |

---

## 五、交互设计要点

- 发现页入口：底部Tab「探索🌍」
- 地图交互：双指缩放，点击学校弹出信息气泡，再点进入广场
- 校园广场顶部：学校封面图+基本信息+「关注该校」按钮
- 跨校动态卡片：左上角显示学校名标签（区别于本校内容）
- 空状态：「这所学校还没有人入驻，邀请你的朋友加入吧」

## 六、模块间依赖

```
跨校探索
├── 依赖：用户体系、位置动态（内容源）、学校数据
├── 被依赖：消息（跨校私信）
└── 关联：运营（校际活动）、增长（邀请新校用户）
```
