## 一、用户故事

### 核心故事

| # | 角色 | 故事 | 价值 |
|---|------|------|------|
| S1 | 想交友的人 | 晚上在宿舍无聊，想看看楼里/附近有谁 | 打发时间、社交好奇 |
| S2 | 害羞的人 | 想认识人但不敢直接搭讪，用游戏方式降低心理负担 | 安全感、低压力社交 |
| S3 | 付费用户 | 今天免费次数用完了但还想继续，愿意花几块钱 | 即时满足 |
| S4 | 被捞到的人 | 收到通知说有人捞到我，有点好奇对方是谁 | 被关注感、好奇 |

### 扩展故事

| # | 角色 | 故事 | 价值 |
|---|------|------|------|
| S5 | 玩家 | 想用「炸弹」一次看到很多人，像开盲盒 | 刺激感、惊喜 |
| S6 | 精准用户 | 想找特定类型的人（同专业/同兴趣） | 效率社交 |
| S7 | 被多次捞的人 | 不想频繁被打扰 | 隐私保护 |
| S8 | 新用户 | 刚注册想体验捞人，需要引导 | 上手顺畅 |

### 异常故事

| # | 场景 | 预期处理 |
|---|------|---------|
| E1 | 附近无人可捞 | 提示「附近暂时没有人，换个时间再来」|
| E2 | 付费失败 | 不扣次数，提示重试 |
| E3 | 捞到已屏蔽/封禁用户 | 跳过，不展示 |
| E4 | 对方设置了「不被捞」| 不出现在池子中 |

## 二、核心流程

### 2.1 基础撒网捞人（Happy Path）

```
1. 用户进入「捞人」Tab
2. 展示当前可用次数（如：今日剩余 2/3 次）
3. 点击「撒网」按钮
4. 动效：网撒出去 → 水花 → 等待2-3秒
5. 结果展示：捞到 1-3 张用户卡片
   ├── 卡片信息：头像/昵称/一句话签名/兴趣标签/距离
   └── 卡片从水面「浮出」的动效
6. 对每张卡片可操作：
   ├── 「打招呼」→ 发送预设/自定义消息
   ├── 「感兴趣」→ 标记（双向感兴趣才配对）
   └── 「跳过」→ 滑走
7. 操作完所有卡片 → 本次捞人结束
8. 更新剩余次数
```

### 2.2 炸弹玩法（付费道具）

```
1. 用户在商店购买「社交炸弹」（5元/个）
2. 进入捞人页，切换到「炸弹」模式
3. 点击「投掷炸弹💣」
4. 动效：炸弹落入 → 爆炸水花 → 大量卡片飞出
5. 一次展示 8-15 张卡片（比普通多很多）
6. 卡片以瀑布流/卡片墙形式展示
7. 逐张浏览并操作
```

### 2.3 精准网玩法（多次尝试型）

```
1. 用户在商店购买「精准网」（3元/个，含3次拉网）
2. 选择筛选条件：
   ├── 性别偏好
   ├── 年级范围
   ├── 兴趣标签（最多3个）
   └── 距离范围（宿舍楼/整个校区）
3. 第1次拉网：出现模糊卡片（只有轮廓+1个标签）
4. 第2次拉网：卡片稍微清晰（头像模糊+2个标签+距离）
5. 第3次拉网：完全清晰（全部信息展示）
6. 每次拉网间隔需等待30秒（制造期待感）
7. 最终展示1-2张精准匹配的卡片
```

### 2.4 打招呼→对话流程

```
1. 在卡片上点击「打招呼」
2. 选择方式：
   ├── 预设模板：「Hi~ 很高兴认识你」/「同校的缘分~」等6-8个
   ├── 自定义文字：限50字
   └── 趣味卡片：选择一张互动卡片（如：「一起去食堂吗？」）
3. 发送 → 对方收到通知「有人在捞你的时候向你打招呼了」
4. 对方查看 → 可回复/忽略/屏蔽
5. 对方回复 → 建立对话（进入私信）
6. 对话开始后双方可互相查看完整资料
```

## 三、业务规则 & 边界 Case

### 3.1 次数与频率

| 规则 | 值 | 说明 |
|------|------|------|
| 每日免费撒网次数 | 3次 | 每日 00:00 重置 |
| 额外撒网单价 | 1元/次 | 无每日上限 |
| 每次捞到人数 | 1-3人 | 随机，附近人多则3人 |
| 炸弹一次人数 | 8-15人 | 固定范围随机 |
| 精准网拉网次数 | 3次/网 | 逐步清晰 |
| 打招呼每日上限 | 10次 | 防骚扰 |
| 同一人被捞频率 | 同一用户最多被同一人捞到1次/天 | 防骚扰 |
| 每日被捞上限 | 20次/天 | 超过后不再出现在池子中 |

### 3.2 匹配池规则

**谁在「池子」里（可被捞到）：**
- ✅ 在校园范围内（GPS确认）
- ✅ 最近24小时内有打开App
- ✅ 未设置「隐身」
- ✅ 今日被捞次数 < 20次
- ✅ 未被当前用户屏蔽/被屏蔽

**匹配权重（决定谁更容易被捞到）：**
- 距离越近 → 权重越高
- 活跃度越高 → 权重稍高
- 资料完整度 → 权重稍高
- 新用户 → 前3天有新手保护加权（容易被捞到）

### 3.3 范围规则

| 捞人方式 | 匹配范围 | 说明 |
|---------|---------|------|
| 基础撒网 | 500米内 | 宿舍楼附近为主 |
| 炸弹 | 整个校园 | 范围大、人多 |
| 精准网 | 用户自选（宿舍楼/校区） | 可精确也可放大 |

### 3.4 安全与隐私

| 规则 | 说明 |
|------|------|
| 距离模糊化 | 展示「约xx米」而非精确距离 |
| 位置不暴露 | 卡片只显示「在你附近」不显示具体坐标 |
| 隐身开关 | 用户可随时设置「不被捞」，立即生效 |
| 屏蔽机制 | 屏蔽后双方永不再匹配 |
| 举报机制 | 卡片/对话中可举报，3次被举报自动临时屏蔽 |
| 性别偏好 | 用户可设置只被同性/异性/不限捞到 |

### 3.5 边界 Case

| Case | 处理方案 |
|------|---------|
| 校园内在线人数<5人 | 放宽到最近48小时活跃用户 |
| 用户反复撒网捞到同一人 | 同一对24小时内只匹配一次 |
| 付费后但匹配不到人 | 不消耗次数，提示稍后重试 |
| 打招呼被忽略 | 72小时无回复提示「对方可能比较忙」 |
| 多人同时捞到同一人 | 正常展示，不互斥 |
| 用户注销后其卡片 | 立即从池子移除 |

## 四、付费体系设计

### 4.1 道具商店

| 道具 | 价格 | 效果 | 说明 |
|------|------|------|------|
| 额外撒网 | 1元/次 | 基础撒网+1次 | 最常用消耗品 |
| 社交炸弹💣 | 5元/个 | 一次炸出8-15人 | 高性价比量大 |
| 精准网🎯 | 3元/个 | 3次渐进式捞取1-2人 | 悬念感+精准 |
| 金色招呼卡✉️ | 2元/张 | 打招呼置顶显示+特殊样式 | 提高回复率 |
| 隐身斗篷🫥 | 1元/24h | 不被别人捞到但自己可以捞 | 隐私保护 |

### 4.2 付费节奏

- 新用户前3天：每天5次免费（引导上瘾）
- 第4天起：每天3次免费
- 首充奖励：首次付费赠送1个炸弹
- 周末活动：周六双倍（免费6次）

### 4.3 付费心理设计

- **稀缺性**：每日有限次数制造「用完了还想」的渴望
- **即时反馈**：付费后立即可用，不是等待
- **不确定性**：每次捞到的人不同（盲盒心理）
- **渐进清晰**：精准网的3次拉网，每次更清楚（好奇驱动）
- **社交压力**：「别人都在用炸弹，你还在慢慢撒网？」

## 五、数据模型概要

### 5.1 核心实体

```
FishingSession（撒网会话）
├── id: string
├── user_id: string (FK → User)
├── type: enum (BASIC | BOMB | PRECISION)
├── matched_users: string[] (匹配到的用户ID)
├── location: { lat, lng }
├── cost: { type: FREE|PAID, amount? }
├── created_at: timestamp
└── status: enum (ACTIVE | EXPIRED)

FishingGreeting（打招呼）
├── id: string
├── from_user_id: string
├── to_user_id: string
├── session_id: string (FK → FishingSession)
├── message_type: enum (TEMPLATE | CUSTOM | CARD)
├── content: string
├── status: enum (SENT | READ | REPLIED | IGNORED)
├── created_at: timestamp
└── read_at: timestamp?

UserFishingConfig（用户捞人配置）
├── user_id: string (PK)
├── is_visible: boolean (是否可被捞)
├── gender_preference: enum (MALE | FEMALE | ALL)
├── daily_fished_count: number (今日被捞次数)
├── last_active_at: timestamp
└── profile_completeness: number (0-100)

FishingItem（道具库存）
├── user_id: string
├── item_type: enum (EXTRA_CAST | BOMB | PRECISION_NET | GOLD_CARD | CLOAK)
├── quantity: number
└── expires_at: timestamp? (某些道具有有效期)

FishingTransaction（购买记录）
├── id: string
├── user_id: string
├── item_type: string
├── quantity: number
├── amount: number (金额，分)
├── payment_method: string
├── status: enum (SUCCESS | FAILED | REFUNDED)
└── created_at: timestamp
```

### 5.2 核心接口

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/fishing/cast | 撒网（基础） |
| POST | /api/fishing/bomb | 使用炸弹 |
| POST | /api/fishing/precision | 使用精准网 |
| POST | /api/fishing/greet | 打招呼 |
| GET | /api/fishing/greetings | 我收到的招呼 |
| GET | /api/fishing/history | 捞人历史 |
| PUT | /api/fishing/config | 更新捞人偏好设置 |
| GET | /api/fishing/items | 我的道具库存 |
| POST | /api/fishing/purchase | 购买道具 |
| GET | /api/fishing/stats | 今日使用统计 |

## 六、交互设计要点

### 6.1 核心视觉隐喻：「海/水」

整个捞人模块用水域/捕鱼作为视觉隐喻：
- 背景：深色水面（夜间感）+ 微弱波纹
- 撒网动效：网从底部甩出 → 落入水面 → 沉入
- 等待：水面冒泡泡 → 暗示有东西
- 结果：卡片从水面浮出 → 水花四溅
- 炸弹：投入水面 → 爆炸 → 大量卡片飞出

### 6.2 卡片设计

```
┌─────────────────────┐
│  ┌───┐              │
│  │头像│  昵称        │
│  └───┘  「一句话签名」│
│                     │
│  🎯 计算机 · 大三    │
│  🏷️ 摄影 游戏 跑步   │
│  📍 约120米          │
│                     │
│  [打招呼]  [感兴趣]  │
└─────────────────────┘
```

### 6.3 操作反馈

| 操作 | 反馈 |
|------|------|
| 撒网 | 网甩出动效 + 震动反馈 |
| 等待中 | 水面冒泡 + 「正在搜索附近的人...」 |
| 捞到人 | 卡片浮出 + 水花音效 + 轻震 |
| 没捞到 | 网收回来是空的 + 安慰文案 |
| 打招呼成功 | 飞出小信封动效 |
| 对方回复 | 特殊通知音 + 高亮提示 |
| 炸弹爆炸 | 大水花 + 震屏 + 卡片雨 |
| 精准网渐进 | 模糊→清晰的高斯模糊递减动效 |

### 6.4 引导设计

- 首次进入：3步引导浮层
  - Step 1：「这里是你的社交渔场，撒网认识附近的人」
  - Step 2：「每天3次免费，试试手气吧」
  - Step 3：「捞到感兴趣的人，打个招呼开始聊天」
- 新手前3天：免费5次+引导完成首次撒网的任务奖励（送1个炸弹）

### 6.5 空状态

| 场景 | 处理 |
|------|------|
| 次数用完 | 「今日额度已用完，明天再来 或 获取更多次数→」|
| 附近无人 | 「附近暂时没有人，试试换个时间段？晚上8-10点最热闹 🌙」|
| 对方无回复 | 72h后「TA可能比较忙，试试跟其他人打招呼吧」|

## 七、模块间依赖

```
捞人社交体系
├── 依赖：用户体系（用户资料+偏好）、位置（匹配范围）、支付（道具购买）
├── 被依赖：消息体系（打招呼进入私信）、数据分析（匹配效果）

```

## 八、核心指标（North Star）

| 指标 | 含义 | 目标 |
|------|------|------|
| 日均撒网次数/人 | 玩法吸引力 | ≥2次（含免费） |
| 打招呼率 | 匹配→社交转化 | ≥40% |
| 打招呼回复率 | 社交质量 | ≥25% |
| 付费转化率 | 商业潜力 | ≥8%（DAU→付费） |
| 次日留存 | 玩法粘性 | ≥45% |
