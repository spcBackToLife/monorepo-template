# 用户体系 — 详细设计

---

## 一、用户故事

### 核心故事

| # | 角色 | 故事 | 价值 |
|---|------|------|------|
| S1 | 新用户 | 用手机号快速注册，选择学校开始使用 | 快速进入 |
| S2 | 新用户 | 完善个人资料（头像/兴趣标签），让别人捞到我时看到 | 被认识 |
| S3 | 用户 | 设置只被异性捞到 | 隐私控制 |
| S4 | 用户 | 后续想认证学校身份（edu邮箱） | 可信度 |
| S5 | 用户 | 想看看自己被多少人捞到过、打过招呼 | 社交数据感知 |

### 异常故事

| # | 场景 | 预期处理 |
|---|------|---------|
| E1 | 手机号已被注册 | 引导登录+找回密码 |
| E2 | 学校不在列表 | 提供手动输入+后台审核 |
| E3 | 上传违规头像 | AI审核拒绝+提示更换 |
| E4 | 忘记密码 | 短信验证码重置 |

## 二、核心流程

### 2.1 注册流程

```
1. 启动App → 欢迎页（3张引导图：位置动态/捞人/回忆）
2. 点击「开始」→ 输入手机号
3. 获取验证码（60秒倒计时）
4. 输入验证码 → 验证通过
5. 设置密码（6-20位，字母+数字）
6. 进入资料完善页：
   ├── 必填：昵称、性别、学校、年级
   └── 选填：头像、一句话签名、兴趣标签（最多5个）
7. 授权位置权限（核心功能依赖，强引导）
8. 注册完成 → 进入主页
```

### 2.2 登录流程

```
1. 输入手机号+密码 → 登录成功
   └── 或：验证码快捷登录
2. 首次在新设备登录 → 短信二次验证
3. 登录后检查位置权限 → 未授权则引导
```

### 2.3 资料编辑

```
可编辑项：
├── 头像（拍照/相册，AI审核）
├── 昵称（2-12字符，唯一性可选）
├── 一句话签名（≤30字）
├── 性别（注册后不可改）
├── 学校（需重新认证才能改）
├── 年级（每年可更新）
├── 兴趣标签（从预设列表选，最多5个）
└── 个人主页背景图（选填）
```

## 三、业务规则

### 3.1 账号规则

| 规则 | 值 |
|------|------|
| 手机号 | 11位，+86，唯一 |
| 验证码 | 6位数字，5分钟有效，同号60秒间隔 |
| 密码 | 6-20位，至少含字母+数字 |
| 昵称 | 2-12字符，不含特殊符号 |
| 每日验证码上限 | 同号10次/天 |
| 注册限流 | 同IP每分钟5次 |

### 3.2 学校认证（预留入口）

- MVP阶段：选择学校即可，不强制验证
- 已认证标识：昵称旁显示「🎓已认证」
- 认证方式（后续）：edu邮箱验证 或 学生证照片审核
- 认证后权益：捞人时优先展示、可信度高

### 3.3 兴趣标签体系

预设分类：
```
├── 运动：篮球/足球/跑步/健身/游泳/羽毛球/乒乓球
├── 文艺：摄影/音乐/绘画/写作/舞蹈/手工
├── 游戏：王者/英雄联盟/原神/Switch/桌游/剧本杀
├── 学术：考研/留学/竞赛/编程/读书/语言学习
├── 生活：美食/旅行/电影/追剧/宠物/穿搭
└── 社交：唱歌/约饭/运动搭子/学习搭子/旅行搭子
```

## 四、数据模型

```
User（用户）
├── id: string (UUID)
├── phone: string (加密存储)
├── password_hash: string
├── nickname: string
├── gender: enum (MALE | FEMALE)
├── avatar_url: string?
├── bio: string? (一句话签名)
├── school_id: string (FK → School)
├── grade: enum (FRESHMAN | SOPHOMORE | JUNIOR | SENIOR | GRADUATE)
├── interest_tags: string[] (最多5个)
├── background_url: string?
├── is_verified: boolean (学校认证)
├── status: enum (ACTIVE | BANNED | DEACTIVATED)
├── created_at: timestamp
└── last_active_at: timestamp

School（学校）
├── id: string
├── name: string
├── province: string
├── city: string
├── geofence: GeoJSON (校园围栏)
└── student_count: number (注册人数)

UserPrivacySetting（隐私设置）
├── user_id: string (PK)
├── fishing_visible: boolean (是否可被捞)
├── fishing_gender_pref: enum (MALE | FEMALE | ALL)
├── moment_default_visibility: enum (PUBLIC | TARGETED)
└── show_distance: boolean (是否展示距离)
```

### 核心接口

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/auth/send-code | 发送验证码 |
| POST | /api/auth/register | 注册 |
| POST | /api/auth/login | 登录 |
| POST | /api/auth/reset-password | 重置密码 |
| GET | /api/users/me | 获取我的资料 |
| PUT | /api/users/me | 更新资料 |
| PUT | /api/users/me/privacy | 更新隐私设置 |
| POST | /api/users/verify-school | 提交学校认证 |
| GET | /api/schools | 学校列表（支持搜索） |

## 五、交互设计要点

- 注册流程尽量精简：一页一步，不要大表单
- 兴趣标签选择：气泡式多选，已选高亮+弹跳动效
- 位置权限引导：单独一屏说明「为什么需要位置」，配插图
- 资料完整度进度条：鼓励完善（「完善资料可提升被捞几率 50%」）
- 头像上传：支持裁剪圆形预览
