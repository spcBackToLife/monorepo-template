# 校园足迹社交 — 产品需求文档 V1.0

---

## 一、产品概述

### 1.1 产品定位

**一款基于地理位置的校园社交产品——让动态「长」在物理世界的坐标上，用空间承载记忆、用游戏化玩法促进破冰。**

### 1.2 目标用户

- **核心用户**：18-24岁在校大学生
- **次要用户**：校友/毕业生（回忆场景）

### 1.3 核心价值

| 价值维度 | 描述 |
|---------|------|
| 位置锚定 | 动态不是「发到网上」而是「留在这个地方」|
| 游戏化社交 | 捞人机制降低搭讪心理门槛 |
| 情感沉淀 | 校园回忆有了物理坐标，触景生情 |
| 趣味互动 | 定向/定时动态让暧昧更有仪式感 |

### 1.4 竞品差异

独特组合：**位置锚定内容 × 游戏化破冰 × 校园场景 × 回忆沉淀**

## 二、已确认产品决策

| 决策项 | 结论 |
|--------|------|
| 位置精度 | GPS米级精确，仅限校园围栏内 |
| 触发距离 | 10米内解锁/触发 |
| 捞人付费 | 每日3次免费 + 1元/次额外 |
| 学校认证 | MVP不强制，预留认证入口 |
| 动态保留 | 永久保留 |
| 校外安全 | 校外不可发布（安全考虑） |

## 四、信息架构

### 4.1 App 页面结构

```
App
├── 启动 & 引导
│   ├── Splash 闪屏
│   ├── 引导页（3张，首次安装）
│   ├── 登录页
│   └── 注册页（分步：手机号→验证码→密码→资料）
│
├── 主 Tab 结构（底部4Tab）
│   │
│   ├── 🗺️ Tab1: 足迹（位置动态）
│   │   ├── 地图视图（默认）：以自己为中心，显示附近动态气泡
│   │   ├── 列表视图：附近动态按距离排序
│   │   ├── 动态详情页
│   │   ├── 评论列表页
│   │   └── 发布入口（中央悬浮按钮）
│   │       ├── 发布页：编辑内容+预览位置
│   │       └── 可见性选择Sheet
│   │
│   ├── 🎣 Tab2: 捞人
│   │   ├── 捞人主页（水面视觉+撒网按钮）
│   │   ├── 结果卡片（浮出展示）
│   │   ├── 打招呼Sheet
│   │   ├── 道具商店页
│   │   └── 捞人历史页
│   │
│   ├── 🌍 Tab3: 探索（V2.0，MVP可用占位）
│   │   ├── 校园地图
│   │   ├── 学校列表
│   │   └── 校园广场页
│   │
│   └── 💬 Tab4: 消息
│       ├── 私信列表
│       ├── 聊天页
│       ├── 互动通知列表（点赞/评论/被捞）
│       └── 系统通知列表
│
├── 个人中心（从Tab1左上头像进入）
│   ├── 个人主页
│   ├── 我的动态（地图标记所有发过的位置）
│   ├── 编辑资料页
│   ├── 隐私设置页
│   ├── 通知偏好页
│   ├── 我的道具/钱包
│   ├── 学校认证入口
│   └── 关于/帮助/反馈
│
└── 全局浮层/弹窗
    ├── 位置权限引导
    ├── 位置动态到达推送弹窗
    ├── 首次捞人引导
    └── 付费确认弹窗
```

### 4.2 页面流转关系

```
注册/登录 ─────→ 主页(足迹Tab)
                      │
         ┌────────────┼────────────┐
         ↓            ↓            ↓
     足迹地图      捞人主页      消息列表
         │            │            │
         ↓            ↓            ↓
    发布动态     撒网→卡片      对话详情
         │            │            │
         ↓            ↓            ↓
    选择可见性    打招呼→        ←─┘(打招呼后进入对话)
                 道具商店
```

### 4.3 导航模式

- **主导航**：底部 Tab Bar（4个Tab）
- **二级导航**：顶部分段控制器（如消息Tab的私信/互动/系统）
- **详情进入**：Push（iOS）/ 右滑进入
- **操作面板**：Bottom Sheet（发布可见性/打招呼方式）
- **全局浮层**：中央悬浮发布按钮

## 五、数据架构概要

### 5.1 核心实体关系

```
User ──1:N──→ Moment（用户发布多条动态）
User ──1:N──→ FishingSession（用户多次捞人）
User ──1:1──→ UserWallet（用户道具钱包）
User ──1:1──→ UserPrivacySetting（隐私设置）
User ──N:1──→ School（归属学校）

Moment ──1:N──→ MomentInteraction（动态被多人互动）
Moment ──1:N──→ LocationTrigger（定向动态的触发器）

FishingSession ──1:N──→ FishingGreeting（一次捞人产生多个打招呼）

Conversation ──1:N──→ Message（对话包含多条消息）

School ──1:1──→ CampusGeofence（学校围栏）
```

### 5.2 核心接口清单（MVP）

| 模块 | 接口数 | 关键接口 |
|------|:------:|---------|
| 认证 | 4 | register, login, send-code, reset-password |
| 用户 | 4 | get-me, update-me, update-privacy, verify-school |
| 位置动态 | 8 | create, nearby, detail, delete, like, comment, mine, targeted |
| 捞人 | 6 | cast, greet, greetings, history, config, stats |
| 消息 | 5 | conversations, messages, send, read, notification-prefs |
| 商店 | 4 | items, purchase, callback, wallet |
| 安全 | 3 | report, credit, appeal |
| **合计** | **34** | |

## 六、非功能需求

### 6.1 性能要求

| 指标 | 目标 |
|------|------|
| 首屏加载 | ≤2秒 |
| 地图渲染 | ≤1秒 |
| 附近动态查询 | ≤500ms |
| 捞人匹配 | ≤2秒（含动效时间） |
| 消息送达 | ≤1秒 |
| 位置触发延迟 | ≤10秒（系统推送） |

### 6.2 安全要求

| 项目 | 要求 |
|------|------|
| 数据传输 | 全站 HTTPS |
| 密码存储 | bcrypt 加密 |
| 位置数据 | 加密存储+最小化采集 |
| 登录安全 | 新设备短信验证+失败锁定 |
| 支付安全 | 接入正规支付渠道+签名验证 |

### 6.3 兼容性

| 平台 | 最低版本 |
|------|---------|
| iOS | 14.0+ |
| Android | 8.0+ (API 26) |
| 定位 | GPS + WiFi + 基站 |

## 七、MVP 范围 & 里程碑

### V1.0 MVP（4-6周）

```
目标：单校试点，验证核心假设
├── 用户注册+资料
├── 位置动态（图文+公开模式）
├── 基础捞人（撒网+打招呼）
├── 简单私信+通知
├── 额外撒网购买（1元/次）
└── 基础内容审核
```

### V1.5 玩法深化（+4周）

```
├── 定向/定时动态
├── 炸弹+精准网道具
├── 完整道具商店
├── 位置触发推送
└── 信用分体系
```

### V2.0 跨校扩展（+6周）

```
├── 跨校浏览+校园广场
├── 跨校交友
├── 视频动态支持
├── 校际活动
└── 完整会员体系
```

## 八、核心指标（North Star Metrics）

| 维度 | 指标 | MVP目标 |
|------|------|---------|
| 增长 | 注册用户数 | 单校500+ |
| 活跃 | DAU/MAU | ≥30% |
| 留存 | 次日留存 | ≥40% |
| 核心行为 | 日均发布动态数 | ≥0.5条/活跃用户 |
| 核心行为 | 日均撒网次数 | ≥2次/活跃用户 |
| 社交 | 打招呼回复率 | ≥25% |
| 商业 | 付费渗透率 | ≥8% |
| 商业 | ARPPU | ≥5元/月 |
