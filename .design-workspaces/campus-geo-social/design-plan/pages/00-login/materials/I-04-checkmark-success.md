# I-04 · checkmark-success · 登录成功 ✓ 图标

> **页面**：00-login · 登录页
> **类型**：Icon（功能图标，P1 优先级）
> **绑定节点**：`pages/00-login/submit-btn`（条件：`state.view.submitState === "success"`）
> **来源**：`design-plan/pages/00-login/visual.md#§5-素材需求清单`

---

## 1. 设计意图

### 基本信息

| 字段 | 值 |
|------|------|
| 编号 | I-04 |
| 名称 | checkmark-success |
| 类型 | Icon |
| 参考框尺寸 | 20×20 px |

### 需求来源

| 来源 | 页面/组件 | 章节 | 需求描述原文 |
|------|---------|------|------------|
| 页面 | 00-login | 素材清单 #I-04 | "登录成功的'勾'反馈，强化仪式感" |
| 视觉分析 | 00-login | §3.6-动效设计 | "spinner → checkmark + bg 草莓粉→薄荷绿" |

### 要解决的问题

- **功能识别**：明确告诉用户"操作成功了"
- **情感传达**：登录成功的瞬间是用户与产品的"小约会成功瞬间"，需要有强正反馈
- **品牌强化**：用项目 success 色 #3FCC93 强化"成功 = 薄荷绿"的语义关联

### 目标用户感受

> 看到草莓粉按钮瞬间变绿 + 白色 ✓ 跳出，用户内心反应：**"成了！进去吧~"**。这种"被肯定"的微仪式感是青春治愈风的核心。

### 与设计系统的关系

- **引用的 Token**：图标本体色 `$token:textInverse` (#FFFFFF)（白色显示在 success 绿色按钮背景上）
- **遵循的规则**：iconSpec.outline + 略加粗线宽（强调"明确成功"）
- **同系列**：未来会扩展 I-05 error-cross / I-06 warning-exclamation 等"状态图标族"，三者必须形态一致

## 2. 风格分析

### 设计方向

| 维度 | 选择 | 理由 |
|------|------|------|
| 简洁 vs 丰富 | 极简（1 笔成型） | ✓ 必须一眼识别 |
| 几何 vs 有机 | 几何（折线） | 标准 ✓ 形 |
| 平面 vs 立体 | 平面 | 全局一致 |
| 规整 vs 随意 | 规整 + 略加粗 | 强调"明确成功" |

### 色彩策略

| 属性 | 值 | Token 引用 |
|------|------|---------|
| 主色 | #FFFFFF | `$token:textInverse` |
| 辅色 | 无 | — |

> 注意：图标本身是白色，背景是 success 绿色（来自 submit-btn 的 background 变化），形成强对比。

### 线条特征

| 属性 | 值 | 理由 |
|------|------|------|
| 线宽 | **2.5px** round | 比常规 2px 略粗 0.5px，强调"明确"（成功不能含糊） |
| 端点 | round | 全局一致 |
| 拐角 | round | 圆头折角，呼应治愈风 |
| 闭合性 | 开放（一条折线） | ✓ 本就是开放 path |
| 细节密度 | 极少（1 个 path） | 越简单越精致 |

### 与同系列素材的关系

属于"状态图标族"（未来含 I-05/I-06）：
- 三者必须填充同样的参考框 20×20
- 都用 outline 2.5px round
- 都用对应语义色（success/error/warning）

## 3. 构图方案

### 参考框与安全区

```
参考框: 20×20 px
安全区: 四周各 3px padding
实际绘图区域: 14×14 px
视觉重心: 略偏下（短臂在左下，长臂在右上）
```

### 图形描述

```
构图分层:

第 1 层: ✓ 折线
  - 形状: 二段折线 path
  - 顶点 1（左起点）: (4.5, 10.5)
  - 顶点 2（拐点底部）: (8.5, 14.5)
  - 顶点 3（右终点）: (15.5, 6)
  - 短臂长度: ≈ 5.6 px（从 1→2）
  - 长臂长度: ≈ 11.1 px（从 2→3）
  - 短长比 ≈ 1:2（经典 ✓ 比例）
  - 填充: 无
  - 描边: textInverse 2.5px round
```

### 正负空间

- 正空间: ~ 15%
- 负空间作用: ✓ 的"轻盈感"来自周围留白

### 视觉重量与平衡

- 视觉重心: 略偏右下（长臂在右上但拐点底在下）
- 非对称: ✓ 本就非对称，这是其语义核心
- 重量分配: 左下 30% + 右上 70%

## 5. 应用效果

### 在目标位置的最终视觉

在 submit-btn 的 success 状态：

```
[submit-btn 49px 高, bg 草莓粉 → bg success #3FCC93 (300ms 过渡)]
  [I-04 20×20 居中 textInverse 白色 ✓]
  [box-shadow 0 0 16px rgba(63,204,147,0.30) glow-success]
```

- **与周围元素的关系**：单独居中于按钮内（替代登录文字）
- **背景对比**：白色 ✓ on 绿色 bg APCA Lc 约 70，清晰
- **缩放清晰度**：20×20 配 2.5px 线宽足够清晰

### 应用后附加样式

```css
width: 20px;
height: 20px;
display: block;
margin: 0 auto;
/* 入场动效：path 渐进绘制 */
animation: checkmark-draw 300ms ease-out 100ms both;
```

```css
@keyframes checkmark-draw {
  0% { stroke-dasharray: 0 30; }
  100% { stroke-dasharray: 30 0; }
}
```

> **进阶**：可以用 SVG stroke-dashoffset 实现"勾画"动画。如果素材是 PNG 则简单 fade-in 即可。

### 应用技术方案

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | submit-btn 内创建 success-icon div 20×20 | 默认 display:none |
| 2 | material-painter 绘制 PNG | 按 §3 |
| 3 | export_and_apply | — |
| 4 | success 状态时 display:block + 入场动画 | 由交互层处理 |

## 6. 绘制要求

### 画布规格

| 属性 | 值 |
|------|------|
| 参考框 | 20×20 px |
| 背景 | transparent |
| 目标节点 | `submit-btn` 内嵌 success-icon |

### 图形描述（自然语言）

> 一个白色 outline 的 ✓（对勾）图标。  
> 由一条折线 path 组成，经过 3 个顶点：  
> - 左下短臂起点 (4.5, 10.5)  
> - 拐点底部 (8.5, 14.5)  
> - 右上长臂终点 (15.5, 6)  
>
> 线宽 2.5px（比标准图标略粗），端点和拐点都 round 圆润。  
> 颜色纯白 #FFFFFF（因为要显示在绿色 success 按钮背景上）。  
>
> 整体形态是经典 ✓，短长臂比例约 1:2，符合视觉黄金比。

### 要求清单

- **色彩**：仅 `$token:textInverse` (#FFFFFF)
- **线宽**：2.5px round（注意比 I-01/02 的 1.5 / 全局 2.0 都粗，是"明确成功"的视觉强调）
- **风格**：纯 outline 折线
- **对齐**：基于 20×20 参考框，按 §3 顶点精确绘制
- **同系列一致性**：与未来 I-05 error-cross / I-06 warning-exclamation 同尺寸同线宽

### 质量标准

- [ ] 在 20×20 内位置正确（按顶点坐标）
- [ ] 2.5px 线宽清晰
- [ ] 颜色纯白（无任何透明度）
- [ ] 拐角和端点都 round（无任何尖角）
- [ ] 透明背景
- [ ] 短长臂比例约 1:2
