# M6 社交关系链

> **优先级**：P0 - 捞网/动态浏览之后的承接，私聊与好友是社交闭环必需
> **决策依据**：D1（投递需好友 → 关系链是必经路径）、D6（治愈风格）
> **关联模块**：M2 捞网（招呼通过 → 临时会话 → 加好友）、M1 动态（评论/分享）、M9 审核（消息内容）

---

## Step A: 用户故事

### 核心故事
- 作为**用户**，我希望对捞到打过招呼的人发起好友请求，对方同意后成为好友
- 作为**用户**，我希望能与好友 1v1 私聊（文字/图片/语音/动态分享/地点）
- 作为**用户**，我希望从招呼回复 → 自动进入"临时会话"，不必先加好友就能开始聊（保持节奏）
- 作为**用户**，我希望好友列表清晰展示在线状态、最近一条动态预览
- 作为**用户**，我希望删除/拉黑某个好友，并屏蔽对方所有信息流

### 扩展故事
- 作为**用户**，我希望长按消息撤回（2 分钟内）
- 作为**用户**，我希望发送动态卡片到私聊（"看，我刚发的这条"）
- 作为**用户**，我希望发送地点卡片（"我在这里，过来吧"）
- 作为**用户**，我希望好友请求附带验证留言（"我们是 X 校园 Y 楼下捞到的"）
- 作为**用户**，我希望聊天有"对方正在输入"提示，"已读"/"未读" 状态可选开关

### 异常故事
- 作为**用户**，被拉黑后无法再发好友请求 → 提示"该用户已不希望被你联系"
- 作为**用户**，临时会话超过 7 天未互动 → 自动归档，但不删除消息记录
- 作为**用户**，发送的消息被审核拦截 → 通知"消息因违规未送达"
- 作为**用户**，对方账号已注销 → 会话标记"用户已不存在"，仍可查看历史

## Step B: 核心流程

### B1. 好友请求主线

```
入口 1：M2 捞网卡片 → 打招呼 → 对方回复 → 私聊中 → "加好友"
入口 2：M1 动态详情 → 用户头像 → 个人主页 → "加好友"
入口 3：搜索（M12）→ 用户主页 → "加好友"

请求流程：
1. 点击"加好友" → 验证留言输入页 → 确认发送 → 进入"待处理"状态
→ 2. 对方收到通知（M8）+ "好友请求"列表中可见
→ 3. 对方操作：
   ├── 同意 → 双向好友 → 双方收到通知 → 自动建立永久会话
   ├── 拒绝 → 我方收到"未通过"，但不告知拒绝详情（保护对方）
   └── 不操作 → 7 天后请求自动过期
```

### B2. 临时会话 → 永久会话

```
M2 招呼通过 → 临时会话建立（标"临时"标签）
→ 临时会话中可正常聊天
→ 任意一方可发起"加为好友"
→ 双方都同意后 → 升级为"永久会话"，标签消失

临时会话规则：
- 最多保留 7 天，期间未互动则自动归档（消息记录可看，不再显示在主列表）
- 任意一方拉黑 → 立即终止
- 7 天内一方未读 ≥ 24h → 提醒"对方还没看到你的消息哦"（仅一次）
```

### B3. 私聊主线

```
1. 消息列表 → 点击会话 → 进入聊天页
→ 2. 消息组件：文字 / 图片 / 语音 / 动态卡片 / 地点卡片 / 表情贴纸
→ 3. 发送 → 上行（带本地 ID）→ 服务端落库 → 推送对方
→ 4. 对方端接收 → 已送达回执 → 用户查看 → 已读回执
→ 5. 长按消息 → 复制 / 撤回（2 分钟内）/ 转发 / 举报
```

### B4. 拉黑与解除

```
个人主页 / 会话设置 → 拉黑 → 警告"双方将无法看见对方"
→ 双向不可见生效：
  - 互不能私聊
  - 互不能在 M2 捞网中出现
  - 互看不到动态、个人主页、评论
  - 现有好友关系自动解除
→ 设置 → 拉黑列表 → 解除拉黑（24h 内不可在重加好友，防反复）
```

### B5. 异常分支

```
好友请求：
├── 已是好友再发请求       → 阻断，提示"已是好友"
├── 已发请求未处理再发      → 阻断，引导"请等待对方处理"
├── 对方设置不允许陌生加友  → 阻断，提示"该用户仅接受熟人加好友"
└── 已被对方拉黑           → 阻断，提示"无法发送"

私聊：
├── 网络中断              → 消息本地保存"发送中"+ 重试机制
├── 对方下线              → 正常发送，对方上线后推送
├── 单条消息违规          → 客户端立即"已发送" + 服务端标记不送达 + 通知发送方
├── 视频/大文件超限       → 客户端阻断，明确提示
└── 消息已读但用户清空了 App 缓存 → 消息从云端拉取
```

## Step C: 业务规则与边界 Case

### C1. 关系数据规则

#### 关系类型
| 关系 | 说明 | 数据表 |
|------|------|-------|
| 好友（双向） | 必须双方同意 | Friendship |
| 临时会话 | 招呼回复建立，7 天活跃期 | TempConversation |
| 拉黑（单向） | 单方面操作即可 | BlockRecord（M2 已定义） |

#### 好友数量限制
- 单用户好友上限 **500** 人（防止滥加）
- 待处理好友请求上限 100 条（清理后才能继续）

### C2. 私聊规则

#### 消息类型
| 类型 | 内容限制 |
|------|---------|
| 文字 | ≤ 500 字 |
| 图片 | jpg/png，单条 1 张，≤ 10MB |
| 语音 | ≤ 60 秒，pcm/aac 编码 |
| 动态卡片 | 引用 Moment ID，预览展示 |
| 地点卡片 | GPS 点 + 地点名 |
| 表情贴纸 | 系统贴纸 + 用户收藏（可有付费贴纸接 M7） |
| 视频 | MVP 不支持视频私聊（避免大流量 + 隐私敏感） |

#### 已读/未读
- 已读功能默认开启
- 用户可在隐私设置（M11）关闭"显示已读状态"（互相关闭后，谁都看不到谁的已读）
- 已读状态在网络好时实时同步，弱网时延迟最多 30 秒

#### 撤回
- 2 分钟内可撤回任意消息
- 撤回后双方仅显示"消息被撤回"
- 撤回不计入历史可恢复（合规审计仅服务端日志保留 90 天）

#### 消息存储
- 客户端缓存最近 6 个月
- 云端永久（除主动删除/账号注销）
- 用户可"清空与某人的聊天记录"（仅本地，云端保留）

### C3. 业务规则

- **临时会话不算好友**，但占用消息列表（带"临时"标签）
- **加好友请求附带留言**，对方有更多上下文判断
- **拉黑生效全方位**（私聊/捞网/动态/评论/主页）
- **消息违规**：内容审核（M9）异步检查，违规消息撤回 + 通知双方"该消息已不可见"
- **机器人/客服账号**：MVP 不支持
- **群聊**：MVP 不做

### C4. 边界 Case

| 场景 | 处理 |
|------|------|
| 已是好友又拉黑 → 解除拉黑后还是好友吗？ | 不是。拉黑会清除好友关系，需重新加 |
| 短时间频繁加同一人为好友 | 单 user → user 30 天最多发 3 次请求 |
| 私聊中互发地点卡片 | 仅展示坐标，不暴露双方实时位置 |
| 网络分区导致重复发送 | 客户端 idempotency token 去重 |
| 多端同时聊天 | 消息走云端转发，多端同步（最多 3 端） |
| 对方账号注销期内 | 会话仍存在，标"用户已不存在"，仅可查看历史 |
| 消息撤回时网络中断 | 客户端排队，恢复后执行；超时（10s）失败提示 |

### C5. 反骚扰与合规

- 单聊新建会话频率限制（防群发骚扰）
- "举报会话"通道：M9 介入审核
- 未成年保护：MVP 不展示（用户都是成年大学生），但保留接口余量
- 合规：消息内容服务端可审计（响应监管要求），但不主动展示给运营

## Step D: 数据模型概要

### 涉及实体

```typescript
Friendship {                    // 好友关系（双向，但用单条记录代表）
  user_a_id: bigint,            // 较小 ID
  user_b_id: bigint,            // 较大 ID
  established_at: timestamp,
  source: enum,                 // fishing_greet | profile_add | search
}

FriendRequest {
  id: bigint,
  from_user_id: bigint,
  to_user_id: bigint,
  message: string,              // ≤ 50 字
  source_context: object,       // 来自捞网/主页/搜索
  status: enum,                 // pending | accepted | declined | expired | withdrawn
  created_at: timestamp,
  responded_at: timestamp | null,
}

Conversation {                  // 会话
  id: bigint,
  participants: [bigint, bigint],
  type: enum,                   // temp | permanent
  established_at: timestamp,
  last_message_at: timestamp,
  // temp 类型 7 天后自动转 archived
}

Message {
  id: bigint,                   // 全局 ID（雪花）
  conversation_id: bigint,
  sender_id: bigint,
  type: enum,                   // text | image | voice | moment_card | location_card | sticker
  content: object,              // 类型相关 JSON
  status: enum,                 // sending | sent | delivered | read | recalled | violation
  client_id: string,            // 幂等
  created_at: timestamp,
  recalled_at: timestamp | null,
}

ConversationMember {            // 会话成员状态（已读位置/通知偏好等）
  conversation_id: bigint,
  user_id: bigint,
  last_read_message_id: bigint,
  unread_count: int,
  notif_muted: bool,
  pinned: bool,
  archived_at: timestamp | null,
}
```

### 涉及接口

```
# 好友
POST   /friend-requests                  发送好友请求
GET    /me/friend-requests?type=incoming|outgoing  好友请求列表
POST   /friend-requests/{id}/accept      同意
POST   /friend-requests/{id}/decline     拒绝
POST   /friend-requests/{id}/withdraw    撤回
DELETE /me/friends/{user_id}             删除好友（含双向同步）
GET    /me/friends                       好友列表

# 拉黑
POST   /me/blocks                        拉黑（与 M11 共用）
DELETE /me/blocks/{user_id}              解除拉黑
GET    /me/blocks                        拉黑列表

# 会话与消息
GET    /me/conversations                 会话列表
GET    /conversations/{id}/messages      消息列表（分页）
POST   /conversations/{id}/messages      发送消息
POST   /messages/{id}/recall             撤回
PATCH  /conversations/{id}/read          标记已读到某 message_id
PATCH  /conversations/{id}/settings      会话设置（置顶/静音）
DELETE /conversations/{id}/local         本地清空（同步给当前用户）
```

## Step E: 交互要点

### 消息列表
- **临时会话标签**：左上角小标签（治愈风浅紫色），到期前 24h 标"即将归档"
- **未读红点**：圆点 + 数字（≥99 显示 99+）
- **置顶会话**：用浅黄背景区分
- **左滑操作**：标记已读 / 静音 / 删除

### 聊天页
- **气泡样式**：自己右侧奶油黄气泡，对方左侧白底浅边
- **送达状态**：发送中（loading）/ 已送达（√）/ 已读（√√）/ 失败（红 !）
- **撤回提示**："你撤回了一条消息"+ 灰色文字
- **图片预览**：双指缩放 + 单击关闭
- **语音消息**：长按录制 + 上滑取消 + 进度条 + 播放转写（V1.x 可加）
- **动态卡片**：引用 Moment 缩略 + 一句话 + "查看完整动态" CTA
- **地点卡片**：迷你地图 + 地址 + "导航过去"

### 加好友页面
- **验证留言**：必填 ≥ 5 字，避免空请求
- **对方设置友好提示**：如"对方仅接受熟人"则给出指引
- **来源标注**："来自 X 校园 Y 楼下的捞网"，提升通过率

### 微动效（D6 治愈）
- 消息发送：右侧气泡微弹 + 一道流光"飞出"
- 收到消息：左侧气泡轻轻"落下"
- 撤回：消息淡出 + 替换文字
- 加好友成功：双方头像间一道连线 + 礼花

## 模块关联

```
M6 → 依赖：M5 认证、M9 内容审核、M2 捞网（临时会话来源）、M7（贴纸付费）
   → 被依赖：M1 动态评论 @ / 分享、M3 胶囊分享给好友（V1.x）
   → 关联：M8 通知（新消息/好友请求/已读回执）、M10 主页（好友列表入口）、M11 隐私（已读开关/接受范围）
```

## 风险标记

| 风险 | 缓解 |
|------|------|
| 私聊违规内容（黄赌毒/诈骗）转化 | M9 内容审核必须前置；高敏感词实时拦截 |
| 用户骚扰：陌生人加好友轰炸 | 频率限制 + 隐私设置（仅熟人可加）+ 拉黑 |
| 消息存储成本高 | 老消息冷存 + 用户可清理 + 媒体走 CDN |
| 已读功能引发社交焦虑 | 提供"互相关闭已读"选项 |
| IM 长连接稳定性 | 走专业 IM 服务（自研或第三方），网络降级走轮询兜底 |
