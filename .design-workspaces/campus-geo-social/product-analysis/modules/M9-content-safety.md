# M9 内容审核与安全

> **优先级**：P0 - 横切所有内容生成模块，合规红线
> **决策依据**：D2 强认证、D6 治愈风格（不是冷冰冰的封禁通知）
> **关联模块**：M1 动态、M2 捞网卡片/招呼、M3 胶囊（极端违法情况）、M6 私聊消息、M5 用户封禁、M11 隐私

---

## Step A: 用户故事

### 核心故事
- 作为**平台合规方**，我希望所有用户生成内容（动态/招呼留言/私聊消息）经过审核，避免违法违规内容传播
- 作为**用户**，我希望对违规内容能一键举报，并知道我的举报后续如何处理
- 作为**用户**，我希望被误判时能申诉并获得人工复核
- 作为**被骚扰用户**，我希望能拉黑骚扰者，骚扰者无法再以任何方式触达我
- 作为**用户**，我希望平台有清晰的社区公约，知道哪些不能做

### 扩展故事
- 作为**用户**，我希望我的私聊**不会被运营随意查看**（只有违法/被举报且严重时才介入）
- 作为**未成年用户**（虽然 MVP 都是大学生 ≥18，但保留机制）→ 自动启用更严格的功能限制
- 作为**用户**，我希望知道平台不存留我的人脸/身份证，仅用于一次性核身

### 异常故事
- 作为**用户**，发布动态被审核拦截 → 明确告知拦截原因（哪类违规），不让用户猜
- 作为**用户**，被多次举报但内容并未违规（恶意举报）→ 系统应识别恶意举报方，不影响我
- 作为**审核员**，需要批量处理工具 + 完整上下文 + 风控提示

## Step B: 核心流程

### B1. 主动审核（发布时）

```
用户提交内容 → 入审核队列 → 三层处理：
├── L1 关键词黑名单（毫秒级）：政治敏感/违法关键词 → 直接拒绝 + 通知用户
├── L2 模型审核（秒级）：图片/视频识别色情/暴力/政治；文本识别情感/分类
│   ├── 高风险（confidence > 0.9）→ 拒绝
│   ├── 中风险（0.5-0.9）→ 进入人工队列，状态"reviewing"
│   └── 低风险（<0.5）→ 通过
└── L3 人工审核（小时级）：仅 L2 中风险 + 用户举报 → 审核员决定通过/拒绝/封禁
```

### B2. 被动审核（用户举报）

```
1. 用户在违规内容上点"举报" → 选择举报类型（多选 + 文字补充）
→ 2. 后端记录举报 + 累计计数
→ 3. 触发条件：
    ├── 单条内容 ≥ 3 个独立用户举报 → 立即下架 + 进人工队列
    ├── 单用户 ≥ 5 条内容被举报 → 限制部分功能 + 进人工队列
    └── 严重举报（涉黄/涉政/涉恐）→ 立刻下架 + 高优先级队列
→ 4. 人工审核 → 处理决定（恢复/下架/封禁/警告）
→ 5. 通知双方（举报方"已处理" / 被举报方"内容因 X 被处理"）
```

### B3. 申诉流程

```
被处罚用户进入"我的申诉"页 → 提交申诉（文字 ≤ 200 字 + 可选证据图片）
→ 客服 48h 内审核 → 申诉成功（恢复 + 致歉）/ 申诉失败（维持原决定）
→ 失败可二次申诉（最多 2 次）
```

### B4. 拉黑（与 M11 共用）

```
入口：用户主页 / 会话设置 / 动态详情 / 捞网卡片 → "拉黑"
→ 双向不可见全方位生效（私聊/捞网/动态/评论/主页）
→ 24h 内不可解除（防误操作）
→ 已建立的好友关系自动解除
```

## Step C: 业务规则与边界 Case

### C1. 审核范围与策略

| 内容类型 | 审核入口 | 策略 |
|---------|---------|------|
| 动态文字（M1） | 发布时 + 编辑时 | 全量过滤 + 模型 |
| 动态图片/视频（M1） | 发布时 | 模型识别 + 高风险人审 |
| 评论（M1） | 发布时 | 文本过滤 |
| 捞网卡片简介（用户资料） | 资料编辑时 | 文本过滤 |
| 招呼留言（M2） | 发送时 | 文本过滤（轻量，避免误拦） |
| 私聊消息（M6） | 异步抽审 + 关键词实时 | 隐私优先：仅高敏感词实时拦，常规消息异步抽审 |
| 头像/学生证（M5） | 上传时 | 模型识别（人脸/身份证检测） |
| 胶囊（M3） | 不入广场，不主动审 | 仅极端违法情况由客服介入 |

### C2. 违规分级

| 级别 | 类型 | 处理 |
|------|------|------|
| L0 极端违法 | 涉政/涉恐/儿童色情/严重诈骗 | 立刻下架 + 永久封禁 + 报警 |
| L1 严重违规 | 一般色情/暴力/赌博/严重谩骂 | 下架 + 24h-7d 禁言 + 警告 |
| L2 中度违规 | 营销广告/虚假信息/低俗内容 | 下架 + 警告 |
| L3 轻度违规 | 错别字/略涉敏感（误判风险高） | 不下架 + 提示用户 |

### C3. 处罚机制

| 处罚 | 触发 | 时长 | 影响 |
|------|------|------|------|
| 警告 | L2-L3 一次 | - | 弹窗提示 |
| 内容下架 | L1+ | 永久 | 该条内容不可见 |
| 禁言 | L1 | 24h / 7d / 30d 递进 | 不能发布动态/评论/招呼 |
| 限功能 | L1 累计 | 7-30d | 不能开网/捞人 |
| 封号 | L0 / L1 严重累计 | 永久 | 账号注销，禁止再注册 |

### C4. 业务规则

#### 举报机制
- 单用户日举报上限 50 次（防滥用）
- 同一内容同一用户只能举报一次
- 恶意举报（屡次举报无效内容）→ 举报权重降低，多次违规者举报失效
- 举报内容 + 类型 + 备注全程留痕

#### 审核员权限
- 审核员可看到举报上下文（被举报内容 + 举报者意见 + 历史风险）
- 审核员**不能**看到完整私聊上下文（仅展示被举报的特定消息 + 前后 3 条）
- 所有审核操作留痕（谁在何时做了什么决定）

#### 申诉政策
- 申诉响应 SLA：48 小时
- 申诉成功率作为审核员考核指标
- 二次申诉失败后不可再申诉

### C5. 隐私数据保护

| 数据 | 加密 | 访问限制 |
|------|------|---------|
| 真实姓名 | AES 加密存储 | 仅认证子系统 + 合规审计 |
| 身份证号 | 哈希 + 加密 | 仅认证 + 风控 + 合规 |
| 人脸照片 | 加密存储 | **仅核身使用，6 个月后自动删除原始照片，仅保留特征向量** |
| 学生证照片 | 加密存储 | 仅认证 + 合规审计 |
| GPS 历史轨迹 | **不存储** | 仅当下定位用于功能判定，不入库 |
| 私聊原文 | 加密存储 | 仅会话双方可读，运营/审核不可读（除被举报且高风险） |

### C6. 边界 Case

| 场景 | 处理 |
|------|------|
| AI 模型误判（如把毕业照判为色情） | 用户申诉 → 人工复核 → 模型反馈训练 |
| 用户群体性举报某政治异见内容 | 仍按规则处理；申诉通道保留 |
| 私聊敏感关键词实时拦截但仅是日常用语 | 关键词需精细分级 + 上下文模型 |
| 同一用户多账号违规 | 设备指纹 + 实名一致性，跨账号封禁联动 |
| 跨校广场内容违规 | 一处下架，双校（发布校 + 浏览校）都不可见 |
| 胶囊 50 年后开启发现内容现在违规（如截图含旧 logo） | 不主动审核，但用户申报后客服介入 |

### C7. 合规要求

- 用户协议、隐私政策、社区公约清晰可访问
- 实名注册 + 一人一号
- 内容存证（关键违规留痕 ≥ 6 个月，配合监管）
- 重大事件应急响应预案
- 未成年保护机制（虽 MVP 用户均成年，但需有标识）
- 数据安全合规（按《个人信息保护法》《数据安全法》）

## Step D: 数据模型概要

### 涉及实体

```typescript
ContentReview {                 // 审核记录（异步任务）
  id: bigint,
  target_type: enum,            // moment | comment | message | profile | avatar | student_id
  target_id: bigint,
  level: enum,                  // L0 | L1 | L2 | L3 | passed
  source: enum,                 // auto_keyword | auto_model | user_report | manual
  reasons: string[],
  decision: enum,               // pending | passed | rejected | manual_required
  reviewer_id: bigint | null,   // 人工审核员
  reviewed_at: timestamp,
  notes: string,
}

UserReport {                    // 用户举报
  id: bigint,
  reporter_id: bigint,
  target_type: enum,            // moment | comment | user | message | greet
  target_id: bigint,
  reasons: string[],            // 多选标签
  description: string,
  evidence_urls: string[],
  status: enum,                 // pending | reviewed | invalid | malicious
  created_at: timestamp,
}

UserPunishment {                // 用户处罚记录
  id: bigint,
  user_id: bigint,
  type: enum,                   // warning | mute | function_limit | ban
  scope: enum,                  // global | post_only | fishing_only
  reason: string,
  related_content_ids: bigint[],
  starts_at: timestamp,
  ends_at: timestamp,            // null = 永久
  appealed: bool,
  appeal_status: enum,
}

Appeal {
  id: bigint,
  user_id: bigint,
  punishment_id: bigint,
  message: string,
  evidence_urls: string[],
  status: enum,                 // pending | upheld | overturned | reappeal
  reviewer_id: bigint | null,
  reviewed_at: timestamp,
  decision_notes: string,
}

KeywordRule {                   // 关键词规则（运营维护）
  id: bigint,
  keyword: string,
  level: enum,                  // L0 | L1 | L2 | L3
  scope: enum,                  // all | text | nickname | message
  active: bool,
  created_at: timestamp,
}

CommunityGuideline {            // 社区公约（用于展示）
  version: string,
  content_md: string,
  effective_at: timestamp,
}
```

## Step E: 交互要点

### 举报流程
- **入口位置**：每个 UGC 内容的"..."菜单中
- **举报弹窗**：分类（清晰大按钮）+ 详细说明 + 可选附图
- **提交后反馈**：温和的"已收到，将在 24h 内处理"
- **处理结果通知**：M8 推送"你举报的内容已被下架"

### 内容被驳回提示（青春治愈风，不冷冰冰）
- **图标**：拟人化"哎呀"小猫/小熊
- **文案**："这条动态有些地方不太合适哦，能调整一下吗？"
- **具体原因**：折叠展示，不直接显示在主提示（避免负面情绪）
- **CTA**："修改后重发" / "查看社区公约" / "申诉"

### 处罚通知
- 警告：在线 toast + 设置中累计可见
- 禁言：弹窗 + 倒计时显示剩余时间
- 限功能：相应入口显示"功能受限，原因：xxx"
- 封号：登录失败时清晰展示原因 + 申诉入口

### 申诉页
- 简洁表单：原因 + 详细说明 + 证据上传
- 历史申诉记录可见
- 处理中、已处理状态清晰

## 风险标记

| 风险 | 缓解 |
|------|------|
| 误伤率高引发用户流失 | 模型分层（高/中/低风险）+ 申诉机制 + 持续校准 |
| 黑产集体注册账号刷恶评 | 设备指纹 + 行为指纹 + 强认证 + 风控 |
| 私聊监控引发隐私争议 | 仅"实时关键词拦"+"举报后审"，不做全量监听 |
| 跨校内容审核滞后 | 跨校广场内容做异步预审 + 二次校验 |
| 监管要求快速响应 | 应急下架开关 + 7×24 客服 + 留痕系统 |
