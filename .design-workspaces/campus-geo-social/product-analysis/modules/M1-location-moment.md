# M1 位置动态系统

> **优先级**：P0 - 产品骨架，没有它没有产品
> **决策依据**：D1 投递需好友、D5 胶囊仅本人、D6 青春治愈
> **关联模块**：M3 时空胶囊（共享数据模型，可见性规则不同）、M2 捞网（消费动态数据）、M4 跨校（动态聚合到广场）、M9 审核

---

## Step A: 用户故事

### 核心故事
- 作为**学生**，我希望在校园某个具体地点（比如 28 楼楼下）发图文/短视频动态，并选择"公开给所有路过此地的人"，以便分享当下心情、贴生活印记
- 作为**学生**，我希望走在校园里，地图上能看到附近发生了什么，以便发现身边的有趣内容
- 作为**好友**，我希望能给特定好友定时定点投递（明天 8 点她到操场时弹一段视频），以便制造惊喜
- 作为**用户**，我希望发完动态后能看到"有 X 个人路过看到了我的动态"，以便有内容被关注的反馈感
- 作为**浏览者**，我希望能给动态点赞、评论，以便表达喜欢/认识发布者

### 扩展故事
- 作为**用户**，我希望选择"匿名发布"（陌生人看到不显示我是谁，好友能看到），以便保护隐私同时享受社交
- 作为**用户**，我希望编辑或删除自己的动态
- 作为**用户**，我希望屏蔽某条动态对某些人不可见（黑名单）
- 作为**用户**，我希望转发/分享某条动态到外部（微信/微博）
- 作为**用户**，我希望看到的动态可以按"最新/最热/相关"切换排序

### 异常故事
- 作为**用户**，发布时网络中断 → 草稿自动保存，恢复网络后断点续传
- 作为**用户**，定位失败/拒绝授权 → 引导手动选点 + 解释为什么需要定位
- 作为**浏览者**，附近没有动态 → 不能展示"空白"，要给"扩大范围/去广场"的引导
- 作为**用户**，被举报内容下架 → 通知发布者，提供申诉

## Step B: 核心流程

### B1. 发布动态主线

```
1. 主页/+号 入口 → 2. 选择类型（图文 / 视频） 
→ 3. 选/拍内容 → 4. 编辑（文字/贴纸/标签）
→ 5. 选择地点（系统默认当前位置，可改）
→ 6. 设置可见性（核心步骤，三选一）：
    ├── 公开 → 所有人到此地点 + 时间窗口内可见
    ├── 投递给好友 → 选择 1+ 好友 + 设置触发时间窗口
    └── 私人胶囊（M3 模块） → 跳转 M3 流程
→ 7. 设置触发条件：
    7.1 半径：50m / 200m / 500m （GPS 容错档位）
    7.2 时效：永久 / 自定义结束时间 / 24h / 7天
→ 8. 内容审核（M9 异步）→ 9. 发布成功 → 10. 显示在地图上
```

### B2. 浏览动态主线

```
A. 地图模式（默认）
   主页 → 地图渲染当前位置 → 地图上散布动态点位（聚合 + 单点）
   → 点击点位 → 展开附近动态预览卡 → 点击进入详情

B. Feed 模式
   主页 → 切换到列表 Tab → 按距离/最新/最热排序的卡片流
   → 上拉加载更多 → 点击进入详情

C. 详情页
   动态详情 → 媒体（轮播）→ 文字 → 地点（迷你地图）→ 互动（点赞/评论/分享）
   → 评论列表 → 输入评论
```

### B3. 异常分支

```
发布异常：
├── 定位授权被拒    → "为什么需要定位"说明卡片 + 跳转设置 / 手动选点
├── 网络中断        → 草稿本地保存 + 恢复后续传 + Toast"已保存到草稿箱"
├── 媒体上传失败    → 失败位置标记 + 单独重试按钮（不重传成功的）
├── 内容审核被驳回  → 通知 + 详情页可见但仅自己 + 申诉入口
├── 选地点超出本校 → 警告"此位置不在你的主校园"，可选切到广场或继续

浏览异常：
├── 定位授权被拒    → 默认显示主校园中心，提示"开启定位获得更精准内容"
├── 附近无内容      → 空状态插画 + "扩大范围 / 去广场看看 / 自己来发一条" 三个 CTA
├── 网络异常        → 缓存上次内容 + 顶部 banner 提示
├── 动态被发布者删除 → 详情页提示"动态已删除" + 返回上一页
```

## Step C: 业务规则与边界 Case

### C1. 数据规则

| 字段 | 规则 |
|------|------|
| 文字内容 | 0-1000 字（可空，纯图也可发） |
| 图片 | jpg/png，单条 ≤ 9 张，单张 ≤ 10MB，自动压缩 |
| 视频 | mp4，≤ 60 秒，≤ 100MB，自动转码 |
| 触发地点 | GeoPoint(lat,lng) + 可选 placeName |
| 触发半径 | 枚举档位 50/200/500m，默认 200m（平衡 GPS 漂移与定向精度） |
| 时效 | enum: forever / 24h / 7d / custom（custom 不超过 365 天） |
| 标签 | ≤ 5 个，每个 ≤ 12 字 |

### C2. 业务规则

#### 可见性判定（核心规则！）

```
某用户 U 是否能看到某动态 M？
1. 必要条件：U 当前 GPS 位置 与 M.location 的距离 ≤ M.radius
2. 必要条件：当前时间 在 [M.start_at, M.end_at] 区间内
3. 可见性维度：
   ├── M.visibility = public：满足 1+2 即可见
   ├── M.visibility = friend_drop：U 必须 ∈ M.target_user_ids（好友定向，依赖 D1）
   └── M.visibility = capsule_self：U 必须 = M.author_id（D5 决策）
4. 排除条件（任一即不可见）：
   ├── U 在 M.author 的拉黑列表
   ├── M.author 在 U 的拉黑列表
   └── M.status ∈ {审核中, 已下架, 已删除}
```

#### 发布频率限制（防刷）
- 单用户每日发布上限 20 条（含公开+定向+胶囊总计）
- 同一地点（500m 内）每小时上限 5 条（防止刷地点 spam）
- 视频日上限 5 条

#### 互动规则
- 点赞：1 用户 1 条 1 次，可取消
- 评论：≤ 200 字，发布者可删除任意评论，评论者只能删自己的
- 评论 @ 提及：仅可 @ 互为好友的用户
- 分享：站内可分享到私聊（M6）；站外仅可分享"动态摘要 + 招募链接"，不暴露原图（隐私保护）

#### 路过统计
- 每次满足"距离+时间"条件的浏览，记录一次 visit
- 同一用户对同一动态的多次浏览只计 1 次 unique visit
- 显示"X 人路过看见"，但**不显示具体是谁**（隐私保护）

#### 草稿与编辑
- 草稿本地保存 30 天
- 已发布动态可编辑文字（仅 1 次，编辑后标"已编辑"），不可改地点/媒体/可见性
- 删除立即生效，但触发记录保留 30 天用于风控

### C3. 边界 Case

| 场景 | 处理 |
|------|------|
| 用户在车里飞速移动，瞬间路过动态触发圈 | 按"路过 ≥ 3 秒"才记 visit，避免抖动 |
| 室内 GPS 漂移到隔壁建筑 | 默认半径 200m 已覆盖典型漂移；用户可设 500m |
| 跨日动态（23:55 发，触发时效 24h） | 严格按发布时间戳 + 24h 计算，不按自然日 |
| 用户切到飞行模式继续浏览 | 缓存最近一次拉取的列表，标"离线视图"，不再触发 visit |
| 同地点多人发动态导致地图 marker 重叠 | 聚合气泡（"此地有 5 条新动态"），点击展开列表 |
| 视频未完成上传发布者就关闭 App | 后台续传 + 通知 |
| 暧昧期"投递给好友" 的好友被对方删了 | 投递自动失效，提示发布者"对方已不是你的好友" |
| 投递给好友的内容到了时间窗口但好友未到地点 | 不主动通知（保留惊喜感）；窗口关闭后投递作废 |

### C4. 安全与隐私
- 媒体走 CDN 鉴权，URL 含时效签名（防外链盗链）
- 发布者地理位置显示**精确点位**（用户自己设的）
- 浏览者位置**不上报到服务端**，可见性判定在客户端 + 服务端二次校验（不上报历史轨迹）
- 拉黑：双向不可见（在 M11 设置）

## Step D: 数据模型概要

### 涉及实体

```typescript
Moment {
  id: bigint,
  author_id: bigint,
  type: enum,                  // text_image | video
  text: string,                // 0-1000
  media_urls: string[],        // 图片或视频
  location: { lat: float, lng: float, placeName?: string },
  radius_m: int,               // 50 | 200 | 500
  start_at: timestamp,
  end_at: timestamp | null,    // null = 永久
  visibility: enum,            // public | friend_drop | capsule_self
  target_user_ids: bigint[],   // 仅 friend_drop 用
  campus_id: bigint,           // 自动归属：发布点所在校区
  tags: string[],
  status: enum,                // reviewing | active | rejected | offline | deleted
  is_anonymous: bool,          // 对陌生人是否匿名（好友始终可见身份）
  visit_count: int,            // 路过 unique 数
  like_count: int,
  comment_count: int,
  created_at: timestamp,
  updated_at: timestamp,
}

MomentMedia {
  id: bigint,
  moment_id: bigint,
  type: enum,                  // image | video | thumbnail
  url: string,
  width: int,
  height: int,
  duration_ms?: int,           // 视频用
  order: int,
}

MomentVisit {
  id: bigint,
  moment_id: bigint,
  visitor_id: bigint,
  visited_at: timestamp,
  // 注意：(moment_id, visitor_id) 唯一索引，重复路过不重复记录
}

MomentLike {
  moment_id: bigint,
  user_id: bigint,
  created_at: timestamp,
  // (moment_id, user_id) 唯一
}

MomentComment {
  id: bigint,
  moment_id: bigint,
  author_id: bigint,
  parent_id: bigint | null,    // 二级评论
  text: string,
  status: enum,                // active | deleted | offline
  created_at: timestamp,
}
```

### 涉及接口

```
# 发布
POST   /moments                       发布动态
PATCH  /moments/{id}                  编辑文字（限一次）
DELETE /moments/{id}                  删除
POST   /moments/{id}/republish        二次发布到附近
POST   /moments/upload-media          媒体上传（断点续传）

# 浏览
GET    /moments/nearby?lat=&lng=&r=   附近动态（地图模式）
GET    /moments/feed?sort=            列表 Feed（最新/最热/距离）
GET    /moments/{id}                  详情
POST   /moments/{id}/visit            上报路过（客户端节流）

# 互动
POST   /moments/{id}/like             点赞
DELETE /moments/{id}/like             取消点赞
POST   /moments/{id}/comments         评论
DELETE /moments/{id}/comments/{cid}   删除评论
GET    /moments/{id}/comments         评论列表

# 我的
GET    /me/moments                    我发的动态
GET    /me/moments/visited            我路过看到的（个人足迹，给 M10）
GET    /me/moments/drafts             草稿（云端备份的）
```

## Step E: 交互要点

### 地图首页核心交互
- **底部抽屉**（半屏）：地图为主，下拉抽屉展示列表 Feed，可在地图/列表之间无缝切换
- **地图聚合标记**：相邻动态自动聚合为气泡（"3+"），点击展开
- **底部"+号"浮动按钮**：永远可见，发布入口
- **标记动效**：地图标记用气泡/光晕动效（青春治愈风），刷新时有"撒米"飘落动画
- **刷新**：下拉地图触发刷新（替代列表的下拉）

### 发布交互
- **从下往上的全屏 sheet**：媒体选择 → 编辑 → 选地点 → 设置可见性 → 发布
- **可见性选择是关键卡片**：三个大卡片（公开/给好友/胶囊）+ 图标 + 一句话说明
- **半径选择是图层叠加**：地图上半径圈圈实时显示，让用户直观感受覆盖范围
- **时效选择**：常用预设（永久/24h/7天）+ "自定义"

### 详情页交互
- **顶部沉浸式媒体**（视频/图片）+ 下方信息区
- **路过统计弹幕样**："5 个路人点了赞"飘出文字
- **评论输入**：浮在底部，键盘弹出时上推

### 微动效（D6 青春治愈）
- 点赞：心跳放大 + 粒子飞散
- 路过触发可见：地图上对应动态点位"绽放"动画
- 加载：奶油色波纹 + 小猫/小熊插画

## 模块关联

```
M1 → 依赖：M5 用户认证（必登录）、M9 内容审核（发布走审核）
   → 被依赖：M3 胶囊（共享 Moment 表）、M2 捞网（捞到的人卡片显示其最近动态）
   → 关联：M4 跨校（公开动态进入广场池）、M8 通知（互动提醒）、M11 隐私（拉黑生效点）
```

## 风险标记

| 风险 | 缓解 |
|------|------|
| GPS 精度差 → 触发不准 | 默认 200m + 用户可调；显示"距离 X 米"让用户校准期望 |
| 视频带宽成本 | 限制 60s + 强制转码 + CDN 缓存 + 移动数据下默认低清 |
| 单用户高频路过同一动态被刷量 | unique visit + 合理冷却（24h） |
| 内容数据库随时间膨胀 | 永久内容做冷热分层（活跃近 30 天热表，老内容冷存档） |
