# M12 搜索与发现

> **优先级**：P2 - MVP 弱化（仅最低限）
> **决策依据**：D6 治愈风
> **关联模块**：M5 用户、M4 校园、M1 地点

---

## Step A: 用户故事

### 核心故事
- 作为**用户**，我希望搜索昵称找到具体某位用户
- 作为**用户**，我希望搜索学校（拼音/简称/全名）找到要漫游的校园
- 作为**用户**，我希望搜索地点（POI 或自定义点）用于发动态选位

### 扩展故事（V1.x）
- 作为**用户**，我希望搜索话题/标签查看相关动态
- 作为**用户**，我希望搜索自己的内容
- 作为**用户**，我希望"今日热门"/"猜你喜欢"等发现入口

### 异常故事
- 作为**用户**，搜索无结果 → 友好空状态 + 推荐
- 作为**用户**，搜索敏感词 → 屏蔽返回结果

---

## Step B: 核心流程

### B1. 搜索主线

```
1. 顶部搜索入口（地图首页 + 我的页 + 校园选择页都可触发）
→ 2. 搜索框 + 历史记录 + 搜索建议（拼音输入实时联想）
→ 3. 输入关键词 → 多维结果 Tab：
   ├── 用户：昵称匹配（精确 + 模糊）
   ├── 校园：学校匹配
   └── 地点：POI（基于地图服务）
→ 4. 点击结果 → 跳转详情页
```

### B2. 异常分支

```
├── 搜索无结果      → 空状态插画 + "试试其他关键词" / 推荐校园 / 推荐地点
├── 搜索敏感词      → 不展示结果，提示"请换个关键词"
└── 搜索频繁（防爬虫） → 限流，提示"操作过于频繁"
```

## Step C: 业务规则

### C1. 数据规则
- 关键词长度 1-20 字
- 历史记录本地保存最近 20 条，可清除

### C2. 业务规则

| 类型 | 匹配方式 |
|------|---------|
| 用户昵称 | 完全匹配优先；模糊次之；最多返回 50 |
| 校园 | 拼音/简称/全称都可匹配 |
| 地点 | 接入腾讯/高德 POI |

#### 隐私
- 不能搜到设置了"隐私模式"或"拉黑我"的用户
- 不能搜到敏感词（M9 关键词库联动）

### C3. 边界 Case

| 场景 | 处理 |
|------|------|
| 用户搜索别人 ID/手机号 | MVP 不开放（防搜身份） |
| 大量同名用户 | 按活跃度 / 共同好友 / 同校排序 |
| 校外地点 | 仍可选（发动态时） |

## Step D: 数据模型概要

```typescript
SearchHistory {                 // 本地缓存（仅客户端）
  query: string,
  searched_at: timestamp,
}

// 索引（服务端，搜索引擎如 ES）
UserIndex { user_id, nickname, nickname_pinyin, campus_id, active_score }
SchoolIndex { school_id, name, abbr, pinyin, region }
// 地点走第三方 POI API，不自建
```

### 涉及接口

```
GET    /search/users?q=               搜索用户
GET    /search/schools?q=             搜索校园
GET    /search/places?q=&lat=&lng=    搜索地点（代理 POI 服务）
GET    /search/suggest?q=             联想建议
```

## Step E: 交互要点

- **统一搜索入口**：放大镜图标在主页/设置/校园选择
- **搜索页**：顶部输入框 + 历史 + Tab（用户/校园/地点）
- **结果卡片**：用户头像 + 昵称 + 院系 + 关系标签；校园校徽 + 简介；地点图标 + 地址
- **空状态**：D6 治愈插画 + 推荐内容
- **联想词**：键入即出，最多 5 条

## 风险标记

| 风险 | 缓解 |
|------|------|
| 用户被搜身（手机号/学号） | 不开放敏感字段搜索 |
| 隐私模式用户仍可被搜到 | 索引时尊重隐私设置，过滤掉 |
| 地点 API 成本 | 限流 + 缓存常见 POI |
