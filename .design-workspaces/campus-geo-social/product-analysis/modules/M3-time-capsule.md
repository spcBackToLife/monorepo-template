# M3 时空胶囊系统

> **优先级**：P0 - 产品差异化第三支柱（与"短时定向投递"形成时间维度互补）
> **决策依据**：D5 仅发布者本人到达才能看 → 纯私人时光信件
> **关联模块**：M1 位置动态（共享 Moment 表）、M8 通知（解锁触达）、M10 个人主页（胶囊地图）

---

## Step A: 用户故事

### 核心故事
- 作为**学生**，我希望给"未来的自己"在某地点埋一封信/一段视频，设定 1/3/5/10 年后才能开启，多年后回到这里看到自己当年的样子
- 作为**毕业多年的校友**，我希望回到母校在大三某教室埋下的胶囊，仪式感地"挖出"它
- 作为**用户**，我希望有一张"我的胶囊地图"，看到自己在校园不同地点埋了多少胶囊，哪些已到期可解锁，哪些还在沉睡
- 作为**用户**，我希望胶囊解锁时有仪式感（不是普通动态滑过去），比如"挖宝"动画 + 当年的日期/天气/心情还原

### 扩展故事
- 作为**用户**，我希望埋胶囊时能附加"当时的环境"（系统自动记录天气/温度/学期），让多年后开启时更有时空感
- 作为**用户**，我希望提前设置胶囊的"密码"（一段文字、一首歌名），开启时需输入对答（防止自己开启时已忘了情境）
- 作为**用户**，我希望能给胶囊留"种子时间"（比如至少 6 个月才能解锁），防止冲动开启
- 作为**用户**，我希望删除还未解锁的胶囊（万一后悔了）
- 作为**用户**，我希望被解锁后能选择"分享给好友"（手动分享，不强制公开）

### 异常故事
- 作为**用户**，胶囊到期但我没回到这里 → 不主动提醒，下次到达时通知
- 作为**用户**，胶囊地点已物理消失（教学楼拆了）→ 系统识别后给"在原坐标解锁"特许，并显示"此处建筑已不在"
- 作为**用户**，我换了手机号但要找回老胶囊 → 走 M5 账号找回流程
- 作为**用户**，长达 N 年才能开启的胶囊期间，我希望能查到"它还在那里"作为安全感

## Step B: 核心流程

### B1. 埋胶囊主线

```
1. 入口（地图首页 +号 / "我的"页"埋胶囊"按钮）→ 2. 进入胶囊编辑页（与发布动态独立 UI，更具仪式感）
→ 3. 选/拍内容（图文/视频，与 M1 同规格） 
→ 4. 写"给未来的自己"标题（必填，留作解锁时的钩子）
→ 5. 选择埋藏地点（默认当前位置）
→ 6. 设置解锁条件：
    6.1 最早解锁时间（必填，最短 1 个月，最长 50 年）
    6.2 解锁触发：仅本人到达（D5 默认）
    6.3 解锁半径：50/200/500m（与 M1 同档位）
→ 7. 可选附加：
    7.1 解锁谜题（一段问答，开启前要答对）
    7.2 附加环境快照（系统自动记录天气/学期/拍照时光线方向，作为"时光胶片"）
→ 8. 确认埋藏（仪式动画：胶囊从手中飞下落入地图，"叮"的声音）
→ 9. 加入"我的胶囊"列表（睡眠状态）
```

### B2. 解胶囊主线

```
1. 触发条件满足（用户到达埋藏点 + 当前时间 ≥ unlock_at）
→ 2. 系统检测（客户端 GPS + 后端二次校验）
→ 3. 推送通知"在 [地点] 你有一个尘封的胶囊正在召唤你"（M8）
→ 4. 用户点击通知 → 进入胶囊解锁页
→ 5. （若设了谜题）回答谜题 → 通过则继续，3 次错误则锁 24h
→ 6. 仪式动画（挖土/拆信封/光线绽放）→ 7. 内容呈现
→ 8. 时光对比卡（当年的情境 vs 现在的情境）
→ 9. 用户操作：保存到相册 / 转发给好友 / 重新封存（再次密封埋一次，刷新解锁时间）
```

### B3. 异常分支

```
埋胶囊：
├── 解锁时间 < 1 个月         → 阻断（防冲动）
├── 在校园外埋胶囊            → 警告"建议埋在有意义的地方"，但允许
├── 网络中断                   → 草稿本地保存，恢复后续传
└── 媒体上传失败               → 同 M1

解胶囊：
├── 到点但 GPS 不准（漂移）   → 给"我已到达"按钮（容错），后端二次校验
├── 谜题忘记                   → 提供"放弃此胶囊"选项（永久封存，永不可开），不可强制开启
├── 胶囊地点已不存在（拆迁）  → 系统标记"特许解锁"，仅在原坐标 1km 内即可开
└── 离线状态遇到胶囊点         → 缓存到达事实，联网后补推通知
```

## Step C: 业务规则与边界 Case

### C1. 数据规则

| 字段 | 规则 |
|------|------|
| 标题 | 必填，1-30 字 |
| 内容 | 同 M1 媒体规则，但视频可放宽到 ≤ 90 秒（仪式感） |
| 解锁时间 | 最短 1 个月，最长 50 年；不可选过去时间 |
| 解锁地点 | 必填，GPS 坐标 |
| 解锁半径 | 50/200/500m，默认 200m |
| 谜题 | 可选，问题 ≤ 50 字，答案 ≤ 30 字（不区分大小写匹配） |
| 环境快照 | 系统自动记录：埋藏时的日期/学期（计算）/天气（接 API）/温度 |

### C2. 业务规则

#### 数据归属
- 胶囊本质上是 `Moment.visibility = capsule_self`，但单独一张表 `Capsule` 存放胶囊特有字段（解锁时间、谜题、环境快照），与 Moment 1:1 关联
- 列表/详情查询时不混在普通动态 Feed 中（地图首页不展示自己的胶囊点位，避免剧透）

#### 解锁规则（核心）
```
某胶囊 C 是否可被开启？
1. 用户必须 = C.author（D5 决策）
2. 当前时间 ≥ C.unlock_at
3. 用户 GPS 距 C.location 的距离 ≤ C.unlock_radius_m
4. 若 C 有谜题 → 必须答对
5. C.status = sealed（未被永久放弃）
全部满足才进入"解锁"状态机
```

#### 状态机
```
sealed（封存中） → unlockable（条件已满足，待用户操作） → opened（已开启）
                                                       → abandoned（永久放弃，不可再开）
opened → resealed（用户选择重新封存，回到 sealed 状态，刷新 unlock_at）
```

#### 编辑/删除规则
- **未到解锁时间**：可删除整个胶囊（数据物理删除）；不可编辑内容（保持原汁原味的承诺）
- **已到解锁时间**：不可删除（视为已对未来的自己承诺），但可"永久放弃"（不再可开）
- **已开启**：可分享/重新封存，不可改原内容

#### 上限
- 单用户最多 100 个未解锁胶囊（防滥用）
- 同地点（500m 内）最多 5 个胶囊（避免点位污染）

#### 通知规则（与 M8 联动）
- 用户接近某胶囊点 + 时间已到达 → 推送通知
- 通知冷却：同一胶囊 24h 内不重复推（避免反复经过被骚扰）
- 静默模式：用户可全局关闭"胶囊到期通知"

### C3. 边界 Case

| 场景 | 处理 |
|------|------|
| 用户在 50 年内换了手机号/学号 | 走 M5 账号找回，胶囊跟随账号迁移 |
| 用户毕业多年后母校已合并/改名 | 后台维护学校映射表，地理坐标不变即可解锁 |
| 用户在多年前埋了胶囊后注销账号 | 注销缓冲期内胶囊可恢复；缓冲期结束后胶囊永久消失（与账号绑定） |
| 用户离世 | 不在 MVP 考虑范围；后续可考虑"胶囊托管给指定亲友"功能 |
| 用户多次刷"在地点等到时间到达"刷胶囊 | unique visit + 防刷统计，但本质上是用户开自己的，不影响他人 |
| 解锁通知触达后用户没立刻看 | 胶囊保持 unlockable 状态，下次进 App 仍展示提醒 |
| 用户在飞机/高铁上路过胶囊点（高速移动） | 同 M1：路过 ≥ 3 秒才视为"到达" |

### C4. 隐私与安全
- 胶囊内容加密存储（与普通动态相比，加密层级更高）
- 胶囊不进入跨校广场池（不公开）
- 胶囊不可被举报（因为不公开），但若内容明显违法（如客服介入），管理员可强制销毁
- 胶囊媒体的 CDN URL 在解锁前不生成（防止扫描）

## Step D: 数据模型概要

### 涉及实体

```typescript
Capsule {
  id: bigint,
  moment_id: bigint,           // 关联 Moment（visibility=capsule_self）
  author_id: bigint,
  title: string,
  buried_at: timestamp,
  unlock_at: timestamp,
  unlock_radius_m: int,
  riddle: string | null,       // 谜题问题
  riddle_answer_hash: string | null,  // 谜题答案（哈希存储，不区分大小写时统一处理后哈希）
  riddle_attempts: int,        // 当前已尝试次数
  riddle_locked_until: timestamp | null,
  environment_snapshot: {
    weather: string,
    temperature: float,
    semester: string,          // "2026春"
    sun_direction: string,     // "东南"（拍照时光线推断）
  } | null,
  status: enum,                // sealed | unlockable | opened | abandoned | resealed
  opened_at: timestamp | null,
  open_count: int,             // 重新封存累计次数
}

CapsuleProximityLog {           // 用户接近胶囊的日志（用于通知去重）
  user_id: bigint,
  capsule_id: bigint,
  last_proximity_at: timestamp,
  notified_at: timestamp,
}
```

### 涉及接口

```
POST   /capsules                  埋胶囊
GET    /me/capsules               我的胶囊（列表 + 地图）
GET    /capsules/{id}             胶囊详情（仅作者，含未解锁状态）
DELETE /capsules/{id}             删除（仅未到解锁时间）
POST   /capsules/{id}/answer      回答谜题
POST   /capsules/{id}/open        请求开启（后端做最终校验）
POST   /capsules/{id}/reseal      重新封存
POST   /capsules/{id}/abandon     永久放弃
GET    /capsules/proximity        客户端上报接近事件（节流）
```

## Step E: 交互要点

### 仪式感设计（核心特色）
- **埋胶囊**：动画从用户头像/手部"长出"胶囊，飞向地图坐标，地表浮现一个金色光点（地图上仅本人可见）
- **沉睡中的胶囊**：在"我的胶囊地图"上显示倒计时（"还有 893 天醒来"）+ 微微脉动光晕
- **接近时**：到达点附近触发地图上的"金色光柱"特效 + 触觉反馈（震动）+ 提示音
- **解锁动画**：3 秒"挖土/拆封"动画 + 加载内容 + 时光对比卡浮现（埋藏日期/当时的天气/学期 ↔ 现在）
- **谜题界面**：复古打字机字体 + 暗角光照，营造"翻阅老物件"氛围

### 我的胶囊地图（M10 紧关联）
- 单独 Tab：地图视图 + 列表视图
- 地图上区分：金色光点（已可解锁）、银色光点（沉睡中带倒计时）、灰色（已开启）、黑色（已放弃）
- 列表按解锁时间排序

### 触觉与声音
- 埋下：轻柔的"叮"+ 短震动
- 解锁：温暖音效（类似翻书声）+ 长震动
- 谜题答错：低沉震动

### 与 M1 发布的视觉差异
- M1 发动态：清新爽朗，主色奶油黄
- M3 埋胶囊：偏暖色 + 棕褐色调（复古信封感），即使整体风格青春治愈，胶囊场景"温度更高"

```
M3 → 依赖：M1（共享 Moment 表）、M5（账户体系）、M8（通知触达）
   → 被依赖：M10 个人主页（"我的胶囊"是核心展示）

```

## 风险标记

| 风险 | 缓解 |
|------|------|
| 用户埋了胶囊后 N 年都不用 App，账号失效 | 长期休眠账号特殊保护策略：注销前必须确认所有胶囊处理；可选"封存账号但保留胶囊"模式（V1.x） |
| 解锁时间过长（50 年）数据库压力 | 冷热分层：未到解锁前 1 年的胶囊存冷库，临近解锁 1 年迁回热库 |
| 物理位置消失（拆迁） | 维护"特许解锁"白名单（地点-坐标映射库），定期人工审核 |
| 谜题死锁 | 必须给"永久放弃"出口（不能强制让用户解，因为答案在用户脑中） |
