# M4 跨校漫游系统

> **优先级**：P0 - 用户明确诉求的差异化第四支柱
> **决策依据**：D4 只看广场聚合（不看具体用户、不可互动）
> **关联模块**：M1 位置动态（公开动态进入广场池）、M5 校园字典、M9 审核

---

## Step A: 用户故事

### 核心故事
- 作为**学生**，我希望好奇外校生活时，能"切到其他校园"看看那里发生什么
- 作为**保研/考研用户**，我希望了解目标学校的校园氛围、生活节奏，作为决策参考
- 作为**毕业的校友**，我希望切回母校的广场，看看现在校园里学生在分享什么
- 作为**新生**，我希望开学前预热，看看自己将要去的学校的广场动态

### 扩展故事
- 作为**用户**，我希望"附近其他校园"列表（按地理位置推荐邻校）
- 作为**用户**，我希望"全国热门校园 Top 10"（看哪里最热闹）
- 作为**用户**，我希望按"高校类别"筛选（985/211/双一流/其他）
- 作为**用户**，我希望收藏几个我关注的校园，下次直接进入
- 作为**用户**，我希望看到外校的"标志性地点"（樱花大道/未名湖/大草坪等），增强代入感

### 异常故事
- 作为**用户**，我希望切到外校后看到的内容**清晰标注"非互动"**，避免我习惯性点赞失败困惑
- 作为**用户**，访问的外校无内容/审核中 → 友好的空状态，不让用户误以为产品坏了
- 作为**用户**，外校广场内容触发我兴趣，希望能"加入这个校园"（前提是认证身份允许）
- 作为**用户**，意外切到了不存在的校园ID → 路由保护，回退到主校园

## Step B: 核心流程

### B1. 切换校园主线

```
1. 主页地图顶部 → 校园切换按钮（默认显示主校园名）
→ 2. 进入"校园选择"页：
    2.1 我的主校园（高亮）
    2.2 我收藏的校园（次级）
    2.3 推荐：附近邻校（按地理） + 热门校园 Top 10 + 我搜索过的
    2.4 搜索框（支持拼音/简称/全名）
→ 3. 选择目标校园 → 进入"外校漫游模式"
→ 4. 主页顶部 banner 标"漫游中：清华大学广场" + 退出按钮
→ 5. 内容区切换为该校"广场 Feed"（与本校地图首页**视觉风格不同**，加深"我在别的地方"的感知）
```

### B2. 外校广场浏览主线

```
广场 Feed 模式（不再有地图）：
1. 顶部：校园介绍卡片（校名/校徽/简介/学生数/标志性地点轮播图）
→ 2. Tab 切换：热门 / 最新 / 标志地点
→ 3. 卡片流：每张卡片 = 一条公开动态（图/视频/文字 + 地点 + 路过统计）
→ 4. 点击卡片 → 详情页
   4.1 详情页与本校相同布局，但**所有互动按钮置灰**（点赞/评论/收藏/分享）
   4.2 顶部明显标"外校内容，仅可浏览"
→ 5. 退出漫游 → 切回主校园（主页地图模式恢复）
```

### B3. 异常分支

```
├── 切换的校园不存在或被删除   → 回退主校园 + 提示
├── 外校无任何公开动态          → 空状态："这个校园还很安静呢" + 推荐其他热闹校园
├── 网络异常                    → 缓存上次浏览结果 + Banner 提示
├── 试图在外校漫游时使用本校功能（如开网/发布）→ 弹窗"请先回到你的主校园"
└── 外校内容被审核下架但用户已加载  → 卡片置灰"该内容已不可见"
```

## Step C: 业务规则与边界 Case

### C1. 内容池规则（核心）

```
某条 Moment M 是否进入校园 X 的"广场池"？
1. M.visibility = public（公开）
2. M.status = active（已通过审核）
3. M.campus_id = X（按发布点位归属校区）
4. M.author 未勾选"不进广场"开关（用户级隐私）
5. M.author 不在校园拉黑名单（管理员级运营操作，极少触发）

广场池内容**不展示具体用户身份**：
- 不显示发布者头像/昵称/院系
- 仅显示："来自此校园的某位同学" + 性别/年级（用户隐私设置允许时）
- 不显示评论列表（评论本身可能含身份信息）
```

### C2. 互动限制（D4 决策）

| 行为 | 本校 | 外校漫游 |
|------|------|---------|
| 浏览 | ✅ | ✅ |
| 点赞 | ✅ | ❌ |
| 评论 | ✅ | ❌ |
| 收藏 | ✅ | ❌ |
| 分享 | ✅（站内/站外） | ❌ |
| 路过统计 | ✅（计入 visit） | ❌（不计入；外校漫游不算到达点位） |
| 举报 | ✅ | ✅（保留举报通道，防违规） |
| 看具体用户 | ✅ | ❌ |
| 加好友 | ✅ | ❌（看不到具体用户，自然无法） |
| 私聊/捞网 | ✅ | ❌ |

### C3. 业务规则

#### 校园字典与归属
- 一所大学一个 School，下挂多个 Campus（如清华有主校区/深研院）
- Moment 按发布时 GPS 自动归属 Campus（命中 Campus.geo_polygon 即归属）
- 未命中任何校区的（校外）→ 不进任何校园广场，但仍是"全网公开"（V1.x "全网广场"用，MVP 不开放）

#### 排序与推荐
- "热门" Tab：按 (likes\*3 + visits + comments) / age_decay 计算热度，每 5 分钟刷新
- "最新" Tab：纯时间倒序
- "标志地点" Tab：按学校预设的 N 个标志点位（运营维护）+ 该点位下最近的动态

#### 退出漫游
- 用户主动点"返回主校园"
- 杀进程 → 下次启动默认回主校园
- 切换主校园（M5）→ 自动切回新主校园

#### 用户隐私开关
- 设置中可开"不让我的公开动态进入跨校广场"（默认开放，用户可关）
- 即便是公开动态，未勾选广场分发的不进 M4

### C4. 边界 Case

| 场景 | 处理 |
|------|------|
| 用户在 A 校发动态时同时身在 A 校与 B 校交界 | 按 GPS 命中的 polygon 决定，单一归属 |
| 校园合并 / 改名 | 后台维护映射，旧名搜索仍可命中 |
| 用户切换主校园后想看老校园 | 漫游回去即可（保留浏览权） |
| 跨校广场内容举报 | M9 介入，处理后双校都隐藏（运营单点处理） |
| 大量爬虫/外部用户访问广场 | 跨校内容也走鉴权 token，未登录无法访问 |
| 标志地点维护成本高 | MVP 先按系统自动算"近 30 天最多动态聚集的 5 个点"，运营可覆盖 |

### C5. 反窥探边界
- 跨校漫游本质是"看公开广告位"，不是"探索他人生活"
- 所以**不做地图模式**（地图模式会精细暴露其他校园用户的生活轨迹）
- 仅 Feed 流（按时间或热度的卡片流）
- 不暴露任何关于"哪些用户在哪个时间在哪里"的信息

## Step D: 数据模型概要

### 涉及实体

```typescript
// 复用 M5 的 School / Campus 定义

CampusPlazaCache {              // 校园广场聚合缓存（提升性能）
  campus_id: bigint,
  hot_moment_ids: bigint[],     // 热门动态 ID 列表（每 5 分钟刷新）
  latest_moment_ids: bigint[],  // 最新 N 条
  landmark_locations: [
    { name: string, geo: { lat, lng }, recent_moment_ids: bigint[] }
  ],
  updated_at: timestamp,
}

CampusFavorite {                // 用户收藏的校园
  user_id: bigint,
  campus_id: bigint,
  added_at: timestamp,
}

CampusBrowseHistory {           // 漫游历史（用于推荐）
  user_id: bigint,
  campus_id: bigint,
  last_visited_at: timestamp,
  visit_count: int,
}

UserPlazaPrefs {                // 用户广场偏好
  user_id: bigint,
  share_to_plaza: bool,         // 是否同意公开动态进入跨校广场
}
```

### 涉及接口

```
GET    /campuses/search?q=               校园搜索
GET    /campuses/recommended             推荐校园（地理 + 热度）
GET    /campuses/{id}                    校园详情卡片
GET    /campuses/{id}/plaza?tab=hot|new  广场 Feed
GET    /campuses/{id}/landmarks          标志地点列表

POST   /me/campuses/favorite             收藏校园
DELETE /me/campuses/favorite/{id}        取消收藏
GET    /me/campuses/history              我的漫游历史

PATCH  /me/plaza-prefs                   更新广场偏好
POST   /campuses/{id}/apply              申请加入校园（V1.x，MVP 不做）
```

## Step E: 交互要点

### 切换校园 Picker
- **底部 sheet**：从底部弹起，包含搜索 + 列表
- **列表分区**：我的主校园 / 收藏 / 附近 / 热门 / 历史浏览
- **每个校园卡片**：校徽 + 名字 + 学生数（虚化展示）+ "今日 X 条" 活跃标签

### 漫游模式视觉差异（关键，让用户清楚知道在外校）
- **顶部漫游 Banner**：浅紫渐变 + "漫游中：xxx 大学" + 一个回家按钮
- **详情页**：所有互动按钮置灰 + tooltip"漫游模式不可互动"
- **Feed 卡片样式略变**：边框颜色更冷调（与本校的暖奶油色形成区分）

### 校园介绍卡片（情绪化）
- 顶部 hero：学校代表性照片（樱花/红墙/校门）轮播
- 简介："建于 1898 年"等基本信息
- 趣味标签："以樱花闻名" / "凌晨 4 点的图书馆" 等运营文案
- 不展示具体用户/数据（统计仅虚化"非常活跃"等）

### 标志地点 Tab
- 卡片化展示 5-8 个地点
- 每个地点显示一张代表照片（可能就是某条公开动态的封面）
- 点击进入该地点的动态合集

### 性能优化
- 跨校广场内容 CDN 缓存 5 分钟
- 列表分页 + 虚拟滚动
- 媒体懒加载

```

   → 被依赖：- 几乎不被其他模块依赖（漫游是叶子功能）

```

## 风险标记

| 风险 | 缓解 |
|------|------|
| 跨校内容被外部抓取造成隐私泄露 | 接口鉴权 + 限流 + 未登录不可访问 |
| "外校观察"演变成窥探 | D4 决策已做严格隔离：不展示具体用户、不可互动、无地图模式 |
| 校园数据维护成本高（学校信息/边界/标志地点） | MVP 阶段仅维护中国大陆全日制本科/硕博院校（约 2000 余所），且边界用主校区中心半径 1km 兜底 |
| 部分校园冷清造成空感 | 推荐机制偏向"热门校园"，避免用户切到"鬼城"；空状态友好引导 |
| 公开动态进广场是否需要二次确认 | 在发布动态时增加一行"允许进入跨校广场（默认勾选）"，把决定权给用户 |
