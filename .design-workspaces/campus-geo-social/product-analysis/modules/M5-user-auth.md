# M5 用户与认证体系

> **优先级**：P0（一切上层模块的依赖）
> **决策依据**：D2 强认证（学信网/学生证 OCR + 人脸核身）
> **关联模块**：被全部模块依赖；与 M9 内容审核（举报/封禁通道）、M11 隐私设置（注销/设备管理）紧耦合

---

## Step A: 用户故事

### 核心故事
- 作为**新用户**，我希望用手机号快速进入注册，以便不用一上来就填一堆资料
- 作为**新用户**，我希望明确知道为什么要做强认证（学信网/人脸），以便理解门槛、愿意配合
- 作为**新用户**，我希望强认证有"为什么需要"的说明、隐私政策入口，以便信任 App 后再交出信息
- 作为**已注册老用户**，我希望用密码或免密登录快速进入，以便日常使用顺滑
- 作为**忘记密码的用户**，我希望通过手机验证码重置密码
- 作为**毕业的用户**，我希望即使学籍失效也能继续使用（看胶囊/老朋友），但能力可降级（不能再发新动态到本校）

### 扩展故事
- 作为**老用户**，我希望管理已登录设备，能远程下线某台
- 作为**老用户**，我希望能注销账号（合规要求），且有缓冲期可撤回
- 作为**保研/换校用户**，我希望切换主校园身份（重新走认证）
- 作为**双重身份用户**（如本科+硕士同校但不同学院），我希望更新院系/年级信息
- 作为**家长/监护人**，如果用户是未成年，我希望能限制部分功能（理论上大学生≥18，本期不做）

### 异常故事
- 作为**学信网验证失败的用户**，我希望知道具体失败原因（信息不一致 vs 系统故障）和重试路径
- 作为**人脸核身失败的用户**，我希望失败 N 次后能转人工审核
- 作为**已注册手机号但忘记的用户**，我希望验证码登录时被识别后引导（"此号码已注册"）
- 作为**正在使用 App 时被异地登录踢出的用户**，我希望立刻收到通知和"是否本人"的确认
- 作为**学校不在系统列表中的用户**（小众/海外/合并院校），我希望能提交申诉

## Step B: 核心流程

### B1. 注册主线（Happy Path）

```
1. 启动页 → 2. 选择"注册" → 3. 阅读并同意《用户协议》《隐私政策》
→ 4. 输入手机号 → 5. 获取验证码 → 6. 输入验证码 → 7. 设置密码（可跳过用免密）
→ 8. 进入【强认证】子流程：
    8.1 选择学校（带搜索） → 8.2 上传学生证照片（OCR 识别姓名/学号/院系）
    → 8.3 跳转学信网 OAuth 授权（拉取在校状态） → 8.4 人脸核身（活体检测）
    → 8.5 提交审核（自动通过 OR 转人工）
→ 9. 完善基础资料（昵称/头像/性别/年级，可跳过）→ 10. 进入新手引导 → 11. 进入主页
```

### B2. 注册流程异常分支

```
├── Step 4 手机号格式错误      → inline 即时提示
├── Step 4 手机号已注册        → 跳转登录页 + 提示"此号码已注册，是否登录？"
├── Step 5 60s 内重复获取      → 倒计时禁用按钮
├── Step 5 短信通道异常        → 备选语音验证码（电话拨入读 6 位数字）
├── Step 6 验证码错误 ≥3 次    → 锁定 1 分钟，需重新获取
├── Step 6 验证码过期(5min)    → 提示过期，重新获取
├── Step 8.1 学校列表无所需校  → "找不到我的学校" 入口 → 提交申诉表（人工录入）
├── Step 8.2 OCR 识别失败      → 允许手动输入 + 后台人工审核
├── Step 8.3 学信网授权拒绝    → 降级路径：仅学生证 + 人工审核（48h 内反馈）
├── Step 8.3 学信网信息不匹配  → 提示具体差异点（如"姓名与身份证不符"），引导修改/申诉
├── Step 8.4 人脸活体失败 ≥3 次→ 转人工审核（上传手持证件视频）
├── Step 8.5 审核被拒          → 状态卡片展示拒绝原因 + 申诉入口
├── Step 9 昵称重复            → 加随机后缀建议
└── 任何步骤中断退出           → 注册进度本地保存 30 分钟内可恢复（仅当前设备）
```

### B3. 登录流程

```
主线（密码登录）：
启动页 → 输入手机号 → 输入密码 → 登录成功 → 进入主页

主线（验证码免密登录）：
启动页 → 输入手机号 → 获取验证码 → 输入验证码 → 登录成功

异常：
├── 密码错误 ≥5 次 → 账号锁定 30 分钟 + 强制改密
├── 异地登录       → 主登录设备收到通知 + 二次验证（旧设备点确认 / 人脸核身）
├── 账号已被封禁   → 展示封禁原因 + 申诉入口
├── 账号注销中     → "您正在注销期内，是否撤回？"
└── 学籍已失效     → 仍可登录但能力降级（提示 + 引导毕业生模式）
```

### B4. 找回密码流程

```
忘记密码 → 输入手机号 → 验证码校验 → 设置新密码 → 自动登录
异常：
├── 手机号未注册   → 提示并引导去注册
└── 24h 内重置 ≥3 次 → 触发风控，需走人工申诉
```

### B5. 注销流程（合规必备）

```
设置 → 账号安全 → 注销账号 → 警告说明 → 确认手机验证码
→ 进入注销缓冲期（15 天）→ 期间登录可一键撤回 → 缓冲期结束彻底清理
异常：
├── 账号有未结订单/未消耗钻石 → 阻断注销，引导处理
└── 缓冲期被举报/封禁          → 立刻终止注销流程，进入封禁状态
```

### B6. 学籍续期与降级

```
毕业生模式触发：
- 学信网定期复核（每学期/每年）→ 状态变为"已毕业"
- 用户主动声明已毕业

毕业生能力变化：
- 保留：登录 / 看自己胶囊 / 看老朋友动态 / 跨校广场 / 私聊
- 受限：不能在本校发新动态 / 不能开捞网捞当前在校生（避免社会人骚扰）
- 升级：可"激活校友模式"，进入校友专属内容池（V1.x 规划，MVP 不做）
```

## Step C: 业务规则与边界 Case

### C1. 数据规则

| 字段 | 规则 |
|------|------|
| 手机号 | 11 位，仅支持中国大陆 +86，唯一性 |
| 密码 | 8-20 位，至少含字母+数字（不强制特殊字符以降低门槛） |
| 验证码 | 6 位数字，5 分钟有效，60 秒重发间隔，单号每日上限 10 次 |
| 昵称 | 2-16 字符（中英数字混合），不含特殊符号，**全局唯一** |
| 真实姓名 | 来自学信网/学生证 OCR，**不可手动改**，仅展示给好友（可选） |
| 学号 | 来自学信网，唯一约束（同一学校同学号即同一人） |
| 院系/年级 | 来自学信网，每学期可手动确认更新 |
| 头像 | jpg/png，≤5MB，自动审核（M9 接入） |

### C2. 业务规则

- **唯一性**：手机号唯一、学号+学校唯一、昵称唯一
- **新用户引导**：注册成功后强制走 3 步引导（首次发动态 / 首次开捞网 / 首次完善头像），完成后送 50 积分（与 M7 联动）
- **认证状态机**：`pending → reviewing → approved | rejected | appealing`，状态变化触发通知（M8 联动）
- **设备管理**：单账号最多同时登录 3 台设备，新设备登录会顶替最老的
- **风控与限流**：
  - 同 IP 每分钟最多 5 次注册请求
  - 同设备指纹每天最多注册 1 个账号
  - 短信通道前置图形验证码（频繁请求时触发）
- **学籍复核**：每学年开学时被动检查学信网状态，状态变化触发通知

### C3. 安全规则

- 密码 bcrypt 存储，盐值 ≥12 轮
- 接口全部 HTTPS，敏感接口（支付/认证）需双重校验
- JWT token 有效期 7 天，refresh token 30 天
- 所有学信网/人脸数据**加密存储**且不可被普通运营访问，仅限合规审计场景脱敏可见
- 用户头像/学生证照片走 CDN 鉴权 URL，外链失效

### C4. 隐私边界（强相关于 M11）

- 真实姓名默认**不展示**，用户可选择"对好友可见"
- 学号、身份证号、人脸照片**永不展示**
- 院系/年级在好友间默认可见，对陌生人默认隐藏（用户可调）
- 注销后 15 天保留期内不暴露任何数据，期满后**完全删除**（合规：保留法律必要的最小日志）

### C5. 边界 Case

| 场景 | 处理方式 |
|------|---------|
| 用户在 2 台设备同时注册同手机号 | 先完成的成功，后者收到"已注册"提示 |
| 双胞胎走人脸核身相互通过 | 学信网+学号唯一性可拦截（同一身份证只能注册 1 个） |
| 学校改名/合并 | 后台维护学校字典，自动迁移历史用户的学校字段 |
| 用户上传他人学生证 | OCR + 学信网交叉验证可拦截，无法通过 |
| 学信网接口故障 | 走人工审核降级，48h 内反馈，期间用户可登录但能力受限（不能发动态） |
| 海外/港澳台学生 | V1.x 规划，MVP 仅支持中国大陆全日制本科/硕博 |
| 用户毕业后又考研同校 | 登录后引导"是否更新学籍"，重走学信网授权 |

## Step D: 数据模型概要

### 涉及实体

```typescript
// 主用户表
User {
  id: bigint,                  // 主键
  phone: string,               // 唯一
  password_hash: string,       // bcrypt
  status: enum,                // active | locked | suspended | logging_off | banned
  created_at: timestamp,
  last_login_at: timestamp,
  device_id_set: string[],     // 当前登录设备列表（最多3）
}

// 用户资料
UserProfile {
  user_id: bigint,             // 主键 = User.id
  nickname: string,            // 唯一
  avatar_url: string,
  gender: enum,                // unknown | male | female
  bio: string,                 // 简介，<= 100 字
  primary_campus_id: bigint,   // 主校园
}

// 学籍认证（敏感数据，独立表 + 加密）
Identity {
  user_id: bigint,             // 主键
  real_name_enc: string,       // 加密存储
  id_card_enc: string,         // 加密存储
  student_no: string,
  school_id: bigint,
  department: string,
  grade: int,                  // 入学年份
  enrollment_status: enum,     // enrolled | graduated | suspended
  verification_status: enum,   // pending | reviewing | approved | rejected | appealing
  verification_method: enum,   // xuexin_oauth | manual_ocr
  verified_at: timestamp,
  rejected_reason: string,
  next_review_at: timestamp,   // 下次自动复核时间
}

// 验证码
VerifyCode {
  phone: string,
  code: string,
  scene: enum,                 // register | login | reset_password | logout
  expires_at: timestamp,
  used: bool,
  attempts: int,
}

// 学校字典
School {
  id: bigint,
  name: string,
  abbr: string,                // 简称（北大/清华）
  campus_list: Campus[],       // 校区
  region: string,              // 省/市
  status: enum,                // active | merged | renamed
}

// 校区（一所学校多个校区）
Campus {
  id: bigint,
  school_id: bigint,
  name: string,                // 燕园/未名/西校区
  geo_polygon: GeoJSON,        // 地理边界（M1 用于"是否在本校"判定）
  center: { lat: float, lng: float },
}

// 注销申请
LogoutRequest {
  user_id: bigint,
  applied_at: timestamp,
  scheduled_clean_at: timestamp,  // 申请时间 + 15 天
  status: enum,                   // pending | revoked | completed
}
```

### 涉及接口（粗粒度）

```
# 注册/登录
POST  /auth/send-code              发送验证码
POST  /auth/register               提交注册基本信息
POST  /auth/login-password         密码登录
POST  /auth/login-code             验证码免密登录
POST  /auth/refresh-token          刷新 token
POST  /auth/logout                 登出

# 强认证子流程
POST  /auth/verify/upload-id-card  上传学生证（OCR）
GET   /auth/verify/xuexin-oauth    跳转学信网 OAuth
POST  /auth/verify/face            人脸活体核身
GET   /auth/verify/status          查询当前认证状态
POST  /auth/verify/appeal          提交申诉

# 资料管理
GET   /user/profile                获取个人资料
PATCH /user/profile                更新资料（受限字段）
POST  /user/avatar                 上传头像

# 学校字典
GET   /schools/search?q=           学校搜索（拼音/简称/全称）
POST  /schools/apply               申请录入学校

# 设备管理
GET   /user/devices                设备列表
DELETE /user/devices/{id}          下线设备

# 账号生命周期
POST  /user/forgot-password        忘记密码
POST  /user/logout-account         注销账号
POST  /user/logout-account/cancel  撤回注销
```

## Step E: 交互要点（给 interaction-designer 的输入）

- **强认证流程必须有"为什么需要"的解释卡片**（在每一步进入前展示），降低用户疑虑
- **进度条始终可见**（注册总共几步、当前第几步），减少未知焦虑
- **每一步都可"暂存退出"**（除人脸核身外，因为人脸需要实时性）
- **审核中状态有明确的"等待页"**（动画 + 预计时长 + "去探索一下" CTA 不让用户干等）
- **失败要有可执行的下一步**（不是只显示"失败请重试"）
- **学校选择器**：拼音搜索 + 历史使用 + 热门推荐 + "找不到我的学校"兜底
- **手机号输入框**：自动 +86 前缀，11 位自动校验
- **验证码输入框**：6 个独立框 + 自动跳格 + 短信预填权限（iOS 14+ 一键填充）
- **密码强度可视化**：弱/中/强进度条
- **人脸核身卡片**：实时镜头预览 + 文字提示（眨眼/张嘴）+ 模糊光线提醒
- **错误提示**：inline（不打断流程），关键错误用 sheet 展示且需用户主动关闭
- **风格基调**（D6 青春治愈）：错误状态用拟人化插画（"哎呀"小人）+ 鼓励性文案

## 风险标记

| 风险 | 缓解 |
|------|------|
| 强认证导致注册转化率低（业内强认证流失约 40-60%） | 注册到学籍认证之间允许"游客试看"3 分钟 → 看到精彩内容后再走认证 |
| 学信网接口故障率不可控 | 必须做好"人工审核降级路径"，且在 SLA 中明确响应时长 |
| 用户因为隐私担忧拒绝授权 | 隐私政策卡片化展示（不要堆 10 屏小字），关键条款单独高亮 |
