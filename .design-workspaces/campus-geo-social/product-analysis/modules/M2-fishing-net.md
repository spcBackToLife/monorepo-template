# M2 捞网交友系统

> **优先级**：P0 - 产品差异化第二支柱，社交破冰核心
> **决策依据**：D1（投递需好友 → 捞网是结识好友的主路径）、D3（混合经济：任务+付费）、D6（青春治愈视觉）
> **关联模块**：M1 位置动态（捞到的卡片展示其最近公开动态）、M7 经济（消耗道具）、M6 关系链（捞到→打招呼→好友）、M9 审核（卡片内容）

---

## Step A: 用户故事

### 核心故事
- 作为**新生**，我希望在宿舍楼下"撒一网"，看到同楼的同学卡片，发现可能认识的朋友
- 作为**学生**，我希望在食堂等地点开网，看到附近活跃过的有意思的人
- 作为**用户**，我希望网捞的过程有趣味性（像开盲盒），不是机械的"附近的人列表"
- 作为**用户**，我希望对捞到的人卡片可以选择"打招呼/无视/拒收同类"
- 作为**被捞到的人**，我希望自己被捞到时不会被打扰（保留神秘感），但收到招呼时能看到对方的简介后再决定接不接
- 作为**用户**，我希望使用稀缺道具（炸弹一次捞 5 人、补网捞同一片再来一次）来增加趣味与"押注感"

### 扩展故事
- 作为**用户**，我希望对捞到的人 1 秒钟内决定，不行就"再撒一网"
- 作为**用户**，我希望看到对方最近的一条公开动态预览，作为破冰话题
- 作为**用户**，我希望在不同场所（宿舍楼/操场/图书馆）开网得到不同氛围的人群（系统按场所标签优先排序）
- 作为**用户**，我希望对方接受我的招呼后，自然进入私聊（M6）
- 作为**保守用户**，我希望关闭"被捞到"功能，让自己不进入任何人的网（隐私模式）
- 作为**有恋人/不希望被异性骚扰的用户**，我希望可设置仅同性可捞到（性别筛选 - **慎重，见下方风险**）

### 异常故事
- 作为**用户**，开网时附近没人 → 不能撒一个空网就扣道具，应"空网保护"返还或赠送其它（提升体验）
- 作为**用户**，捞到的人卡片信息违规（头像/简介脏话）→ 一键举报 + 系统秒级下架
- 作为**用户**，频繁被同一陌生人捞 → 提示"已多次被某用户捞到"+ 一键拉黑
- 作为**用户**，同一地点反复开网捞重复 → 系统冷却"24h 内不重复捞到同一人"
- 作为**捞到了不喜欢的人的用户**，我希望"无视"后此人未来 30 天不再被我捞到
- 作为**被举报方**，如果误举报，我希望有申诉

## Step B: 核心流程

### B1. 开网主线

```
1. 主页 → 选择"开网"入口（地图主屏快捷入口/Tab 栏）
→ 2. 进入"开网"页：
    2.1 显示当前位置（地图 + 当前撒网半径预览圆）
    2.2 选择网类型（普通网/超大网/炸弹/补网，对应不同道具消耗）
    2.3 显示"此处约有 X 人活跃过"作为预期
→ 3. 确认开网（消耗道具 - 与 M7 联动）
→ 4. 撒网动画（青春治愈：渔网从空中撒下 + 水波纹）2-3 秒
→ 5. 揭晓结果：
    5.1 普通网：捞出 1 人 / 空网（30% 空网率，但有保护机制）
    5.2 超大网：捞出 1 人，但偏远
    5.3 炸弹：捞出 3-5 人（不同付费档）
    5.4 补网：再开一网（同一地点 1h 内 50% 折扣）
→ 6. 卡片展示：上下滑动浏览每张卡片
→ 7. 卡片操作：
    ├── 打招呼：消耗一句"破冰话术"或自定义留言
    ├── 收藏：保留这张卡片到"我的捞捕"，稍后再决定
    ├── 无视：30 天不再被你捞到
    └── 跳过：本次捞到结束，不影响未来
→ 8. 进入"等待回应"状态：对方收到招呼后才知道你的存在，对方选择回复/拒绝/加好友
```

### B2. 被捞到 → 收到招呼流程

```
1. 用户 A 捞到用户 B → 数据落库（B 不收到任何通知，纯静默）
→ 2. A 选择"打招呼"，附带话术 → 推送通知给 B（M8）
→ 3. B 收到通知"有人在 [地点] 捞到你了，并向你打招呼"
→ 4. B 进入"收到的招呼"页 → 看到 A 的卡片 + 留言
→ 5. B 三选一：
    ├── 回复："你好！" → 自动建立"临时聊天"会话（M6）
    ├── 加为好友 → 走好友请求流程（M6）
    └── 拒绝 → A 收到"对方暂未回复"（不告知拒绝详情，保护双方）
→ 6. B 也可"举报" → M9 介入
```

### B3. 异常分支

```
开网：
├── 余额不足（道具不够）         → 弹"购买道具"sheet（接 M7）
├── 当前位置 GPS 信号差          → 警告"定位不准，可能影响结果"，可继续或取消
├── 当前位置无人活跃            → 撒网动画照走，但揭晓时显示"空网"（视为捞过）
│   └─ 空网保护：返还 50% 道具 / 第 N 次空网赠送"补网"道具
├── 已开过网刚结束              → 1 分钟冷却（防止机械连开）
└── 被系统识别为刷量行为        → 临时禁用 24h，需走风控申诉

被捞处理：
├── 设置了"隐私模式"（不可被捞）  → 不会出现在任何人捞到的卡片中
├── 设置了"仅同性可捞"（V1.x 特性，MVP 待定）
├── 收到 ≥ 5 条未读招呼          → 仅 push 一次"你今天收到了多个招呼"，不轰炸
└── 对方主动撤回招呼              → 招呼消失，B 看不到（A 1h 内可撤回）
```

## Step C: 业务规则与边界 Case

### C1. 候选用户筛选规则（核心算法）

```
被捞到的人 B 必须同时满足：
1. B 与 A 不互为拉黑
2. B.fishing_visibility = visible（B 没开隐私模式）
3. B 在过去 14 天内 在距离 A.location ≤ R 米内出现过（活跃过 / 发过动态 / 接受过招呼）
4. B ≠ A（不能捞自己）
5. (A, B) 在过去 30 天内未出现过"无视"关系（B 之前被 A 无视过）
6. (A, B) 在过去 24h 内未被 A 捞到过（去重）
7. 若 A 设置了"仅同校"过滤 → B.primary_campus = A.primary_campus
8. （V1.x）若 A 设置了性别偏好 → B 性别符合
```

#### 撒网半径与候选池
- 普通网：500m 半径
- 超大网：2km 半径
- 炸弹：1km 半径，多人
- 补网：跟随原网的半径

#### 打分排序（候选池过大时）
- 同校园 +10 / 同院系 +5 / 同年级 +3
- 最近 24h 活跃 +8 / 最近 7 天 +4
- 共同好友数加权（若无则 0）
- 内容互动相关性（A 路过过 B 的动态 +3）
- 随机扰动 ±2（避免完全相同结果）

### C2. 卡片展示内容

每张捞到的卡片包含：
- 头像（必显）
- 昵称（必显）
- 简介一句话（用户自填，可空）
- 性别 / 年级 / 院系（用户隐私设置决定是否展示）
- 最近一条公开动态预览（图片缩略 + 摘要 50 字）
- 被多少人路过的"足迹值"（虚化指标，类似活跃度）
- 共同好友数（≥1 时展示，≤3 不展示具体名单）
- "打招呼" / "无视" / "举报"按钮

### C3. 业务规则

#### 道具消耗（与 M7 联动）
| 操作 | 消耗 |
|------|------|
| 普通网 | 1 个"网" 道具（积分可换 / 钻石可买 / 任务可得） |
| 超大网 | 3 个"网" |
| 炸弹（小） | 5 个"网" + 1 个炸弹（仅钻石购买） |
| 炸弹（大） | 10 个"网" + 1 个大炸弹（仅钻石） |
| 补网 | 1 个"补网"道具（50% 概率任务获得） |
| 打招呼 | 1 个"招呼" 道具（每日免费 3 次，超出消耗钻石） |

#### 频率限制
- 单用户单日开网上限 50 次（含失败/空网，防刷量）
- 单用户单日打招呼上限 30 次
- 同一目标 30 天内最多被同一发起人主动招呼 3 次
- 收到的招呼 7 天内未回复自动归档"过期"

#### 反骚扰规则
- B 拒绝 A 的招呼后，A 在 30 天内不可向 B 再次招呼（但仍可能再次捞到，仅不能联系）
- B "无视" A 的卡片 → A 接下来 30 天不能再捞到 B
- 单日被举报 ≥ 3 次的卡片 → 系统自动隐藏，进入人工审核（M9）

#### 性别筛选政策（敏感，需明确立场）
- **MVP 不上线**性别筛选功能（避免平台被误用为"附近异性 dating"工具）
- V1.x 可上线"屏蔽某性别捞到我"（被动保护），但不允许"主动只捞某性别"（防恶意筛选）
- 这条对产品调性影响大，**需用户在 Phase 2 评审时再次确认**

### C4. 边界 Case

| 场景 | 处理 |
|------|------|
| 用户在校园外（如外地）开网 | 允许，但提示"此处用户密度低" |
| 凌晨 2-6 点开网 | 允许但部分校园人少；无强制限制 |
| 校外人员（毕业生）想开网捞在校生 | 毕业生模式禁止开网（在 M5 已说明） |
| 同一用户两次开网捞到完全一样的人 | 24h 去重已处理；7 天后允许重复 |
| 卡片数据延迟（B 已删除某动态但仍展示） | 卡片刷新机制：捞到后展示是即时拉取，老数据不缓存 |
| 用户开网时 GPS 漂移到 1km 外 | 后端二次校验上次有效定位，距离过大警告并要求重新定位 |
| 大量虚假账号开网骚扰 | 强认证（D2）+ 设备指纹 + 行为指纹，多维拦截 |

### C5. 数据隐私
- 被捞到的人**不知道**自己被捞（仅在收到招呼时知道）
- A 看到的 B 的卡片信息**不上报到 B 的设备**（不进 B 的"被谁查看了"列表）
- 历史捞到记录 90 天后自动清理

## Step D: 数据模型概要

### 涉及实体

```typescript
FishingNet {                    // 一次开网事件
  id: bigint,
  user_id: bigint,              // 开网者
  net_type: enum,               // normal | large | bomb_small | bomb_large | refish
  cast_at: timestamp,
  location: { lat: float, lng: float },
  radius_m: int,
  cost_items: { item_id, quantity }[],  // 消耗道具
  result_count: int,            // 实际捞到的人数（0 = 空网）
  status: enum,                 // success | empty | refunded | cancelled
}

FishingCatch {                  // 捞到的具体卡片记录
  id: bigint,
  net_id: bigint,
  fisher_id: bigint,
  fish_id: bigint,              // 被捞者
  caught_at: timestamp,
  action: enum,                 // pending | greeted | ignored | skipped | reported
  greet_id: bigint | null,      // 若打了招呼，关联 GreetRequest
}

GreetRequest {                  // 招呼请求
  id: bigint,
  from_user_id: bigint,
  to_user_id: bigint,
  message: string,              // 留言或破冰话术
  context: {                    // 来自哪次捞网
    net_id: bigint,
    location: { lat, lng },
  },
  status: enum,                 // pending | replied | declined | expired | withdrawn | reported
  created_at: timestamp,
  responded_at: timestamp | null,
}

UserFishingPrefs {              // 用户在 M2 的偏好
  user_id: bigint,
  visible: bool,                // 是否可被捞到（隐私模式开关）
  same_campus_only: bool,       // 仅本校
  // V1.x: gender_filter
}

IgnoreRecord {                  // 无视记录
  fisher_id: bigint,
  fish_id: bigint,
  ignored_at: timestamp,
  expires_at: timestamp,        // ignored_at + 30 天
}

BlockRecord {                   // 拉黑记录（与 M11 共享）
  blocker_id: bigint,
  blocked_id: bigint,
  blocked_at: timestamp,
}
```

### 涉及接口

```
# 开网与捞结果
POST   /fishing/cast                      开网（指定类型 + 位置）
GET    /fishing/nets/{id}                 查询某次开网结果
POST   /fishing/catches/{id}/greet        对某张卡发起招呼
POST   /fishing/catches/{id}/ignore       无视
POST   /fishing/catches/{id}/report       举报

# 我的招呼
GET    /me/greets/sent                    我发出的招呼
GET    /me/greets/received                我收到的招呼
POST   /greets/{id}/reply                 回复（建立临时会话）
POST   /greets/{id}/decline               拒绝
POST   /greets/{id}/withdraw              撤回（1h 内可撤）

# 偏好设置
GET    /me/fishing-prefs                  查询
PATCH  /me/fishing-prefs                  更新（隐私模式/范围等）

# 我的"捞到的人" 收藏
GET    /me/fishing/collections            收藏列表
POST   /fishing/catches/{id}/save         加入收藏
```

## Step E: 交互要点

### 撒网仪式（核心体验）
- **开网前预热**：揉手指/搓动渔网图标，让用户参与"准备"动作
- **撒网动画**：从手部抛出渔网，扇形展开 → 在地图上落下 + 涟漪扩散 → 收网（金色光点回收）
- **揭晓节奏**：先收网（2 秒）→ "嗖"特效逐张展开卡片 → 用户决策时卡片悬浮微动
- **空网处理**：渔网回来时空空荡荡 + 一片落叶 + "唉，这次没捞到（返还 50%）" + "再撒一网"快捷按钮

### 卡片展示
- **卡片样式**：奶油色 + 圆角 + 浅阴影，类似拍立得照片
- **滑动决策**：左滑无视，右滑打招呼，上滑收藏（类 Tinder 但更温和）
- **可上下滚动看详情**：点开卡片 = 全屏铺开，查看更详细资料

### 招呼输入
- **破冰话术推荐**：8-10 条预设话术（"我注意到你也在 X"、"嗨，刚捞到你"），点击直接发送
- **自定义输入**：≤ 50 字
- **附加表情/贴纸**：可附加单个

### 收到招呼
- **聚合通知**：收到 ≥ 3 条新招呼时合并通知"今天有 X 个人想认识你"
- **招呼列表页**：卡片瀑布流，已过期/已回复有视觉区分

### 微交互
- 打招呼成功：粒子飞出 + 提示"招呼已经送出，TA 收到时会知道"
- 收藏卡片：卡片飞入"收藏夹"图标
- 无视：卡片"飘走"动画

## 模块关联

```
M2 → 依赖：M5 认证、M7 道具经济、M9 内容审核（卡片信息）
   → 被依赖：M6 关系链（招呼通过 → 临时聊天 → 加好友）
   → 关联：M1（卡片显示其动态）、M8（招呼通知）、M11（隐私模式开关）、M10（我的捞到 / 收到的招呼入口）
```

## 风险标记

| 风险 | 缓解 |
|------|------|
| **变成"附近的人"陌生人速配工具，偏离产品定位** | 强调地理 + 时间 + 趣味性；卡片不主推性别/颜值；MVP 不做性别筛选 |
| 高活跃用户过度被捞，骚扰感强 | 单人单日被招呼上限（如 20 次），超出后系统降权 |
| 付费道具被刷子滥用 | 服务端校验 + 风控引擎（开网频率、招呼通过率等指标） |
| 校园人口少时空网率过高 | 空网保护机制（返还 / 赠送补网）+ 引导扩大半径 |
| 招呼变成骚扰话术轰炸 | 招呼有数量上限 + 拒绝后冷却 + 内容审核（M9） |
