# M8 通知与触达

> **优先级**：P0 - 多场景汇总（投递触发/招呼/胶囊解锁/任务/安全）
> **决策依据**：D6 治愈风文案、不打扰原则
> **关联模块**：所有依赖即时触达的模块

---

## Step A: 用户故事

### 核心故事
- 作为**用户**，我希望好友给我发动态/招呼/消息时收到推送，但不希望被无关消息打扰
- 作为**用户**，我希望推送内容能让我快速判断"是否值得现在打开 App"
- 作为**用户**，我希望可以按通知类别精细控制（如关闭"任务提醒"，但保留"招呼"）
- 作为**用户**，我希望免打扰时段（夜里 23:00-7:00）不被吵醒

### 扩展故事
- 作为**用户**，我希望未读通知聚合在通知中心一目了然
- 作为**用户**，我希望关键通知（异地登录、认证状态变化）即使关闭推送也强制送达
- 作为**用户**，多条通知能聚合"今天有 X 个新招呼"减少打扰

### 异常故事
- 作为**用户**，App 杀进程后仍能收到推送（系统级）
- 作为**用户**，无网络时通知不丢失，恢复后批量送达

## Step B: 核心流程

### B1. 通知触发主线

```
事件源（M1/M2/M3/M5/M6/M7/M9）→ 入"通知中心服务" → 决策：
├── 用户偏好检查（是否关闭此类通知）
├── 免打扰时段检查（不在静默期）
├── 频率控制（聚合 N 分钟内同类）
├── 重要级别检查（强制类不受偏好影响）
↓
分发：
├── 推送通道（APNs / FCM / 厂商通道）
├── In-App 通知（顶部 banner / 红点）
├── 通知中心列表（持久存储）
└── 邮件 / 短信（仅安全类）
```

### B2. 用户查看通知

```
1. App 入口红点/数字 → 通知中心 Tab
→ 2. 通知列表按时间倒序，按类别可筛选
→ 3. 点击通知 → 跳转对应详情（动态/会话/招呼/胶囊/钱包等）
→ 4. 已读自动标记
→ 5. 长按可标记已读 / 删除单条
```

### B3. 异常分支

```
├── 用户从未授权系统推送 → 关键事件用 in-app 顶部 banner 兜底
├── 推送失败重试（厂商通道故障） → 切备用通道
├── 通知巨量积压（如关注用户多） → 按聚合规则压缩
└── 用户清空 App 缓存 → 通知列表从云端拉取（保留 30 天）
```

## Step C: 业务规则与边界 Case

### C1. 通知分类

| 大类 | 子类 | 默认开关 | 可关 | 聚合规则 | 跳转目标 |
|------|------|---------|------|---------|---------|
| **互动** | 动态被点赞 | 开 | 可 | 1 小时内同动态聚合 | 动态详情 |
| | 动态被评论 | 开 | 可 | 不聚合（每条独立） | 评论列表 |
| | 评论被回复 | 开 | 可 | 不聚合 | 评论列表 |
| | 动态被路过查看 | 关 | 可 | 24h 聚合 | 动态详情 |
| **关系** | 好友请求 | 开 | 可 | 不聚合 | 请求列表 |
| | 好友请求通过 | 开 | 可 | 不聚合 | 好友主页 |
| | 好友新动态（精选）| 关 | 可 | 1 天聚合 | 好友主页 |
| **招呼** | 收到招呼 | 开 | 可 | ≥3 条聚合"今天 X 个招呼" | 招呼列表 |
| | 招呼被回复 | 开 | 可 | 不聚合 | 临时会话 |
| **私聊** | 新消息 | 开 | 可（按会话） | 不聚合（每条独立或合并到 N+ 数字角标） | 会话 |
| **胶囊** | 接近可解锁的胶囊 | 开 | 可 | 同胶囊 24h 内只 1 次 | 胶囊详情 |
| | 投递触发（friend_drop 当好友进圈） | 开 | 可 | 不聚合（仪式感） | 动态详情 |
| **任务/经济** | 每日签到提醒 | 开 | 可 | 每日 1 次 | 任务页 |
| | 任务奖励到账 | 开 | 可 | 同时段聚合 | 钱包 |
| | 充值结果 | 开 | 可 | 不聚合 | 钱包 |
| **安全（强制）** | 异地登录 | **强制** | 不可关 | 不聚合 | 设备管理 |
| | 认证状态变化 | **强制** | 不可关 | 不聚合 | 认证页 |
| | 处罚通知 | **强制** | 不可关 | 不聚合 | 处罚详情 |
| | 申诉结果 | **强制** | 不可关 | 不聚合 | 申诉页 |
| | 大额充值确认 | **强制** | 不可关 | 不聚合 | 钱包 |
| **运营** | 活动/公告 | 开 | 可 | 1 周内最多 1 条 | 落地页 |
| | 版本更新 | 开 | 可 | 不聚合 | 升级 |

### C2. 业务规则

#### 推送通道选择
- iOS：APNs
- Android：FCM（海外）/ 小米/华为/OPPO/vivo（国内厂商通道，提升送达率）
- 通道失败回退顺序：厂商 → FCM/APNs → in-app

#### 免打扰
- 用户可设全局静默时段（默认 23:00-7:00 不推送，急 / 强制类除外）
- 免打扰期间通知仍入通知中心，仅不出系统弹窗

#### 频率控制
- 单用户 1 小时内推送上限 10 条（超限则聚合最近 5 条为 1 条）
- 单用户 1 天推送上限 50 条
- 同类型 1 小时聚合规则按上表

#### 内容文案规则（D6 治愈风）
- 禁止"震惊体""焦虑营销"
- 用户昵称放前面（个性化）
- 表情符号克制（最多 1 个）
- 重要事件保留仪式感（如胶囊解锁的文案要打动人："在 X 楼下，你曾埋下的胶囊正在召唤你"）

### C3. 边界 Case

| 场景 | 处理 |
|------|------|
| 用户多端登录 | 推送给所有设备；某端阅读后其他端取消 |
| 用户禁用了系统推送 | 无法系统推送，仅能 in-app + 通知中心 |
| 通知巨量并发（明星活动） | 队列削峰 + 用户级 QoS |
| 历史通知过期清理 | 通知中心保留 30 天，老数据自动清理 |
| 厂商通道临时故障 | 切备用 + 失败上报 |

## Step D: 数据模型概要

```typescript
Notification {
  id: bigint,
  user_id: bigint,
  type: enum,                   // ~30 种细分类型
  title: string,
  body: string,
  payload: object,              // 跳转参数
  read: bool,
  pushed: bool,
  pushed_at: timestamp | null,
  delivery_channels: string[],  // ['apns', 'in_app', 'email']
  delivered_channels: string[],
  group_key: string | null,     // 用于聚合
  created_at: timestamp,
}

NotificationPrefs {
  user_id: bigint,
  // 每个子类一个 bool（除强制类）
  // 例：fishing_greet_received: true, post_liked: false ...
  silence_start: int,           // 静默时段（分钟数 0-1440）
  silence_end: int,
}

DeviceToken {                   // 设备 token
  user_id: bigint,
  device_id: string,
  platform: enum,               // ios | android
  token: string,
  channel: enum,                // apns | fcm | xiaomi | huawei | oppo | vivo
  updated_at: timestamp,
}
```

### 涉及接口

```
GET    /me/notifications                通知列表
PATCH  /me/notifications/read           标记已读（按 ID 或全部）
DELETE /me/notifications/{id}           删除
GET    /me/notification-prefs           查询偏好
PATCH  /me/notification-prefs           更新偏好
POST   /devices/register                注册设备 token
DELETE /devices/{id}                    注销设备 token
```

## Step E: 交互要点

### 通知中心
- **顶部 Tab 筛选**：全部 / 互动 / 招呼 / 系统
- **聚合卡片**："3 个人点赞了你的动态"展开看具体
- **未读样式**：左侧浅色圆点 + 加粗文字
- **下拉刷新 + 上拉加载**

### 偏好设置（在 M11）
- **分组开关**：每个大类一个总开关 + 子项细控
- **静默时段**：滑块/时间选择器
- **强制类**：标灰 + 说明"为了你的安全/资金/账号，此类通知无法关闭"

### 推送文案设计原则（D6 治愈）
- 例（胶囊）："你在 28 楼下，曾埋下一封信，它在等你 📜"
- 例（投递触发）："小心心 💗 给你 @ 在 X 教学楼"
- 例（招呼）："早安☁️ 有 3 个新朋友想认识你"
- 例（动态点赞）："小白点亮了你"

## 风险标记

| 风险 | 缓解 |
|------|------|
| 推送骚扰导致用户卸载 | 频率控制 + 用户偏好细粒度 + 默认保守 |
| 厂商通道送达率波动 | 多通道兜底 + 监控告警 |
| 关键安全通知被用户错过 | 强制类 + 多通道发送 + in-app 兜底 |
| 通知服务高并发 | 队列削峰 + 优先级队列 |
