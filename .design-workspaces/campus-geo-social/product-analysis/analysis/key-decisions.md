# 关键产品决策记录（Phase 1 → Phase 2 过渡）

> 这些决策驱动 Phase 2 所有模块的设计，不可逆改无需重做大量分析。
> 所有后续模块文档必须遵循以下决策。

| 编号 | 决策点 | 选择 | 对设计的影响 |
|:----:|--------|------|-------------|
| **D1** | 定时定点投递的目标识别 | **必须是已加好友** | 暧昧场景必须先经"捞网→认识→加好友"链路才能用，社交破冰路径前移到 M2 捞网 |
| **D2** | 学生身份认证强度 | **强认证**：学信网/学生证 OCR + 人脸核身 | 注册流程需 4-6 步，时长预估 2-5 分钟，失败重试机制必备 |
| **D3** | 捞网经济模式 | **混合**：任务/签到赚币 + 付费补充 | M7 需双向账本：积分(任务赚) + 钻石(付费购)，部分道具仅限钻石 |
| **D4** | 跨校漫游可见范围 | **只看广场聚合**，看不到具体用户 | M4 大幅简化：仅一个"全球校园广场"feed，不需用户卡/不能加好友 |
| **D5** | 时间胶囊解锁规则 | **仅发布者本人到达** | M3 是纯私人「时光信件」性质，简化数据模型，失去公共回忆维度 |
| **D6** | 视觉风格基调 | **青春治愈**：明亮配色 + 手绘元素 | theme-generator 产出方向：暖色系（如奶油黄/草莓粉/薄荷绿）+ 圆角 + 插画感图标 |

## 决策的二阶影响

### D1（必须好友才能定向投递）的连锁影响
- 「捞网→陌生人卡片→打招呼→加好友」必须是顺滑闭环，否则 D1 锁死了暧昧玩法
- 个人主页需要清晰的"加好友"入口
- 好友请求需要"附言"（"我捞到你了！"）来提升通过率

### D2（强认证）的连锁影响
- 注册页是高流失风险点，需做"为什么需要这些信息"的说明
- 必须有"暂存中、人工审核中"的状态页，不能让用户等死
- 失败重试 ≥ 3 次后需有人工客服通道
- 涉及个人敏感信息收集，必须有隐私政策弹窗与勾选

### D3（混合经济）的连锁影响
- 任务体系（每日签到、首次发动态、邀请好友等）是 M7 必须设计的子模块
- 钱包页面需展示双账本（积分余额 + 钻石余额）
- 商品页需标注"积分可购"vs"仅钻石"

### D4（跨校仅广场）的连锁影响
- M4 实质等价于"全球热门动态聚合 feed"，复杂度从"完整社交"降为"内容聚合"
- 不需要为外校用户做隐私保护判断（因为根本看不到具体人）
- 但广场内容如何按校园分组、如何排序（热门/最新）需明确

### D5（胶囊仅本人）的连锁影响
- 胶囊不需要权限模型（解锁条件只有 1 种：本人 + 到达地点）
- 但失去了"留给某人的胶囊"这个产品诉求 → 需要确认：D1 已说"定时定点投递"覆盖了"留给某人"的场景，所以胶囊纯粹是给自己的时光机
- 胶囊与"定时定点投递给好友"的功能边界需在 M3 严格区分

### D6（青春治愈）的连锁影响
- 地图配色不能用赛博风（需用浅色卡通地图）
- 图标必须手绘风（不用线性几何图标）
- 微动效需柔和（不能机械感），偏向"漂浮、晃动、生长"等有机动效
- 关键插画素材：捞网、泡泡、信封、樱花/落叶（季节）、像素小人等
