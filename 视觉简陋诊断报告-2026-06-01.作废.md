## 0. TL;DR（先看结论）

| 问题 | 量级 | 责任层 |

| **C. 文档承诺的多个视觉信号没出现在最终视觉**（如"暖白米"=#FCFCFD 视觉接近纯白；卡片/输入框圆角值正确但没有"大圆角柔和"的辨识度；BrandSlogan 用 body-lg 14px 不够份量） | 🟠 设计 | design-planner styles 数值与文档辞令的视觉效果有落差 |

## 3. 渲染链路体检（问题 ③："schema 渲染如图有没有问题，有没有 schema 其实样式很完善但是前端没渲染出来的情况"）

> 入口：apps/design_front/src/views/editor/Canvas/index.tsx 行 660+
> editor 非 preview 模式 → SchemaRenderer + buildEditorPreviewState → resolveNodeStyles → resolveTokensInStyles → resolveExpression

### 3.1 token 解析能力实测（resolveTokens.ts）

```ts
// 第 41-67 行 resolveTokenValue：
// - value.includes(' ') && value.includes(TOKEN_PREFIX) → 空格 split + 单段 starsWith 替换 + join
// - 否则 value.startsWith(TOKEN_PREFIX) → 整段替换
```

**实测：**
- `"$token:colors.primary"` → ✅ 解析为 `#5B6CFF`
- `"1px solid $token:colors.borderLight"` → ✅ split 后能命中（虽然包含逗号空格的情况会失败，但纯空格分隔 OK）
- `"radial-gradient(circle at 50% 50%, $token:colors.primaryLight 0%, transparent 70%)"` → ⚠️ 部分能解析（因为 `$token:colors.primaryLight` 周围有空格，单段 startsWith 命中），但右括号 `70%)` 这种依然属于"非 token 段"原样保留——**整段 string 解析后是合法 CSS**，**装饰是能渲染的**

→ **design 阶段 craft-decoration-rebalance.md 把"字符串内嵌 token 不解析"列为根因是误判**。真正的根因是 **alpha 12% / 8% 在暖白米底色上视觉接近 0**，不是不渲染。前端 console 也不会有 warning（资源命中）。

### 3.5 ❌ 真正的渲染层 bug A：childrenStates 不响应 autoVisualState

SchemaRenderer.tsx 行 359-375：

```ts
} else {
  // 非 interactionPreview 路径（默认编辑/预览模式）
  const effectiveActiveState = node.activeState ?? 'default';
  if (effectiveActiveState === 'default') {
    cvMap = defaultStateDef?.childrenVisibility;
    csMap = defaultStateDef?.childrenStates;
  } else {
    const activeStateDef = node.states?.find((s) => s.name === effectiveActiveState);
    cvMap = mergeStateMaps(...);
    csMap = mergeStateMaps(...);
  }
}
```

→ **这段只读 `node.activeState`，没读「activeWhen 表达式自动激活的态」**。
对比 resolveStyles.ts 行 132-145 在样式层有 `autoActivatedState` 计算并参与 effectiveStateName，**两层分裂**。

**实际后果**（场景）：用户勾选政策 → state.view.form.policy=true → PolicyCheckVisual.checked.activeWhen 命中 → **styles 切换正确（边色+底色变主色）**，但 **CheckMark 不会切到 visible 态（display:none → block 不会触发）** → ✓ 永远不显示。

**修复方向**：把 resolveStyles.ts 的 `autoActivatedState` 计算抽到 SchemaRenderer 通用工具，在选 cvMap/csMap 时也参考。

### 3.6 ❌ 真正的渲染层 bug B：截图显示「登录中…」

schema：
- `screen.stateInit.view.submitting.defaultValue = false`
- SubmitBtn.props.textContent = `"{{ state.view.submitting ? '登录中…' : '登录' }}"`

期望编辑模式下 submitting=false → 文字="登录"。但截图显示"登录中…"，意味着表达式求值时 `state.view.submitting` 被当成 truthy。

可能原因（未深查）：
1. buildEditorPreviewState 把 boolean false 误转成 "false"（字符串）→ `'false' ? a : b` 总是 a
2. resolveExpression 的某条 fallback 让 undefined 变成 truthy
3. submitting 字段定义里没显式 `previewValue` 导致编辑期某段代码用了 fallback="loading"

**修复方向**：跑一次 `console.log(activeDataContext.state.view.submitting)`（或加临时 dev 日志），追到 EditorPreviewState 构造里 boolean 处理逻辑。

|---|---|
| Logo 真画了但截图肉眼仍觉得 logo 不"校园" | 「两圆相遇」抽象到大脑识别需要叙事支持，截图无 caption 让人 get 不到 |
| Slogan 字小 | schema 用了 body-lg(16)+weight=500，文档说应当 600；并且文档自身没把 slogan 升到 h4(20) | 
| 卡片"漂浮感"不强 | shadow=sm 在 #FCFCFD 上几乎看不见，但文档没要求更强 shadow |

**核心结论 ③**：渲染管线**整体正常吃下了 schema**——
- ✅ token 解析（含字符串内嵌）OK
- ✅ visualState autoActivated 在 styles 层 OK
- ✅ BrandLogo PNG OK
- ❌ visualState autoActivated 在 childrenStates 层**有 bug**（影响 PolicyCheckVisual 勾选时 ✓ 显示）
- ❌ EditorPreviewState 对 boolean false 处理疑似有 bug（让默认编辑预览出现"登录中…"）
- 装饰看起来"接近无"是设计参数（12%/8% alpha），不是渲染问题

```ts
// 新建 features/design-engine/src/styles/resolveAutoState.ts
export function resolveAutoActivatedState(
  node: ComponentNode,
  dataContext: DataContext,
): string | null {
  if (!node.states?.length) return null;
  for (const state of node.states) {
    if (state.activeWhen && hasExpression(state.activeWhen)) {
      const result = resolveExpression(state.activeWhen, dataContext);
      if (result) return state.name;
    }
  }
  return null;
}

// SchemaRenderer.tsx 行 359 改为：
const autoActivated = resolveAutoActivatedState(node, dataContext);
const effectiveActiveState =
  autoActivated ?? node.activeState ?? 'default';
```
