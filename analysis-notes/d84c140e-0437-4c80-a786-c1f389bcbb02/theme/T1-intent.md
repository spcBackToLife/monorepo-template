# T1：风格意图提取 — 校园社交-登录页

## 1. 用户原始描述

来源：`project.meta.styleDirection.summary`（product-analyst 阶段沉淀）

> "简约时尚 + 校园温度：极简留白 / 大字号品牌强调字 / 圆角输入框 / 单一强调色 / 极细几何装饰；**避开浓郁渐变、卡通插画、纯黑白极简**"

辅助语境（来自 `project.meta.targetUser` + `coreScenarios`）：
- 目标用户：18-22 在校大学生（开学季 / 换机 / 重登；对学生范儿敏感）+ 23-26 研究生应届（"不像学生玩具"，对设计审美敏感）
- 屏类型：登录页（功能性强，信任 + 安全感是底线）
- 业务约束：手机号 + 密码/验证码两种登录、协议必勾、错 5 次锁 30min（**安全感不可降级**）

## 2. 7 维度结构化提取

### 2.1 summary（一句话定性，≤30 字）

> **「简约时尚 + 校园温度（极简留白 + 单一蓝紫强调色）」** (26 字)

理由：
- 「简约时尚」直接复用用户原话，确保 brand voice 不漂移
- 「极简留白」是装饰策略的关键词锚点
- 「单一蓝紫强调色」预告了 T2 的 seedColor 决策——主色锁死 brand 蓝紫，不走多彩

### 2.2 aesthetics（取 2 个）

| 标签 | 是否选用 | 提取证据 / 否决理由 |
|------|--------|---------------------|
| `glassmorphism` | ❌ | 用户没提"透明感/毛玻璃/苹果风"；登录页用毛玻璃会显得"不够正式" |
| `minimal` | ✓ | 用户**原话「极简留白」** 直接命中；"避开纯黑白极简"只是排除极端版本，不否认 minimal 本体 |
| `luxury` | ❌ | toC 校园场景；学生群体追求"不油腻"，金色/香槟会显得装腔 |
| `brutalist` | ❌ | 用户**明确「避开纯黑白极简」**，brutalist 黑白厚边框正中其负面雷区 |
| `organic` | ❌ | 用户**明确「避开卡通插画」**；organic 偏柔和治愈，与"时尚"基调冲突 |
| `futuristic` | ❌ | 用户**明确「避开浓郁渐变」**，futuristic 的 neon/mesh gradient 正中其负面雷区 |
| `flat` | ✓ | 用户原话「**单一强调色** + **极细几何装饰**」是 flat 教科书特征：纯色块、弱阴影、几何感 |
| `playful` | ❌ | 学院温度 ≠ 童趣可爱；目标用户里有"研究生应届"明确反对玩具感 |

**结论 → `["minimal", "flat"]`** （2 个，聚焦）

### 2.3 decoration（装饰密度）

- 取值：**`minimal`**
- 理由：用户原话"极简留白" + "极细几何装饰" 两次强调"少"，决策无歧义。
- 影响 T4：背景 strategy=solid（不渐变/毛玻璃）、阴影 strategy=soft、边框 strategy=subtle 且 width=1px。

### 2.4 colorTemperature（色温）

- 取值：**`neutral`**
- 推理：
  - 「简约时尚」偏冷（理性/克制），但「校园温度」要一丝暖意（不冷漠）
  - 候选：cool → 太疏离；warm → 与"时尚"基调冲突
  - 折中 → **neutral**；温度感通过 seedColor 微调（蓝紫带紫即偏暖一点）
- 影响 T2：灰阶用纯中性灰，textPrimary 不带蓝/红偏。

### 2.5 brightness（明暗倾向）

- 取值：**`both`**
- 理由：
  - 用户没明确，按方法论默认值 `both`
  - 登录页是高频入口屏，必须兼容系统暗黑模式（学生群体 18-25 暗黑模式占有率 > 60%）
  - light 主推，dark 兜底
- 影响 T6：必须出 light + dark 两套等权变体（不允许只出 light + dark 兜底）

### 2.6 seedColors（种子色，取 1）

| 色号 | 来源 | HSL | 理由 |
|------|------|------|------|
| `#5B6CFF` | AI 推断 | HSL(233, 100%, 68%) | 蓝紫主色，理由见下 |

**为什么是 #5B6CFF：**

| 候选 | HSL | 否决 / 入选理由 |
|------|------|----------------|
| #1677FF Antd 默认蓝 | HSL(212, 100%, 54%) | ❌ 太通用，没品牌识别度；学生群体一眼看出是"模板蓝" |
| #FF6F91 草莓粉 | HSL(347, 100%, 72%) | ❌ 偏可爱治愈（example 的青春治愈风），与"时尚"基调冲突 |
| #00C896 薄荷绿 | HSL(165, 100%, 39%) | ❌ 偏治愈/医疗，校园温度不假但缺少"时尚" |
| #4A6CF7 中性蓝紫 | HSL(229, 91%, 63%) | △ 备选，方法论默认推断色，但识别度仍偏弱 |
| **#5B6CFF 知性蓝紫** | **HSL(233, 100%, 68%)** | ✅ 蓝紫调（往紫挪 5°）兼有「理性的蓝」+「时尚的紫」；S=100% 表达"单一强调色"的纯度；L=68% 比 Antd 蓝亮 14pt，年轻化 |
| #6366F1 Indigo 500 | HSL(239, 84%, 67%) | △ Tailwind indigo，几乎等价候选；但 #5B6CFF S 更饱满更"强调" |

**多重验证 #5B6CFF：**
- HSL 三元组：233° 蓝紫 + 100% 饱和 + 68% 亮度
- 学院联想：剑桥蓝紫袍、知性、不过分活泼
- 时尚联想：2024-2026 数字时尚趋势色之一（Klein Blue → Digital Indigo）
- 与"避开"集合的距离：远离粉/橙（卡通）、远离金（轻奢）、远离纯白黑（极简到极端）、远离 neon（科技感）

### 2.7 references（参考品牌）

- **`[]` 空数组**——用户没给参考品牌；不允许 AI 强加 "Notion/Soul/小红书" 等臆测。

## 3. 候选 aesthetics 标签穷举与否决

> 完整 8 类全否决推理已在 §2.2 表内。最终取 `["minimal","flat"]`。

## 4. 多角度验证

| 维度 | 结论 |
|------|------|
| 用户角度 | 18-25 学生群体看到「白底 + 蓝紫主色 + 极细装饰」第一反应：「这 App 看起来挺正经的，不像学生作业；又不会太严肃像银行 App」——契合 Secondary 用户群"不像学生玩具"的隐性期待 |
| 品牌角度 | 蓝紫色与同类校园社交 App（Soul 黑紫 / 探探橙红 / 即刻黄黑）形成明显区分；且 #5B6CFF 与微信/QQ/支付宝等国民产品主色无冲突，品牌识别独立 |
| 技术角度 | minimal + flat 是落地难度最低的组合（无毛玻璃/渐变/mesh）；登录页 RN/H5/小程序都能 1:1 还原；APCA 验证容易过 |
| 竞品角度 | Soul(暗紫)/Tantan(橙)/Jike(亮黄) 都偏"个性鲜明"；本项目走「克制的时尚」差异化——这对刚上线 / 求口碑期的产品很重要 |
| 合规角度 | 协议必勾是合规红线（M5），蓝紫主色 → 协议链接（同主色）天然有"重要提示"信号，配合登录主 CTA 同色构成视觉一致性 |

## 5. 关键决策与假设

- **假设 1**（明暗）：用户没说 → 取 `both`，必须出齐 light + dark；理由：学生群体暗黑模式占有率高 + 系统级深色模式 iOS 13+/Android 10+ 普及
- **假设 2**（seedColor）：在 #4A6CF7 / #5B6CFF / #6366F1 三个高度相近候选中选 #5B6CFF，因 S=100% 更能表达"单一强调色"的纯度宣言；如用户后续觉得"太亮太鲜"，备选退回 #4A6CF7（S=91%, L=63%）
- **假设 3**（colorTemperature）：折中 neutral 而非 warm/cool；理由：登录页是"功能屏"不是"情感屏"，温度感由 seedColor 蓝紫的"紫"成分自然带出
- **假设 4**（references）：用户没给参考品牌时不主动臆测，保留 `[]` 空数组，避免下游错误对标

## 6. 替代方案

若用户对 #5B6CFF 不满意，备选方向：
- 觉得"太蓝紫不够时尚" → 试 #6366F1（Indigo 500，相同色相低饱和）
- 觉得"颜色单调" → 加 secondary 暖橘 #FF8A65 作辅助强调（仍保 minimal，仅 CTA/标签用）
- 觉得"不够校园温度" → 将主色饱和度降至 S=85%，亮度提至 L=72%（偏粉紫）
