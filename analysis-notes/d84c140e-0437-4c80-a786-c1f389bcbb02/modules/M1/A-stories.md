# Step A: 用户认证-登录（M1）— 用户故事穷举

## 推理过程

### 角色识别

登录场景下只有一种主角色：**未登录或被登出的用户**。无管理员视角（管理员登录走另一套后台系统，不是本产品入口）。

按用户细分：
- **新用户**：第一次下载 / 注册后首次登录
- **老用户**：换机 / 清缓存 / token 过期后回到登录页
- **找回中用户**：忘记密码后回到登录页

### 核心故事（缺一则模块无意义）

C1. 作为**新用户**，我希望用**手机号 + 验证码**免密登录，以便不用记密码就能进入产品
C2. 作为**老用户**，我希望用**手机号 + 密码**快速登录，以便比验证码更快进入（不用等短信）
C3. 作为**任何用户**，我希望能**自由切换"验证码 / 密码"两种登录方式**，以便选择当下最方便的方式
C4. 作为**任何用户**，我希望提交前**勾选用户协议和隐私协议**，以便明确知晓自己同意了什么（合规红线）
C5. 作为**任何用户**，我希望登录成功后**自动跳转主屏**，以便立即开始用产品

### 扩展故事（提升体验但不致命）

E1. 作为**回访老用户**，我希望**自动填充上次登录的手机号**，以便减少输入（依赖本地缓存，由 interaction 阶段实现）
E2. 作为**用户**，我希望验证码输完 6 位**自动提交**，以便少点一次按钮
E3. 作为**用户**，我希望密码输入框有**显示/隐藏密码按钮**，以便核对自己是否输错（眼睛 toggle）
E4. 作为**没有账号的用户**，我希望从登录页能**点击"注册账号"**跳转到注册流程（本期跳占位屏）
E5. 作为**忘记密码的用户**，我希望从登录页能**点击"忘记密码？"**跳转到找回流程（本期跳占位屏）

### 异常故事（错误 / 边界 / 异常场景）

X1. 作为**用户**，如果**手机号格式不对**（非 11 位 / 不是 1[3-9] 开头），我希望**立即在输入框下看到行内提示**，而不是提交后才知道（失焦校验）
X2. 作为**用户**，如果**协议未勾选**，我希望**登录按钮 disabled**，以便明确知道还差一步
X3. 作为**验证码用户**，如果在 **60 秒内重复点击"发送验证码"**，我希望**按钮 disabled + 显示倒计时**，以避免重复发送
X4. 作为**验证码用户**，如果短信通道**发送失败**，我希望**Toast 明确提示"请稍后重试"**且**不消耗当日额度**
X5. 作为**密码用户**，如果**密码错误**，我希望**Toast 提示错误**+ 失败次数累加；连续错 5 次后**账号锁定 30 分钟**并显示锁定剩余时间
X6. 作为**用户**，如果**网络断开**，我希望看到**离线 banner（全局）+ 登录按钮阻断**，且我已经输入的内容**不丢失**
X7. 作为**用户**，如果**服务端 5xx**，我希望看到**兜底 Toast**（不是技术错误码），并提示我重试
X8. 作为**提交中的用户**，如果**重复点击登录按钮**，我希望系统**忽略后续点击**（防抖 + submitting 守卫），避免发起多次请求
X9. 作为**用户**，如果在请求进行中**离开页面**（截屏/切后台/锁屏唤醒），我希望进行中的 fetch **被取消**避免脏数据

### 管理 / 后台故事

无（登录页前端无管理员入口）。

## INVEST 自检

按 INVEST 6 维度检视上述 19 条故事：

| 维度 | 是否满足 | 备注 |
|------|:-------:|------|
| Independent | ✅ | 故事互不依赖（C2 不依赖 C1，X1~X9 各自独立） |
| Negotiable | ✅ | 验收阈值（60s 冷却 / 5 次锁 / 30min 锁定时长）可调整 |
| Valuable | ✅ | 每条都对应明确用户价值（不用记密码 / 减少输入 / 防错 / 兜底）|
| Estimable | ✅ | 实现路径清楚（state.set / effect.fetch / ui.startTimer） |
| Small | ✅ | 每条 ≤ 1 个迭代可完成 |
| Testable | ✅ | 每条都有可观察的验收点（Toast 出现 / 按钮 disabled / 倒计时显示） |

## 常见遗漏清单核对

| 项 | 是否覆盖 | 落到哪 |
|----|:-------:|--------|
| 首次使用 vs 老用户回访 | ✅ | C1/C2/E1 |
| 无数据/空状态 | N/A | 登录页无列表 |
| 数据过多分页 | N/A | 登录页无列表 |
| 权限不足处理 | N/A | 登录页本身就是建立权限的屏 |
| 网络异常/超时 | ✅ | X6/X7/X9 |
| 并发操作冲突 | ✅ | X8（重复提交防抖）|
| 多设备同步 | ⚠️ 不覆盖 | session 多端登录策略本期不做（后端处理）|
| 用户删除/注销后数据 | N/A | 不在登录页处理 |
| 内容违规/敏感词 | N/A | 登录页无 UGC |
| 国际化 | ⚠️ 占位 | preferences.lang='zh-CN'，本期仅中文 |
| 无障碍 | ⚠️ 留白 | 输入框有 label，进一步 a11y 后续 |

## 关联其他模块

- **依赖**：M5（协议合规——必勾才能提交）/ M6（防刷规则——倒计时、锁定）
- **被依赖**：M2/M3/M4 占位屏的入口；session 一旦建立后，后续所有屏都依赖
- **跨屏跳转**：
  - `00-login` → `01-home`（C5 登录成功）
  - `00-login` → `00-register`（E4 注册账号）
  - `00-login` → `00-forgot-password`（E5 忘记密码）
