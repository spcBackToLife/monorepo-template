## 2. 设计目标（Phase B 输出，≥3 个）

### G1: mood-conveyance（P0）
- **statement**: 让用户进登录页 0.5 秒感受到"克制温度"调性，区别于纯白冷峻 SaaS 风，降低首次注册防备
- **whyMatters**: 用户带'好奇 + 略防备'情绪进屏；视觉先安抚 = 注册转化前置条件；差异化定位（克制温度 vs 即刻多色 / 小红书插画 / Notion 冷峻）的可视化首层
- **impactMode**: mood-conveyance
- **successCriteria**:
  1. "屏底偏暖区（取屏中下 1/3 中心点像素采样）RGB 与 #FFFFFF 距离 ≥ 15pt"
  2. "出现 ≥ 2 处 soft-glow 单一族装饰元素（圆形光斑 / 极细几何线，单一族不混杂），核心 alpha ≥ 20%"
  3. "主色 #5B6CFF 与辅色 #A776FF 仅以'点缀'形态出现，不形成连续 > 30% 屏面积的实色块"
  4. "整屏灰阶占比 ≤ 35%（按截图主色统计；> 35% 即触发冷峻调）"
- **forbiddenSignals**:
  - "屏底像素 #FFFFFF 或与之距离 ≤ 5pt"
  - "装饰核心 alpha < 15% 接近隐形"
  - "出现直角元素（border-radius < 4px）"
  - "主色 #5B6CFF 形成连续 > 30% 屏面积的实色块"
  - "灰阶占比 > 40%（含纯白底 + 灰字 + 灰边）"
  - "装饰族混杂（光斑 + 几何 + 插画 同时出现）"

### G2: cta-clarity（P0）
- **statement**: 让 SubmitBtn 成为首屏唯一主角，用户视线 BrandLogo→FormCard→SubmitBtn 三跳必落于此，与 GetCodeBtn 形成 ≥ 4 权重差
- **whyMatters**: 登录核心动作；用户复登求快场景下'找不到主按钮'即流失；medium 紧迫度 + minimal 风格下视觉权重差是硬约束
- **impactMode**: cta-clarity
- **successCriteria**:
  1. "SubmitBtn 与 GetCodeBtn 视觉权重差 ≥ 4（按 weight 公式：色对比+尺寸比+字重差+装饰密度+动效 累加）"
  2. "SubmitBtn 视觉高度 ≥ 48px + 字号 ≥ 16px + 字重 ≥ 600 + 上方留白 ≥ 24px"
  3. "SubmitBtn 占 FormCard 内部宽度 100% 且面积占比 ≥ 25%（按 48px / 内容区高度计）"
  4. "SubmitBtn 在 default / hover / pressed / loading / disabled 5 态相邻像素差均 ≥ 3%（每态视觉差异显著可分辨）"
- **forbiddenSignals**:
  - "SubmitBtn 与 GetCodeBtn 同色 / 同字号 / 同字重（主从混乱）"
  - "SubmitBtn 在 disabled 态仍显示主色填充"
  - "SubmitBtn 字号 < 16px 或字重 < 600"
  - "SubmitBtn 上方留白 < 16px（与字段视觉粘连）"

### G3: trust-signal（P1）
- **statement**: 让协议勾选 + 错误反馈以友善温和方式触达，避免合规暴击 / 错误暴击激化用户初见的防备心理
- **whyMatters**: 用户进屏带'略防备'；合规红线（必勾协议）若用 native checkbox + 红字暴击会激化防备 → 注册流失；错误暴击让用户怀疑被刁难
- **impactMode**: trust-signal
- **successCriteria**:
  1. "PolicyCheckVisual 视觉为'圆角 ≥ 4px + 主色 #5B6CFF 对勾'，未勾态为细边框 + 透明底，勾选态为主色填充 + 白色对勾，非 native 黑色方框"
  2. "错误态文字色（PhoneError / CredentialError）使用 theme.error #DD4747 (距纯红 #FF0000 ≥ 30pt) + caption 12px 字号软化 + 紧贴 input 下方 4px"
  3. "协议链接（TermsLink / PrivacyLink）使用主色 + textDecoration:underline，与正文 textPrimary 0.88 黑形成可视区分"
  4. "状态切换（勾选 / 错误出现 / 错误消失）有 ≥ 200ms transition，非瞬间 pop"
- **forbiddenSignals**:
  - "Checkbox 视觉为黑色 native 方块或灰冷无主色"
  - "错误字色 #FF0000 或与之距离 < 20pt（暴击）"
  - "错误字号 ≥ 14px（与正文同字号，视觉抢戏）"
  - "协议链接无可视区分（与正文同色无 underline）"
  - "状态切换无 transition（瞬间）"

### G4: brand-recognition（P0）
- **statement**: 让 BrandLogo 在屏顶撑起品牌识别第一锚点，承载'校园社交 + 现代时尚'双重定位（真画素材非占位虚线，与主色形成视觉联动）
- **whyMatters**: 登录页是品牌门面屏；BrandLogo 真画 = 第一印象差异化的载体；23-26 应届对'学生范儿'敏感 = Logo 风格直接决定他们留存判断
- **impactMode**: brand-recognition
- **successCriteria**:
  1. "BrandLogo node.materialProjectId 非空（真调 material-painter 画了 PNG/SVG 素材，非占位虚线 / 文字 placeholder / 空白 div）"
  2. "BrandLogo 视觉边界占首屏面积 ≥ 3%（≥ 100×100 px @ 393×852 viewport，相比 v3 80×80 ≈ 1.4% 是显著台阶）"
  3. "Logo 视觉中含 theme.primary #5B6CFF 或 secondary #A776FF（形态可为描边 / 主体形 / 强调点 / 渐变锚点，非纯黑/纯灰/纯白）"
  4. "Slogan（BrandSlogan）字号 ≥ 16px（body-lg） + 与 BrandLogo 垂直间距 ≤ 12px（视觉成对，非断开）"
- **forbiddenSignals**:
  - "BrandLogo materialProjectId 为空（仍占位 v8 残留）"
  - "BrandLogo 视觉占首屏 < 2%"
  - "Logo 视觉为纯黑 / 纯灰 / 纯白（与主色完全无关）"
  - "Slogan 与 Logo 间距 > 24px（视觉断开成两组）"
  - "Logo 形态为单字字标'C'（v3 残留方案，用户已否决字母 logo）"

### G5: state-feedback（P1）
- **statement**: 让 loginMode 切换 / 锁定态触发 / 倒计时变化等 5 种业务状态显著可分辨，用户始终知道当前状态 + 还需等多久
- **whyMatters**: 登录页业务状态密度高（6 view 字段 + 主流程/locked 双状态机 + 倒计时）；视觉模糊让用户困惑/重试 → 失败率↑ → 后端撞库压力 + 客服成本
- **impactMode**: state-feedback
- **successCriteria**:
  1. "loginMode 切换 (code → password) 前后截图像素差 ≥ 3%：active Tab 与 inactive Tab 字色 / 字重 / 底部 indicator 至少 2 维差异，TabIndicator 位移可见"
  2. "LockedView ↔ NormalFormView 整屏切换截图像素差 ≥ 50%（两个完全不同的子树，含锁形 icon + 倒计时数字 + 找回密码链接）"
  3. "LockedCountdown 倒计时数字字号 ≥ 28px + 字体含 monospace 等宽降级（避免数字变化时 layout shift），辅以柔和而非冷峻的视觉"
  4. "所有状态切换 transition ≥ 200ms（normal 250ms 缓动），非 fast 150ms 一晃而过"
  5. "SubmitBtn loading 态显示 spinner + GetCodeBtn counting 态显示倒计时秒数（'58s'），均与 default 态像素差 ≥ 3%"
- **forbiddenSignals**:
  - "loginMode 切换无任何视觉变化（像素差 < 1%，TabIndicator 缺失）"
  - "LockedCountdown 字号 < 24px 或非等宽（数字闪跳）"
  - "LockedView 视觉冷峻（无任何安抚元素，像 404 错误页）"
  - "状态切换无 transition（瞬间切换）"
  - "SubmitBtn loading 与 default 视觉无差（看不出在加载）"

## 4. 装饰系统策略（必读）

选定装饰系统：`soft-glow`

**必须读取**：`.codebuddy/skills/design-planner-v3/references/recipes/decoration-systems/soft-glow.md`

**关键策略**：
1. **视觉特征**：圆形 / 椭圆 / 不规则圆润 blob，填充为径向渐变（中心实色 → 边缘透明），色调为单色或近似色，透明度 opacity 0.2-0.6
2. **CSS 配方**：
   ```jsonc
   {
     "position": "absolute",
     "top": "-40px",
     "right": "-60px",
     "width": "200px",
     "height": "200px",
     "backgroundColor": "$token:colors.primaryLight",
     "backgroundImage": "radial-gradient(circle, currentColor 0%, transparent 70%)",
     "color": "$token:colors.primaryLight",
     "opacity": "0.4",
     "borderRadius": "9999px",
     "pointerEvents": "none",
     "zIndex": "0"
   }
   ```
3. **多光斑组合**：推荐 1-2 个光斑（节制），右上角溢出 + 左下角溢出（对角呼应），光斑大小 ≥ 150px，第 2 个光斑透明度 ≤ 第 1 个的 0.7
4. **光斑色规则**：visualConcept 主色调为"暖白 + 蓝紫强调" → 光斑色用 primaryLight（蓝紫淡）；禁止光斑用纯 primary（太重）或 error/warning 等语义色（误导）
5. **红线**：
   - ❌ 光斑用字符串内嵌 token（`background: "...$token..."`）→ 渲染失败
   - ❌ 光斑硬编码 #xxx → 必须 token 引用
   - ❌ 光斑 ≥ 3 个（视觉爆炸）
   - ❌ 光斑 opacity > 0.7（喧宾夺主）
   - ❌ 同屏混用 soft-glow 与其他装饰系统

1. **装饰系统单一族**：选定 `soft-glow` 后，禁止混用 `geometric-line` / `illustration` / `texture` / `organic-curve`

4. **截图对账**：每个 craft 任务完成后必须截图，逐条核对 successCriteria，不达标不进入下一个任务
5. **verbose 拒绝**：禁止"放字母/emoji 凑数"，所有素材必须按 `decoration-systems/soft-glow.md` 策略设计
