# G2 (P0) — cta-clarity 拆解

> 目标：用户进入页面后，必须在 0.5 秒内识别"我要点哪个按钮登录"

## Step1 — 涉及元素扫描

全屏节点遍历，问"这个节点改动会影响 G2 的 successCriteria 吗？"

| 节点 | 角色 | 是否涉及 | 理由 |
|---|---|---|---|
| SubmitBtn | 主角 | ✅ | CTA 是核心，必须主角化 |
| GetCodeBtn | 邻居 | ✅ | 获取验证码按钮，必须明显弱于提交按钮 |
| Links | 邻居 | ✅ | "忘记密码"等链接，必须弱化不抢戏 |
| FormCard | 邻居 | ✅ | 卡片容器，提供视觉边界但不抢戏 |
| PhoneField | 邻居 | ⚠️ 可选 | 输入框与 CTA 的视觉层级差 |
| CredentialField | 邻居 | ⚠️ 可选 | 同上 |

**最终 involvedElements**（4 个核心）：
`["SubmitBtn", "GetCodeBtn", "Links", "FormCard"]`

## Step5 — 协同关系明确化

```jsonc
coordination: {
  主角: "SubmitBtn",
  邻居: ["GetCodeBtn", "Links", "FormCard"]
}
```

**协同模式**：cta-clarity（CTA + 周围弱化邻居）
- SubmitBtn 主角化（主色填充 + 大字号 + 字重 600 + 阴影）
- GetCodeBtn 弱化（幽灵按钮 + 小字号 + 次字色）
- Links 弱化（caption 字号 + 次字色 + 常规字重）
- FormCard 提供边界但不抢戏（弱阴影 + 浅边框）
