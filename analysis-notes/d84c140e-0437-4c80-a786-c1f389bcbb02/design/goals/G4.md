# G4 (P1) — state-feedback 拆解

> 目标：用户每个操作必须有明确的视觉反馈，不能"点了没反应"

## Step1 — 涉及元素扫描

全屏节点遍历，问"这个节点改动会影响 G4 的 successCriteria 吗？"

| 节点 | 角色 | 是否涉及 | 理由 |
|---|---|---|---|
| ModeToggle | 主角 | ✅ | Tab 切换指示器，状态反馈核心 |
| LockedView | 主角 | ✅ | 锁定态视觉，区别于正常态 |
| NormalFormView | 邻居 | ✅ | 正常态表单，与锁定态对比 |
| Countdown | 邻居 | ✅ | 倒计时数字变化，可见非静止 |

**最终 involvedElements**（4 个）：
`["ModeToggle", "LockedView", "NormalFormView", "Countdown"]`

## Step5 — 协同关系明确化

```jsonc
coordination: {
  主角: ["ModeToggle", "LockedView"],
  邻居: ["NormalFormView", "Countdown"]
}
```

**协同模式**：state-feedback（状态切换前后联动）
- ModeToggle: Tab 切换有指示器（下划线/背景色变化）
- LockedView + NormalFormView: 锁定态与正常态视觉明显区别（警告色边框 + 显示/隐藏切换）
- Countdown: 数字变化可见（大字号 + 主色 + 等宽数字 + 脉冲动效）
