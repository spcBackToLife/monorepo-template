# 项目总览 — 校园社交-登录页

## 1. 产品定位四要素

### 1.1 一句话定位

**面向在校大学生的校园社交 App 的"登录入口"** —— 用户用手机号免密 / 密码两种方式快速进入校园社交空间，是校园社交产品的第一触点。

**注**：本项目交付范围仅"登录页 (00-login)"单屏；登录后跳转的主屏 / 注册页 / 找回密码页本期不实现（用 nav.go 跳到占位 screenId 即可，由后续项目接力）。

### 1.2 核心价值

校园社交的"门槛屏"决定了用户对产品**简约时尚**的第一印象：
- **极致克制的视觉负担**——一屏内只放真正的关键决策点（手机号 / 凭证 / 登录），不堆功能
- **校园温度感**——通过几何装饰 + 标语暗示"这是大学生的社交场，不是冰冷的工具"
- **现代时尚感**——大字号品牌强调字 + 圆角输入框 + 极细线 + 单一强调色，避开土味渐变与卡通

### 1.3 目标用户

- **Primary**：18-22 岁在校本专科生
  - 场景：开学季 / 课间 / 宿舍下载 App 后第一次进入；老用户换机后重新登录
  - 痛点：不想记密码（手机短信免密）；不想看到广告/弹窗（开屏简洁）；对"学生范儿"敏感
  - 替代方案：微信 / QQ（关系链已固化）、其他校园社交 App（往往设计土）

- **Secondary**：23-26 岁研究生 / 应届毕业生
  - 场景：续用产品到工作初期
  - 痛点：希望"看起来不像学生玩具"，对设计审美敏感
  - 关键：风格必须"简约时尚"而不是"卡通可爱"，覆盖到这一层

### 1.4 使用场景

| 频次 | 场景 | 设备 | 网络 | 情绪 |
|------|------|------|------|------|
| 高频（首次） | 下载 App 后第一次打开 / 注册后首次登录 | 手机（iPhone / Android 中高端为主） | Wi-Fi / 4G-5G | 期待 + 略有焦虑（陌生产品） |
| 中频 | 换机 / 清缓存 / token 失效后重新登录 | 手机 | 任意 | 略不耐（已是老用户，希望快速过） |
| 低频 | 切换账号 / 忘记密码后回登录页 | 手机 | 任意 | 焦躁（出问题了） |

**关键洞察**：
- 首次场景"略有焦虑"——风格必须友好、降低戒备（圆角 + 留白 + 校园元素暗示）
- 中低频场景"求快"——首屏 1 秒内能找到手机号输入框；手机号已记忆自动填充
- 弱网兜底必须有（4G/5G 切换、地铁场景）
- 键盘弹起时主 CTA 不能被遮挡（粘底按钮）

## 2. 领域识别推理

### 2.1 产品类型判定

**社交类**——校园社交 App 的入口屏。按"4 类产品速查表"，社交类必有：用户体系 / 内容体系 / 社交体系 / 通知体系 / 搜索体系 / 安全体系 / 设置体系。

本项目作用域仅 **用户体系 → 登录子模块 → 登录屏**。

### 2.2 必有模块清单（含主动挖掘的隐含模块）

| 模块 ID | 名称 | 优先级 | 一句话 | 用户提及 / AI 主动挖掘 |
|--------|------|:-----:|--------|---------------------|
| M1 | 用户认证-登录 | **P0** | 手机号免密 + 密码两种登录入口（本项目唯一交付） | 用户提及（核心诉求） |
| M2 | 用户认证-注册 | P3 | 新用户注册流程 | 主动挖掘（登录页必有"注册账号"出口；本期跳占位屏） |
| M3 | 用户认证-找回密码 | P3 | 密码登录的找回入口 | 主动挖掘（登录页必有"忘记密码？"出口；本期跳占位屏） |
| M4 | 主屏（登录后落地） | P3 | 登录成功后跳转的首屏 | 主动挖掘（nav.go 目标必须存在；本期跳占位屏） |
| M5 | 协议体系 | P0 | 用户协议 + 隐私协议（合规必需） | 主动挖掘（toC App 合规红线，登录页必须勾选） |
| M6 | 安全体系-防刷 | P0 | 短信冷却 60s / 日限 / 密码错误锁定 | 主动挖掘（用户没说，但短信网关 / 账号安全必备） |

### 2.3 主动挖掘理由

- **M2/M3/M4**：登录屏的"注册账号"、"忘记密码？"、"登录成功跳哪"是登录页必有的 3 个出口；本期不实现这些目的地，但骨架必须留出入口 + nav.go 写到占位 screenId（让 interaction 阶段有目标可跳）
- **M5 协议**：toC App 合规红线，未勾选不可提交；登录页必须有勾选框 + 协议文本链接
- **M6 防刷**：短信网关成本 + 撞库攻击是登录场景两大现实威胁，必须在规则层固定下来——交给 interaction 阶段实现倒计时 / 锁定状态机

## 3. MVP 分级推理

| 优先级 | 模块 | 切分理由 |
|--------|------|--------|
| **P0** | M1（登录）/ M5（协议）/ M6（防刷规则） | 没它登录页无法交付/合规 |
| P3 | M2（注册）/ M3（找回）/ M4（主屏） | 本项目作用域仅登录页；这些是导航出口的"目的地"，留占位 screenId |

### 3.1 砍掉模块的理由

- **注册 / 找回密码 / 主屏跳转后**：用户明确说"做下登录页面"，作用域单屏。这三个屏跳转用 `nav.go` 占位 screenId（`00-register` / `00-forgot-password` / `01-home`），由后续项目接力。本期不在登录屏内塞替代方案（如"扫码登录"、"刷脸登录"）。


## 5. 风格方向

**简约时尚 + 校园温度** —— 极简留白 + 大字号品牌强调字 + 圆角输入框 + 单一强调色（候选偏紫/偏绿/偏粉，最终由 theme-generator 定）+ 极细几何装饰暗示校园感。

**为什么是这个方向**：
- 反推自 1.3 Secondary 用户的"看起来不像学生玩具"诉求 —— 必须现代感而非卡通
- 反推自 1.4 高频"略有焦虑"情绪 —— 留白 + 圆角降低紧张
- 反推自 1.2 核心价值"极致克制" —— 一屏不堆功能

**避免**：
- ❌ 浓郁渐变 + 多色块（土味）
- ❌ 卡通插画占大半屏（"学生玩具"感）
- ❌ 黑白纯极简（少了校园温度）

## 6. 多角度验证（关键决策）

| 维度 | 结论 |
|------|------|
| 用户角度 | 高频首次场景下"简约 + 友好"双需求 → 风格定为"简约时尚 + 校园温度"，圆角 + 留白 + 暗示性几何装饰 |
| 商业角度 | 校园社交差异化在审美 → 登录页第一印象 = 留存关键；卡通=拉低留学生付费心智，时尚=提升 |
| 技术角度 | 登录页低复杂度（单屏 + 1 API），但有状态机（loginMode / failureCount / lockedUntil / codeCountdown），交给 interaction 处理 |
| 竞品角度 | 现有校园社交 App 普遍设计土，"简约时尚"路线有差异化空间；参考 Notion / Linear 的极简感 + 小红书的圆角友好 |

## 7. 替代方案与否决

- **第三方授权登录（微信 / Apple / QQ）**：否决。本项目仅交付 1 个登录屏，加授权链路扩大作用域；且校园场景下微信授权需多一步获取手机号；本期手机号 + 密码已覆盖 90%+。决策记录到 D1。
- **扫码登录**：否决。Web 端配套能力，移动端登录页不需要。
- **生物识别（指纹 / Face ID）**：否决。属于"快速复登录"增强项，依赖 token 缓存 + 系统 API；P2 增强。
