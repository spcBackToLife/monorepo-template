## 2. B1：validateStyles 拒字符串内嵌 token

### 2.1 背景与症状

设计 AI 经常写：
```jsonc
backgroundImage: "linear-gradient(135deg, $token:colors.primary 0%, $token:colors.secondary 100%)"
```

resolveTokens.ts 当前虽然能 split-by-space 部分支持，但碰到 `linear-gradient(135deg,$token:colors.primary 0%,$token:colors.secondary)`（逗号紧贴无空格）会失败。**最关键的是**：v3 SKILL 教 AI 把渐变写成 GradientDef 结构化对象（见 canvas.set_fill），所以应当从源头禁止字符串内嵌 token。

### 2.2 落点

新建：`features/design-operations/src/utils/style-token-lint.ts`

调用方：
- `features/design-operations/src/operations/style.ts` 的 `executeUpdateStyle` / `executeBatchUpdateStyle`
- `features/design-operations/src/operations/element.ts` 的 add/insertSubtree（节点初始 styles）
- `features/design-operations/src/operations/visual-state.ts` 的 add/update（visualState.styles）

### 2.3 实现

```typescript
// utils/style-token-lint.ts
const TOKEN_PREFIX = '$token:';

/**
 * 校验单个样式值。两条规则：
 *
 *  R-STYLE-TOKEN-01：字符串值要么完全是 `$token:xxx` / `$token:xxx $token:yyy`（纯 token + 空格分隔），
 *                    要么完全没有 `$token:` 子串。中间夹 token 一律拒。
 *
 *  R-STYLE-TOKEN-02：渐变（linear-gradient / radial-gradient）严禁出现 $token: —— 应改用
 *                    canvas.set_fill 的 GradientDef 或 design-engine 的渐变 token 列表（typography.* 等）。
 */
export function lintStyleValue(key: string, value: unknown): { ok: boolean; code?: string; message?: string } {
  if (typeof value !== 'string') return { ok: true };

  const hasToken = value.includes(TOKEN_PREFIX);
  if (!hasToken) return { ok: true };

  // R-STYLE-TOKEN-02：渐变内不准出现 token
  if (/(linear|radial|conic)-gradient\s*\(/i.test(value)) {
    return {
      ok: false,
      code: 'R-STYLE-TOKEN-02',
      message: `styles.${key}：渐变字符串内不能嵌 $token:xxx —— 请用 canvas.set_fill GradientDef，或预先解析后写裸色（hex/rgba）`,
    };
  }

  // R-STYLE-TOKEN-01：要求纯 token 形式
  // 合法：单值 "$token:colors.primary" / 双值（margin 等） "$token:spacing.sm $token:spacing.md"
  const parts = value.trim().split(/\s+/);
  const allTokens = parts.every((p) => p.startsWith(TOKEN_PREFIX));
  if (!allTokens) {
    return {
      ok: false,
      code: 'R-STYLE-TOKEN-01',
      message: `styles.${key}：字符串内嵌 $token: 不允许（"${value}"）—— 整体值要么纯 token + 空格分隔，要么完全裸字面量`,
    };
  }

  return { ok: true };
}

export function lintStyleObject(styles: Record<string, unknown>, nodeId: string) {
  const issues: Array<{ nodeId: string; fieldPath: string; code: string; message: string }> = [];
  for (const [k, v] of Object.entries(styles)) {
    const r = lintStyleValue(k, v);
    if (!r.ok) issues.push({ nodeId, fieldPath: `styles.${k}`, code: r.code!, message: r.message! });
  }
  return issues;
}
```

调用：在 `executeUpdateStyle` 现有 `lintStyleFields` 之后追加调用 `lintStyleObject`，把 issues 合进 result.issues。

### 2.4 测试

```typescript

expect(lintStyleValue('padding', '$token:spacing.sm $token:spacing.md')).toEqual({ ok: true });
expect(lintStyleValue('color', '#ff00aa')).toEqual({ ok: true });

// 拒绝
```

## 3. B4：expectedArtifacts 机器对账升级

### 3.1 背景

v3 plan 任务的 `expectedArtifacts` 需要新 kind 来对账新产物：

| 新需求 | 新 kind |
|--------|--------|
| 节点 styles 的某 key 是 token 引用 | `kind: 'tokenRef', path, key` |
| 节点 visualStates 数组含特定 name | `kind: 'hasVisualState', path, name` |
| 节点子树的某层级深度有非空 children | `kind: 'subtreeDepth', path, min, max` |
| 节点的 materialProjectId 已绑定 | （已存在 hasKeys 即可） |

### 3.2 落点

文件：`features/design-schema/src/integrity/verify-artifact.ts`
类型：`features/design-schema/src/types/meta.ts` 的 `ArtifactCheck` 联合类型

### 3.3 改造

```typescript
// types/meta.ts
type ArtifactCheck =
  | { kind: 'nonEmpty'; path: string; message?: string }
  | { kind: 'arrayMin'; path: string; min: number; message?: string }
  | { kind: 'hasKeys'; path: string; keys: string[]; message?: string }
  | { kind: 'eachItem'; path: string; check: ArtifactCheck; message?: string }
  | { kind: 'anyNodeHasEvents'; path: string; min?: number; message?: string }
  | { kind: 'nodeHasEvent'; nodeId: string; trigger: string; min?: number; path?: string; message?: string }
  // ↓ v3 新增
  | { kind: 'tokenRef'; path: string; styleKey: string; message?: string }
  | { kind: 'hasVisualState'; path: string; stateName: string; message?: string }
  | { kind: 'subtreeNonEmpty'; path: string; minDepth?: number; message?: string };

// verify-artifact.ts 新增 case
case 'tokenRef':
  return checkTokenRef(root, check.path, check.styleKey, check.message);
case 'hasVisualState':
  return checkHasVisualState(root, check.path, check.stateName, check.message);
case 'subtreeNonEmpty':
  return checkSubtreeNonEmpty(root, check.path, check.minDepth ?? 1, check.message);

function checkTokenRef(root: unknown, path: string, key: string, message?: string): ArtifactVerifyResult {
  const values = resolvePath(root, path);
  for (const v of values) {
    const styles = (v as any)?.styles;
    if (!styles || typeof styles[key] !== 'string') {
      return { ok: false, failedPath: `${path}.styles.${key}`, detail: message ?? `styles.${key} 未设置` };
    }
    if (!styles[key].includes('$token:')) {
      return { ok: false, failedPath: `${path}.styles.${key}`,
               detail: message ?? `styles.${key}="${styles[key]}" 未使用 token 引用` };
    }
  }
  return { ok: true };
}

function checkHasVisualState(root: unknown, path: string, stateName: string, message?: string): ArtifactVerifyResult {
  const values = resolvePath(root, path);
  for (const v of values) {
    const states = (v as any)?.visualStates;
    if (!Array.isArray(states) || !states.some((s: any) => s?.name === stateName)) {
      return { ok: false, failedPath: `${path}.visualStates`,
               detail: message ?? `visualStates 中未找到 name="${stateName}"` };
    }
  }
  return { ok: true };
}
```

### 4.2 落点

### 4.3 改造

```typescript
// theme/validation.ts 加常量
export const ALLOWED_THEME_INTENT_TONES = [
  'minimal', 'trustworthy', 'warm',
  'playful', 'premium', 'clean',
  'bold', 'natural',
] as const;
export type ThemeIntentTone = typeof ALLOWED_THEME_INTENT_TONES[number];

export const ALLOWED_DECORATION_SYSTEMS = [
  'soft-glow', 'geometric-line', 'illustration', 'texture', 'organic-curve',
  'none',
] as const;

// validateThemeConfig 加 R-THEME-11 / R-THEME-12
function validateIntentTone(theme: ThemeDefinition): string[] {
  const errors: string[] = [];
  if (!theme.intent?.tone) {
    errors.push(`R-THEME-11: theme.intent.tone 缺失`);
  } else if (!(ALLOWED_THEME_INTENT_TONES as readonly string[]).includes(theme.intent.tone)) {
    errors.push(`R-THEME-11: theme.intent.tone="${theme.intent.tone}" 不在枚举中（${ALLOWED_THEME_INTENT_TONES.join('/')}）`);
  }
  // 装饰系统校验
  const sys = theme.decorationRules?.system;
  if (sys && !(ALLOWED_DECORATION_SYSTEMS as readonly string[]).includes(sys)) {
    errors.push(`R-THEME-12: decorationRules.system="${sys}" 不在枚举中`);
  }
  return errors;
}
```

## 5. B2：4 画面对账 query

### 5.1 背景

v3 design 阶段最大盲区：**design 自己看不到效果**。当前 query 工具只能拿 schema 字段，不能直接看到"画面是不是真的协调"。

需要 4 个新 query action 让 design 自审：

| action | 用途 | 数据来源 |
|--------|------|--------|
| `query.weight_pyramid` | 列出当前屏所有节点的视觉权重打分（fontSize+fontWeight+color+size+position 综合） | rootNode 树 + computed styles |
| `query.color_ratio` | 当前屏 60-30-10 实际占比（按节点 size 加权） | rootNode 树 + theme tokens |
| `query.decoration_density` | 装饰节点密度（meta.design.kind='decoration' 计数 / 屏 area） | rootNode 树 |
| `query.visualstate_coverage` | 每个交互节点的 visualState 覆盖率（hover/pressed/disabled 是否齐） | rootNode 树 |

### 5.2 落点

文件：`apps/design-mcp/src/tools/domain/query-grouped.ts`

新建：`features/design-engine/src/audit/`（核心算法）— 让算法 reusable

### 5.3 实现思路（weight_pyramid 为例）

```typescript
// features/design-engine/src/audit/weight.ts
export interface WeightScore {
  nodeId: string;
  nodeName: string;
  weightScore: number;       // 0-100
  factors: { fontSize: number; fontWeight: number; colorContrast: number; size: number; zIndex: number };
}

export function computeWeightPyramid(screen: Screen, theme: ThemeConfig): WeightScore[] {
  // 1. 遍历 rootNode 所有 leaf
  // 2. 解析 styles（含 visualState='default' 与 token resolution）
  // 3. 综合打分:
  //    fontSizeScore = (fontSize - 12) * 2  (12px=0, 24px=24, 48px=72)
  //    weightScore   = fontWeight >= 600 ? 20 : 0
  //    colorScore    = contrastRatio(color, bgPage) > 4.5 ? 15 : 5
  //    sizeScore     = log(width * height) * 5
  //    zIndexScore   = (zIndex || 0) * 2
  //    total = sum / 100
  // 4. sort desc 返回前 N 个
}
```

```typescript
// query-grouped.ts 加 action
case 'weight_pyramid':
  const weights = computeWeightPyramid(screen, theme);
  return ok({
    items: weights.slice(0, 10),
    interpretation: weights[0].weightScore - weights[1].weightScore > 30
      ? '✅ 主次清晰（主体比次主体高 30+）'
      : '⚠️ 主次模糊，主体不够突出',
  });
```

## 8. 风险与回滚

| 改造 | 风险 | 回滚策略 |
|------|------|---------|
| B5 | meta 字段名打架（briefing 已被某个外部工具占用） | meta.setScreen `mode='replace'` 时仅 strict reject 新字段；deepmerge 模式向前兼容 |
| B1 | 历史 schema 含 string-embedded token | lint 仅在 op 入口生效；批量迁移脚本扫一遍 schemas DB,把已有的 `linear-gradient(...,$token:...)` 替换为 GradientDef |
| B4 | 老任务 expectedArtifacts 仍用旧 kind | 新 kind 可选；SKILL 引导用新的,旧的继续工作 |
| B6 | 历史 theme intent.tone 是 "professional" 等老值 | 加 deprecation warning（不报错），SKILL 文档说明迁移到 8 枚举 |
