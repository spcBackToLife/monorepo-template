# 绑定请求事件 + 页面跳转 + 提示 Toast 展示
目前已经完成了登陆页面的设计，我想预览的时候点击登陆按钮的：
- 可以看到失败的时候，错误提示 toast
- 如果成功了，有成功提示的 toast，并且跳转到一个新的页面：主页
这就要求我们在产品设计上得思考：
- 如何给按钮绑定一个请求的事件（我看有个自定义事件是这个么？还是要设计一个），这个请求的事件应该是怎么设计，要完整的描述吗？
比如，请求方式、路径、参数、请求头等等信息，为后续设计稿还原代码作为核心组成部分；但是在预览的时候肯定不是真实的发请求，只是基于请求路径和
方式进行的mock，当让看起来我们得设计一个请求管理的模块，根据请求方式和路径，给予接口mock 不同场景的数据，并可以灵活的开关某个返回情况，能够
在预览的时候，看到成功和失败的不同表现。
