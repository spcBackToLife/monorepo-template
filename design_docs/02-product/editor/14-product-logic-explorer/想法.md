# 产品全景
我想要的产品全景是这样的：
- 交互图都是有渲染展示的，并且有连线比如：主页（渲染主页） -> 登陆（渲染登陆按钮）-> 成功（主页成功状态）等等这样绘制的所有完整交互图，包含一个页面的所有交互分叉
- 单个页面的功能纬度描述应该如：登陆页面
   - 顶部
   - 表单
   - 提交
   - 登陆渠道
  这几个模块，每个模块分别详细描述，如果是顶部的，就是有图标、标题、副标题三个元素，用表格描述：元素、描述、截图，然后如果是表单区域：
  - 则需要对每个字段详细描述所有逻辑：label、placeholder、错误验证逻辑提示文案、教研逻辑等等交互细节功能点，每个功能点都是一行描述。
  - 提交按钮也是：登陆、成功提示、失败提示等等
  都应该是详细的产品描述才对，任何 schema 上的所有细节表达，包括全局状态、事件、样式、显示与隐藏等等所有细节，都应该从产品视角表达成产品语言，沉淀下来。

  我想要的不是敷衍了事的页面，而是结合设计数据完整的产品方案表达，图文并貌的，你之前实现的就是走个过场，是他吗给人看的方案么？？
  结合第一性原理仔细分析下这些需求，要从资深产品的视角出发进行分析，我想要的是什么，要满足什么产品诉求，怎么达到效果，怎么保证极致的交互体验。
