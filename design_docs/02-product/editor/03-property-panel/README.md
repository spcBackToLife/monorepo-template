## 一、第一性原理：属性面板解决什么根本问题？

### 1.1 本质推导

```
画布解决了"看到" Schema 的问题
属性面板解决"精确修改" Schema 的问题

画布上的操作是粗粒度的：拖拽移动、resize 调尺寸
属性面板的操作是精确的：padding 从 8 改到 12、字号从 16 改到 14、绑定一个 click 事件

Schema 有五个维度：
  结构 → 左侧组件树管（改父子关系、增删节点）
  样式 → 属性面板管（CSS 属性的精确控制）
  交互 → 属性面板管（事件绑定）
  状态 → 属性面板管（状态定义、样式覆盖、状态切换）
  数据 → 属性面板管（数据集管理、数据绑定表达式）

结论：属性面板承载了五维中的四维精确控制 → 它是编辑器中信息密度最高的区域。
```

### 1.2 设计挑战

```
四维信息 × N 种元素类型 = 巨大的信息量

如果全部平铺 → 面板会非常长，用户找不到想要的控件
如果过度折叠 → 操作变多，效率降低

解法：Tab 分维度 + 分组折叠 + 上下文裁剪

Tab 分维度：5 个 Tab 按维度分离（一次只看一个维度）
分组折叠：每个 Tab 内按逻辑分组，可折叠/展开
上下文裁剪：只显示当前元素类型相关的控件
  · 选中 div → 样式 Tab 不显示 Typography 组
  · 选中 img → 属性 Tab 显示 src / alt / loading
  · 选中 button → 属性 Tab 显示 type / disabled
```

### 1.3 设计原则

```
1. CSS 原名 → 所有样式属性直接用 CSS 属性名，不发明新术语
2. 即时反馈 → 修改任何值 → 画布实时更新（blur/Enter 提交 Operation）
3. 上下文敏感 → 面板内容根据选中元素类型动态变化
4. 可发现性 → 所有可编辑的维度都通过 Tab 暴露，不藏在二级菜单
5. 专业与效率并重 → 常用属性直接可见，高级属性折叠但可展开
```

### 2.2 元素信息条

无论在哪个 Tab，顶部始终显示当前选中元素的基本信息：

```
┌──────────────────────────────────────┐
│  🏷 div.header      │  [🔒] [👁] [⋮] │
│  ↑类型  ↑名称(可编辑)│  锁定 可见 更多│
└──────────────────────────────────────┘

  类型标签 (🏷):
    · 根据 NodeType 显示：div / span / button / img / component:LoginForm ...
    · 组件实例额外显示 📦 图标 + 模板名

  名称 (可编辑):
    · 来自 ComponentNode.name
    · 双击进入编辑 → blur/Enter 提交
    · 若未命名 → 显示灰色占位 "未命名"

  快捷操作:
    · [🔒] → 切换锁定状态 (locked: true/false)
    · [👁] → 切换可见性 (visible: true/false)
    · [⋮]  → 更多菜单: 复制 / 删除 / 包裹为容器 / 保存为组件资产 ...

  多选时:
    · 显示 "已选择 3 个元素"
    · 锁定/可见操作同时应用于所有选中元素
```

### 2.3 Tab 栏设计

```
┌────────────────────────────────────────────┐
│  [样式]  [属性]  [交互]  [状态]  [数据]     │
│   ━━━                                       │  ← 当前 Tab 下方有蓝色下划线

Tab 栏规范:
  · 高度: 36px
  · 字号: 13px
  · 当前 Tab: 蓝色文字 (#0D99FF) + 蓝色 2px 底部下划线
  · 其他 Tab: 灰色文字 (#999)

  · 交互 Tab: 如果该元素已绑定事件 → 显示蓝色小圆点

## 三、样式 Tab — CSS 可视化编辑器

这是使用频率最高的 Tab，占编辑时间的 60% 以上。

### 3.1 整体结构

样式 Tab 由 8 个属性组组成，每组可折叠：

```
┌──────────────────────────────────────┐
│  样式                                 │
├──────────────────────────────────────┤
│                                      │
│  ▸ 布局 Layout ──────────────────── │
│  ▸ 尺寸 Size ────────────────────── │
│  ▸ 间距 Spacing ─────────────────── │
│  ▸ 定位 Position ────────────────── │
│  ▸ 背景 & 填充 ──────────────────── │
│  ▸ 边框 Border ──────────────────── │
│  ▸ 文字 Typography ──────────────── │
│  ▸ 效果 Effects ─────────────────── │
│                                      │
└──────────────────────────────────────┘

折叠规则:
  · 有值（非默认）的组 → 默认展开
  · 无值（全默认）的组 → 默认折叠
  · 用户手动折叠/展开 → 记忆该状态

智能隐藏:
  · 选中 img / input 等非容器元素 → 隐藏 "布局" 组（它没有 children）
  · 选中 div / section 等非文本元素 → "文字" 组默认折叠（但可展开，因为 div 可以有文字样式影响子元素）
```

## 四、属性 Tab — 元素 Props 编辑

### 4.1 设计思路

属性 Tab 管理的是 `ComponentNode.props`，即元素的 **非样式属性**。它的内容完全由选中元素的 `type` 决定。

```
属性 Tab 的内容 = f(node.type)

  选中 div   → 几乎无内容（div 没什么 props）
  选中 img   → 显示 src / alt / loading
  选中 input → 显示 type / placeholder / maxLength / disabled
  选中 component:LoginForm → 显示该组件模板暴露的 propDefinitions
```

### 4.2 整体结构

```
┌──────────────────────────────────────┐
│  属性                                 │
├──────────────────────────────────────┤
│                                      │
│  元素类型  [button ▾]  name: [提交]   │
│                                      │
│  ▸ 元素属性                           │
│    （按 node.type 动态渲染）          │
│                                      │
│  ▸ 组件 Props                        │
│    （仅组件实例可见）                  │
│                                      │
│  ▸ 自定义属性 (data-*)               │
│    （高级功能，手动添加任意 HTML attr）│
│                                      │
└──────────────────────────────────────┘
```

### 4.3 元素类型切换

```
元素类型 [button ▾]

  · 下拉可切换当前节点的 type
  · 切换后:
    - Schema 的 node.type 更新
    - 已有 props 中不适用的字段自动清除
    - 新 type 的默认 props 自动填充
  · 生成 changeElementType Operation

  可切换范围:
    · 块级元素: div / section / nav / header / footer / main
    · 行内元素: span / a
    · 文本元素: p / h1 / h2 / h3
    · 表单元素: button / input / textarea / select

## 九、统一输入控件设计规范

### 9.1 数值输入框

属性面板中最常见的控件，需要统一规范：

```
┌────────────────────────────────────────────────────┐
│  数值输入框的交互规范                                 │
├────────────────────────────────────────────────────┤
│                                                    │
│  外观:  [200] px                                   │
│         ↑数值  ↑单位标签                            │
│  宽度: 64px (数值框) + 24px (单位标签)              │
│  高度: 28px                                        │
│  边框: 1px solid #E5E5E5 → focus 时 #0D99FF       │
│  圆角: 4px                                         │
│                                                    │
│  输入行为:                                          │
│  · 聚焦时: 全选文本（方便直接输入新值）              │
│  · 输入纯数字 "200" → 自动补全为 "200px"            │
│  · 输入带单位 "50%" → 使用该单位                    │
│  · 输入表达式 "calc(100% - 20px)" → 原样存储       │
│  · 输入 "auto" → 设为 auto                         │
│                                                    │
│  提交时机:                                          │
│  · blur (失去焦点) → 提交                           │
│  · Enter → 提交 + 焦点移到下一个输入框              │
│  · Escape → 取消修改，恢复原值                      │
│  · 不逐字符提交（避免每打一个数字都触发 Operation）  │
│                                                    │
│  拖拽调值:                                          │
│  · 在数值标签上按住拖拽（左右）→ 连续调整值          │
│  · 默认步长: 1px                                    │
│  · 按住 Shift: 步长 10px                            │
│  · 按住 Alt: 步长 0.1px                             │
│  · 拖拽时实时更新画布（临时应用，松手后提交 Op）     │
│                                                    │
│  上下键:                                            │
│  · ↑ → 值 +1                                       │
│  · ↓ → 值 -1                                       │
│  · Shift + ↑↓ → ±10                                │
│  · Alt + ↑↓ → ±0.1                                 │
│                                                    │
└────────────────────────────────────────────────────┘
```

### 10.1 画布 (01-canvas) → 属性面板

```typescript
// 选中变化时画布通知属性面板
interface SelectionChangeEvent {
  selectedNodeIds: string[];
  selectedNodes: ComponentNode[];
}

// 属性面板订阅选中状态
function onSelectionChange(event: SelectionChangeEvent) {
  if (event.selectedNodes.length === 0) {
    showPageProperties();      // 无选中 → 显示页面属性
  } else if (event.selectedNodes.length === 1) {
    showNodeProperties(event.selectedNodes[0]);  // 单选
  } else {
    showMultiSelectProperties(event.selectedNodes); // 多选
  }
}
```

## 十一、边界情况与异常处理

| 场景 | 预期行为 |
|------|---------|
| 选中元素被远程删除（AI 操作）| 面板自动切到无选中状态，不报错 |
| 输入非法 CSS 值（如 width: "abc"）| 输入框边框变红 + 提示 "无效的值"，不提交 Operation |
| 输入超长文本（如 placeholder 100 字）| 允许，输入框横向不滚动（截断显示），完整值在 tooltip 中 |
| 数据绑定路径不存在（如 data.xxx）| 画布显示 `{{data.xxx}}`原始文本 + 绑定框显示 ⚠️ 警告 |
| 组件实例的 props 被模板更新 | 引用模式下自动同步新 props 定义；脱离模式下不受影响 |
| 面板宽度拖拽到极窄（< 280px）| 限制最小宽度 280px |
| 快速切换选中元素 | 面板更新做 debounce（50ms），避免频繁闪烁 |
| 同时修改多个属性（复制粘贴样式）| 批量操作合并为一个 Operation（batchUpdateStyle） |
| JSON 数据集格式错误 | 编辑器实时校验 + 红色波浪线标记错误位置，[应用] 按钮置灰 |
| Tab 徽标更新延迟 | 事件/状态/数据绑定增删后立即更新徽标，不等渲染完成 |

## 十三、核心设计决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| Tab 数量？ | 5 个 (样式/属性/交互/状态/数据) | 1:1 映射 Schema 的五个维度，认知负担最低 |
| 样式属性名？ | 直接用 CSS 属性名（不翻译、不抽象） | 设计师和开发者都熟悉 CSS；不造概念 |
| 值提交时机？ | blur / Enter（不逐字符） | 避免输入过程中频繁触发 Operation，减少噪音和性能消耗 |
| 多选时哪些 Tab 可用？ | 样式 Tab（合并模式）；属性/交互/状态/数据不可用 | 多选修改样式是高频操作；交互/状态/数据是单元素概念 |
| 间距编辑方式？ | 可视化盒模型 | 直观、零学习成本、所见即所得；Figma/Chrome DevTools 验证过的方案 |
| 状态编辑联动？ | 状态 Tab 选中状态 → 样式 Tab 切换到覆盖模式 | 不需要在两个面板来回复制样式；一处选状态，一处改样式 |
| 组件 Props 放哪？ | 属性 Tab 的"组件 Props"区域 | Props 是属性的一部分，Tab 语义一致 |
| 数据绑定输入方式？ | 文本输入 + 路径自动补全 | 比可视化连线更高效；自动补全降低记忆负担 |
| 颜色选择器？ | HSB 拾取器 + HEX + 项目色板 | 标准方案，覆盖绝大多数场景 |
