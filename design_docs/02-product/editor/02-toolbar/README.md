## 一、第一性原理：工具栏解决什么根本问题？

### 1.1 本质推导

```
编辑器操作的频率分布是极其不均匀的：
  · 80% 的时间 → 选中、移动、调整尺寸（选择工具）
  · 15% 的时间 → 创建新元素（绘制工具）
  · 5% 的时间  → 全局控制（缩放、视口切换、撤销、预览等）

工具栏的本质 = 把这些高频操作放在"0 次思考、1 次点击"的位置

进一步拆分：
  操作按"范围"分两类：
  · 局部操作 → 作用于画布上的具体元素 → 底部工具栏（手在下面，靠近画布）
  · 全局操作 → 作用于整个编辑器状态   → 顶部工具栏（信息条，一眼可见）
```

### 1.2 Figma 的工具栏哲学（我们的借鉴与差异）

| 维度 | Figma | 我们的差异 |
|------|-------|-----------|
| 底部工具栏 | 几何绘制为主（矩形/椭圆/线段/箭头/钢笔） | **语义元素为主**（容器/方块/组件/文本）—— 因为我们画的是 DOM 结构，不是矢量图形 |
| 顶部工具栏 | 项目名 + 协作按钮 + 分享 | **增加全局状态切换 + 数据集切换** —— 我们的 Schema 是数据驱动的 |
| 下拉子菜单 | 几何变体（矩形/椭圆/多边形/星形） | **语义变体**（Flex 容器 / Grid 容器 / 绝对容器）和（h1 / h2 / h3 / p / span） |
| 代码视图 | 无（需要插件） | **内置代码视图** —— Schema 到代码的映射是核心能力 |

### 1.3 设计原则

```
1. 零认知负担 → 图标即含义，看到就知道是什么
2. 快捷键全覆盖 → 每个底部工具都有单键快捷键（V/F/R/T/C）
3. 上下文适应 → 选中不同元素时，工具栏选项动态调整
4. 最小干扰 → 工具栏不遮挡画布核心区域
5. 记忆性 → 下拉菜单记住上次选择（如"容器"记住用户上次选的 Flex 还是 Grid）
```

## 二、底部工具栏

### 2.1 整体布局

参考 `底部工具栏.png`（Figma 风格），底部工具栏采用浮动胶囊式设计，水平居中于画布底部：

```
                    底部工具栏（浮动在画布底部，圆角胶囊）
┌───────────────────────────────────────────────────────────────────┐
│                                                                   │
│  [↖ ▾]  [# ▾]  [□ ▾]  │  [⊞]  [T]  [💬]  [💾]  [🖼]  │  [</>]   │
│   ↑       ↑      ↑     │   ↑     ↑     ↑     ↑    ↑   │    ↑      │
│  选择    容器   元素    │ 组件库 文本  注释  存模板 素材 │  代码     │
│                         │                            │           │
│     编辑工具区           │        内容工具区           │ 开发工具区 │
│                         │                            │           │
└───────────────────────────────────────────────────────────────────┘

  · 三个区域用竖线分隔符分开
  · 当前激活的工具高亮（蓝色背景），参考 Figma 的选择工具高亮
  · 带 ▾ 的工具点击图标激活默认子项，长按或点击 ▾ 展开下拉
  · 胶囊外形：白色背景 + 阴影 + 圆角(12px)，悬浮于画布上方
```

### 2.3 各工具详解

#### 编辑工具区

##### ↖ 选择工具

```
图标: ↖ (指针箭头)
快捷键: V
下拉菜单: 无（单功能）

行为:
  · 默认工具 —— 编辑器启动时激活
  · 单击画布元素 → 选中
  · 拖拽已选中元素 → 移动
  · 拖拽空白区域 → 框选
  · 双击元素 → 深层选择（进入子元素）
  · 拖拽 resize handle → 调整尺寸
  · 其他绘制工具完成创建后 → 自动切回选择工具

光标:
  · 空闲: 默认箭头
  · hover 元素: 箭头（不变）
  · 拖拽移动中: move 十字
  · hover resize handle: 对应方向的 resize 光标 (nw-resize, n-resize, 等)
  · 框选中: crosshair
```

##### # 容器工具

```
图标: # (井号/网格图标)
快捷键: F
下拉菜单:
  ┌──────────────────────────┐
  │  ✓  Flex 容器          F │  ← 默认选中
  │     Grid 容器            │
  │     绝对定位容器          │
  └──────────────────────────┘

行为:
  · 在画布上拖拽画矩形 → 创建对应类型的容器 div
  · 点击工具图标 → 激活上次使用的容器子类型（记忆性）
  · 创建完成后 → 自动切回选择工具

创建的 Schema 节点:
  Flex 容器:   div + display:flex; background:#F5F5F5;
  Grid 容器:   div + display:grid; background:#F5F5F5;
  绝对定位容器: div + position:relative; background:#F5F5F5;

光标: 十字准星 (crosshair)
```

##### □ 元素工具

```
图标: □ (方块)
快捷键: R
下拉菜单:
  ┌────────────────────────────┐
  │  ✓  div                 R │  ← 默认
  │     section               │
  │     nav                   │
  │     header                │
  │     footer                │
  │     main                  │
  └────────────────────────────┘

行为:
  · 在画布上拖拽画矩形 → 创建对应语义的 HTML 元素
  · 区别于"容器工具"：元素工具创建的是通用块级元素，不自带 flex/grid

创建的 Schema 节点:
  div:      div + position:absolute; background:#E5E5E5;
  section:  section + position:absolute;
  nav:      nav + position:absolute;
  (等等，语义不同但默认样式类似)

光标: 十字准星 (crosshair)
```

#### 内容工具区

##### ◇ 组件工具

```
图标: ◇ (菱形)
快捷键: 无（通过下拉选择具体组件）
下拉菜单:
  ┌────────────────────────────┐
  │  常用 HTML 交互元素         │
  │  ─────────────────────────│
  │     button   按钮          │
  │     input    输入框         │
  │     textarea 多行输入       │
  │     select   下拉框         │
  │     img      图片     ⇧⌘K │
  │     a        链接          │
  │     video    视频          │
  └────────────────────────────┘

行为:
  · 选择具体元素类型后
  · 在画布上点击 → 在点击位置创建（默认尺寸）
  · 或拖拽 → 自定义尺寸创建
  · 创建后自动切回选择工具

各元素的默认尺寸/样式:
  button:    padding:8px 16px; border-radius:4px; background:#0D99FF; color:#FFF;
  input:     width:240px; height:36px; border:1px solid #D9D9D9; padding:4px 8px;
  textarea:  width:240px; height:100px; border:1px solid #D9D9D9; padding:8px;
  select:    width:200px; height:36px; border:1px solid #D9D9D9;
  img:       width:200px; height:150px; background:#F0F0F0; (显示占位图标)
  a:         display:inline; color:#0D99FF; text-decoration:underline; 文本="链接"
  video:     width:320px; height:180px; background:#000;

光标: 十字准星 (crosshair)
```

##### T 文本工具

```
图标: T
快捷键: T
下拉菜单:
  ┌────────────────────────────┐
  │  ✓  段落文本 (p)         T │  ← 默认
  │     标题 H1                │
  │     标题 H2                │
  │     标题 H3                │
  │     行内文本 (span)        │
  └────────────────────────────┘

行为（两种模式）:
  点击模式:
    · 点击画布某位置 → 创建一个文本节点（默认宽度 auto）
    · 立即进入行内文本编辑（光标闪烁，可直接输入）
    · 按 Escape 或点击其他位置 → 退出文本编辑

  拖拽模式:
    · 拖拽画一个矩形 → 创建一个定宽文本节点（width = 拖拽宽度）
    · 文本自动换行
    · 立即进入文本编辑

创建的 Schema 节点:
  段落: p + font-size:16px; line-height:1.5; color:#333;
  H1:   h1 + font-size:32px; font-weight:700;
  H2:   h2 + font-size:24px; font-weight:600;
  H3:   h3 + font-size:20px; font-weight:600;
  行内: span + font-size:16px; (宽度 auto)

光标: 文本光标 (text / I-beam)
```

##### 💬 注释工具

```
图标: 💬 (评论气泡)
快捷键: A（与「组件库」的 C 区分）
下拉菜单: 无

行为:
  · 点击画布任意位置 → 创建一个注释标记
  · 注释标记在 Canvas 覆盖层渲染为 📌 图标 + 编号
  · 不在 React DOM 层渲染（不影响最终产品）
  · 不导出到代码

  · 点击注释标记 → 弹出文本编辑气泡
  · 支持 @提及、文字格式、图片粘贴（远期）
  · 注释列表可在左侧面板的"注释"Tab 中查看（远期）

创建的 Schema 节点:
  type: "annotation"（特殊节点类型，不参与渲染）
  props: { text: string; position: { x, y }; author?: string; createdAt: string }

光标: 十字准星 + 评论图标
```

### 2.4 下拉菜单的通用交互

```
下拉菜单行为规范：

触发方式：
  · 长按工具按钮（> 300ms） → 展开下拉
  · 点击工具按钮右下角的 ▾ 小箭头 → 展开下拉
  · 短按工具按钮（< 300ms） → 不展开，直接激活上次选择的子项

下拉菜单 UI：
  ┌──────────────────────────────┐
  │  暗色背景 (#1E1E1E)，圆角 8px │
  │  宽度自适应（最小 180px）      │
  │                              │
  │  菜单项高度: 32px             │
  │  内边距: 8px 12px             │
  │  菜单项 hover: #333333        │
  │  ✓ 勾选标记: 当前选择的子项    │
  │  快捷键右对齐: 灰色 (#999)    │
  │                              │
  │  出现方式: 从工具按钮向上弹出  │
  │  动画: 150ms ease-out 淡入+上滑│
  └──────────────────────────────┘

记忆性：
  · 每个工具记住上次选择的子项
  · 下次短按直接激活该子项
  · 工具按钮的图标更新为当前选择的子项图标
  · 记忆保存在 localStorage，跨会话持久

快捷键与下拉的关系：
  · 快捷键（如 R）激活该工具的"当前选择子项"
  · 如果要切换子项，需通过下拉菜单或子项自己的快捷键（如有）
```

顶部工具栏是固定的水平条，横跨编辑器全宽：

```

│  └──────────┘  └──────────────┘  └───────────┘  └────────────────────┘  │
│                                                                          │
```

### 3.2 四个功能区

#### 区域 A：项目信息（左对齐）

```
┌────────────────────────────────────┐
│  DesignUI   [← 返回]   登录页 ✏️   │
└────────────────────────────────────┘

  [← 返回]
    · 点击 → 返回项目列表/首页
    · 离开前检查未保存变更 → 提示保存

  登录页 ✏️
    · 显示当前编辑的页面名称
    · 点击 ✏️ 或双击文字 → 进入编辑模式，可修改页面名
    · 修改后 blur/Enter → 生成 renameScreen Operation
    · Escape → 取消修改
```

#### 区域 B：编辑控制（居中偏左）

```
┌──────────────────────────────────┐
│  [↩ 撤销] [↪ 重做]  [⊖ 80% ⊕]  │
└──────────────────────────────────┘

  [↩ 撤销] / [↪ 重做]
    · 快捷键: Cmd+Z / Cmd+Shift+Z
    · 基于 OperationExecutor 的 undo/redo 栈
    · 无可撤销操作时 → 按钮置灰 (disabled)
    · tooltip 显示将要撤销/重做的操作描述
      例："撤销: 移动 div.header"

  [⊖ 80% ⊕]
    · 缩放控制组件
    · ⊖ 每次缩小 10%（最小 10%）
    · ⊕ 每次放大 10%（最大 800%）
    · 百分比数字可点击 → 弹出输入框，直接输入精确值
    · 百分比数字右侧可选下拉:
      ┌────────────────────┐
      │  适配画布     Cmd+0 │
      │  100%        Cmd+1 │
      │  ──────────────── │
      │  50%               │
      │  75%               │
      │  100%              │
      │  150%              │
      │  200%              │
      └────────────────────┘
```

### 4.1 状态定义

```typescript
type ToolType =
  | "select"       // 选择工具
  | "container"    // 容器工具 (子类型: flex / grid / absolute)
  | "element"      // 元素工具 (子类型: div / section / nav / header / footer / main)
  | "component"    // 组件工具 (子类型: button / input / textarea / select / img / a / video)

interface ToolState {
  activeTool: ToolType;
  subType?: string;        // 当前子类型
  previousTool: ToolType;  // 用于临时工具切换后的回退
}
```

### 4.3 临时工具切换

```
临时工具的核心理念：按住修饰键临时进入另一个工具，松开后自动回退。

┌──────────────────────────────────────────────────────────┐
│  修饰键        │  临时工具     │  行为                      │
├──────────────────────────────────────────────────────────┤
│  Space（按住） │  抓手 (hand) │  拖拽平移画布               │
│                │              │  松开 → 回到之前的工具      │
├──────────────────────────────────────────────────────────┤
│  Cmd（按住）   │  —           │  与滚轮组合 → 缩放          │
│                │              │  不改变当前工具              │
├──────────────────────────────────────────────────────────┤
│  Alt（按住）   │  —           │  与 hover 组合 → 间距标注   │
│                │              │  不改变当前工具              │
└──────────────────────────────────────────────────────────┘

关键实现细节：
  · Space 检测用 keydown/keyup（不是 keypress）
  · 防止 Space 在文本输入框中触发抓手（检查 document.activeElement）
  · 如果在拖拽过程中松开 Space → 完成拖拽后再切回，不中断拖拽
```

### 4.4 绘制工具的自动回退策略

```
绘制工具（container / element / component / text / annotation）
在创建完元素后，应该自动切回选择工具吗？

策略：默认自动切回，"持续创建"需手动锁定

自动切回（默认行为）：
  · 用户创建一个元素后 → 工具自动切回 select
  · 新元素自动选中，可以立即在右侧面板编辑属性
  · 这是 Figma 的默认行为，符合大多数场景

持续创建模式（双击工具按钮锁定）：
  · 用户双击底部工具栏的某个工具 → 进入"锁定模式"
  · 工具按钮出现一个小锁图标 🔒
  · 创建完元素后不切回 select，可以继续创建下一个
  · 按 V 或 Escape → 退出锁定模式，切回 select
  · 场景：批量创建多个相同类型的元素
```

## 五、快捷键体系

### 5.1 设计原则

```
1. 单键优先 → 最高频的工具用单键（V / F / R / T / C）
2. Cmd 组合 → 编辑操作（复制/粘贴/撤销/重做/包裹等）
3. 无冲突 → 快捷键之间不冲突；与浏览器默认行为不冲突
4. 可预测 → 遵循行业惯例（Figma / Sketch / VS Code 的共有键位）
5. 文本输入豁免 → 当焦点在文本输入框 / input / textarea 时，
                   单键快捷键不生效（避免输入 "v" 变成切换工具）
```

### 5.3 快捷键冲突处理

```
冲突场景及解决方案：

1. 浏览器默认快捷键冲突
   · Cmd+S → 浏览器"保存网页" → 我们用 preventDefault 拦截，改为"保存项目"
   · Cmd+P → 浏览器"打印" → 我们拦截，改为"预览模式"
   · Cmd+A → 浏览器"全选文本" → 我们拦截，改为"全选元素"
   · Cmd+D → 浏览器"添加书签" → 我们拦截，改为"复制副本"
   · 原则: 只拦截与编辑器功能冲突的，不拦截 Cmd+T(新Tab) 等无关快捷键

2. 文本编辑时的冲突
   · 当用户正在编辑文本节点的内容（光标闪烁中）
   · 单键快捷键（V/F/R/T/C/[/]）不生效 → 正常输入字符
   · Cmd 组合键照常生效（Cmd+B 加粗、Cmd+I 斜体等 → 未来支持）
   · Escape → 退出文本编辑，恢复快捷键

3. 面板输入框的冲突
   · 当焦点在右侧面板的数值输入框中
   · 单键快捷键不生效
   · Enter → 提交输入值（不触发"选中子元素"）
   · Escape → 取消输入、恢复焦点到画布

判断规则:
  const isInputFocused =
    document.activeElement?.tagName === 'INPUT' ||
    document.activeElement?.tagName === 'TEXTAREA' ||
    document.activeElement?.contentEditable === 'true';

  if (isInputFocused && !event.metaKey && !event.ctrlKey) {
    return; // 不处理单键快捷键
  }
```

## 八、边界情况与异常处理

| 场景 | 预期行为 |
|------|---------|
| 快捷键在 iframe 内失效 | 确保画布容器获得焦点（focus）后才监听键盘事件 |
| 触控板用户无鼠标中键 | Space+拖拽 和 双指滑动 覆盖平移需求 |
| 无 Cmd 键（Windows/Linux）| 自动映射 Cmd → Ctrl（全局配置） |
| 撤销栈溢出（操作极多）| 限制撤销栈深度（默认 100 步），超出后丢弃最早的操作 |
| 导出时 Schema 有错误 | 导出前校验 Schema → 弹出错误提示 + 定位到问题节点 |
| 全局状态变量为空（未定义）| 顶部工具栏不显示全局状态按钮 |
| 数据集为空（未定义）| 顶部工具栏不显示数据集按钮 |
| 连续快速按快捷键 | 工具切换做 debounce（50ms），避免闪烁 |
| 缩放到极限值 | 10% 时 ⊖ 按钮置灰；800% 时 ⊕ 按钮置灰 |
| 长按工具按钮但不选择子项 | 鼠标移出下拉菜单 → 菜单 300ms 后自动关闭，不改变当前工具 |
| 预览模式下按工具快捷键 | 忽略（预览模式不可编辑），但 Escape / Cmd+P 可退出预览 |

## 十、核心设计决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 工具栏位置？ | 底部浮动（局部工具）+ 顶部固定（全局控制） | 底部靠近画布操作区，手部移动距离最短；顶部放全局信息，一眼可见 |
| 底部工具栏形态？ | Figma 式浮动胶囊 | 不遮挡画布边缘内容；视觉优雅；用户已有心智模型 |
| 下拉菜单语义？ | 按 HTML 语义分组（不按几何形状）| 我们画的是 DOM 结构不是矢量图，语义分组更直觉 |
| 绘制后自动切回？ | 默认切回选择工具 + 双击锁定可持续创建 | 80% 场景创建一个就够了；需要批量时双击锁定 |
| 全局状态放哪？ | 顶部工具栏右侧 | 它影响整页渲染，属于全局操作，且需要高频切换查看效果 |
| 快捷键冲突策略？ | 拦截冲突的浏览器键 + 文本输入时豁免单键 | 在编辑器内提供专业体验，同时不影响正常文本输入 |
| 代码视图形态？ | 左右分屏（画布+代码）| Schema 到代码的映射是核心能力，需要可视化对照 |
| 临时工具机制？ | Space=抓手，松开自动恢复 | Figma 的标准交互，零学习成本 |
