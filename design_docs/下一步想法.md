目前这个设计软件的初衷，就是通过设计软件，将完整的产品逻辑表达出来，放到 schema 里，我们相当于是用 schema 在描述一切产品逻辑和设计。
然后我们可以用这个schema 原始数据，转换成各种我们想要的产物，比如：
- react 前端代码、vue 前端代码、html 前端代码、flutter 代码
等等，都是为了实际上线使用。
因为 schema 其实描述了完整逻辑，接下来可能有两种方式：
- ai 基于schema 制定流程，将 schema 100% 还原成所需产物。
- 我们建设工具层，通过 adapters 适配不同产物转换，同时不同的产物什么组件，什么逻辑，如何进行转化可以定制化的配置模板，基于模板就可以快速完成代码转化。
我倾向于第二个方案，稳定不消耗 token：
- 建设 adapters 层来处理转换差异
- 通过各种场景模板：组件、api、逻辑等等，来适配不同的代码转换诉求
从而能够快速的将想法变为企业级代码落地上线。
原则上来说，如果这个平台设计的足够完美，一切逻辑都可以表达，则可以做到突破。
