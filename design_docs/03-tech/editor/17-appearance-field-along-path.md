# 沿路径的外观场：问题域与发散（素材 / MCP / 编辑器）

> 本文档做 **第一性原理与产品抽象发散**，不定具体 API 名、不定排期。与仓库红线见根目录 `agents.md`。

## 1. 动机（一句话）

大量装饰性 UI 不是「一块矩形 + 纯色」，而是 **在某条几何载体上，让一个或多个视觉量随位置连续变化**；若只能用均宽 stroke、全局渐变或大量 path 拼凑，**意图与表达落差**会迫使设计师和 Agent 付出指数级成本。

---

## 2. 三维分解（你提的三条 + 延伸）

### 2.1 载体：「沿路径」的路径族

路径是 **参数域** 的来源：\(t \in [0,1]\) 通常取 **归一化弧长**（或闭合曲线的 **归一化角度**）。路径模板可发散为：

| 类型 | 说明 |
|------|------|
| **解析曲线** | 圆、椭圆、圆弧、直线段、贝塞尔 / 样条（可闭合或开放） |
| **折线 / 多边形** | 三角形、矩形、任意多边形 **边界**（注意顶点处切向不连续，场如何定义需约定） |
| **复合路径** | 多子路径、带洞轮廓、导入 SVG path |
| **隐式 / 区域边界** | 将来若扩展到「沿等值线」等，仍可把边界抽成参数曲线 |

**延伸思考**

- **方向**：同一路径顺时针 / 逆时针影响「色环」阅读顺序；\(t\) 方向应与 UI 箭头、预览一致。  
- **拓扑**：闭合 vs 开放 → 端点策略（cap / fade / 与另一路径衔接）不同。  
- **奇异点**：尖角、自交、零长度段 → 场是否 **延拓**、是否 **分段定义** \(t\)。

---

### 2.2 外观场：在路径上「挂」什么量

「场」即 **标量或向量随 \(t\) 变化的函数**（离散色标是其采样表示）。可发散为：

- **几何类**：线宽、线宽 + 不对称（如压力笔）、虚线相位与占空比  
- **颜色类**：纯色、多 stop 渐变（沿弧）、与法向相关的假立体高光（若引擎支持）  
- **光学类**：模糊半径、外发光强度、多 pass 光晕（可拆成 **标量场 + 合成顺序**）  
- **材质类**：纹理坐标沿弧滚动（若未来与贴图管线结合）

**延伸思考**

- **多场耦合**：例如「宽度大处自动略提亮」——是 **独立两通道** 还是 **约束 / 预设** 减少自由度。  
- **离散 vs 连续**：色标是 UI 友好离散；底层可 **插值** 或 **纹理化**。  
- **与图层合成关系**：场只描述「这一条 stroke」，还是 **整条 stroke + 下层背景** 的合成模型。

---

### 2.3 渐变与插值：从「两点」到「任意策略」

在 \(t\) 上给定 **锚点** \(\{(t_i, v_i)\}\)，需要 **插值策略** 连接它们：

| 策略 | 用途直觉 |
|------|-----------|
| **分段线性** | 默认、可预测、易实现 |
| **单调三次样条 / Hermite** | 拐点光滑、控制斜率（避免过冲） |
| **幂律 / ease** | 强调近某一端的变化率（如「快细到断」） |
| **阶跃 / 平台** | 分段常数、仪表盘刻度感 |
| **多段独立曲线** | 不同弧段不同 law（「左半边线性、右半边 ease-out」） |
| **解析式** | 高级用户或 MCP 写入公式（产品上要权衡暴露面） |

**延伸思考**

- **非线性** 不仅指 ease，还包括 **\(t\) 本身的重参数化**（弧长 vs 均匀角参 → 圆上同色标在两种参数下视觉不同）。  
- **高维 stop**：一个锚点同时钉 **宽度 + 颜色 + 模糊**（多维锚点），或 **一维锚点 + 多维向量值**。  
- **与「全局渐变」工具的心智对齐**：线性 / 径向 / 角向是 **二维区域** 上的场；沿路径是 **一维子流形** 上的场——编辑器里是否 **共享同一套「色标条」组件**，仅换预览几何，可降低学习成本。

---

## 3. 正交性：避免「每出现一个 UI 就一个新 primitive」

把能力拆成 **可组合轴** 有助于复用：

1. **PathTemplate**（几何从哪来）  
2. **FieldChannels**（哪些量在变、每通道的锚点 + 插值）  
3. **Compositor**（多通道如何合成、与 blur 顺序）  
4. **ExportStrategy**（矢量可解析部分 vs 必须栅格部分）

「语音光圈」只是 **(圆, 宽度场 + 色场, 外发光)** 的一个实例；「进度条描边」可能是 **(圆弧, 宽度常数 + 色场, 无发光)**。产品化时优先 **抬轴** 而不是 **堆实例**。

---

## 4. 与编辑器、MCP、导出的关系（原则级）

- **画布与 JSON 同源**：锚点与曲线数据应 **一处编辑、多处消费**（预览、导出、MCP 读写）。  
- **MCP 暴露的是「意图层参数」**：例如「在圆上增加宽度锚点」而非「插入第 37 条线段」——否则 Agent 与人都难维护。  
- **导出能力边界**：浏览器 / SVG 对「渐变 stroke、变宽 stroke」的支持不均时，需 **显式降级策略**（细分段近似、栅格化、文档说明），避免「画布好看、导出糊掉」。

---

## 5. 分期与风险（方向性）

- **一期**：单一模板（如圆）+ 少数字段（宽、色）+ 简单插值 → 覆盖高频装饰。  
- **二期**：任意 path + 多通道场 + 较丰富插值。  
- **风险**：过度通用导致 **检查器爆炸**；需 **预设库 + 高级折叠面板** 平衡。

---

## 6. 时间与动画（可选延伸）

沿路径的场若引入 **时间 \(u\)**，即 \(v(t, u)\)，则自然衔接 **加载动画、描边绘制、声纹** 等；可与 **环境态 / 时间轴** 产品打通。本文不展开，仅标为 **同一抽象的一维推广**。

---

## 7. 开放问题（留给 spec / ADR）

- 多边形 **顶点** 处：宽度场是否强制连续？是否允许 **尖点断裂**？  
- **多条路径共享一套场**（对称复制）是否一等公民？  
- **与组件 props 绑定**（数据驱动宽度）时的 **约束与校验**。
